# Использование встроенной расширенной аналитики


При работе с данными, размещенными в таблицах и областях данных отчёта,
 таблицах экспресс-отчета или аналитической панели, доступны возможности
 расширенной аналитики:


[![](../opened.gif)![](../closed.gif)Эконометрика](javascript:TextPopup(this))


	Эконометрика предназначена
	 для изучения количественных и качественных экономических взаимосвязей
	 с помощью математических и статистических методов и моделей. Эконометрика
	 используется для прогнозирования экономических процессов.


	Примечание.
	 Работа с эконометрикой доступна, если источником данных таблицы или
	 области данных является база данных временных рядов.


	Методы эконометрики могут быть применены к данным с помощью следующих
	 инструментов продукта «Форсайт. Аналитическая платформа»:
	 «Анализ временных рядов» и
	 «Моделирование и прогнозирование».


	Для применения методов эконометрики:


		- Выделите диапазон данных, размещенный в области данных отчета.


		- Откройте вкладку «Расширенная
		 аналитика» ленты инструментов.


		- В группе «Эконометрика»
		 нажмите кнопку, соответствующую инструменту «Форсайт. Аналитическая платформа»:


			- Анализ временных рядов.
			 Будет открыта [рабочая книга](UiDw.chm::/UiDw_Title.htm),
			 содержащая выделенные данные и использующая в качестве источника
			 данных ту же базу данных временных рядов, что и область данных
			 отчета;


			- Моделирование и прогнозирование.
			 Будет открыта [задача моделирования](UiModelling.chm::/UiModelling_Purpose.htm),
			 которая рассчитывает цепочку моделей, основанных на выделенных
			 данных.


	Используя функциональность выбранного инструмента, выполните эконометрический
	 анализ данных.


[![](../opened.gif)![](../closed.gif)Валидация
 данных](javascript:TextPopup(this))


	Валидация предназначена
	 для проверки данных временных рядов на соответствие заданным условиям
	 и ограничениям.


	Примечание.
	 Работа с валидацией данных доступна, если источником данных области
	 данных отчета является база данных временных рядов.


	Для выполнения валидации данных:


		- Создайте [правило
		 валидации](Data_validation.htm#creat_validationrule) или убедитесь, что требуемое правило уже существует
		 в базе данных временных рядов, которая используется в качестве
		 источника данных области данных отчета.


		- Выделите диапазон данных, размещенный в области данных.


		- Откройте вкладку «Расширенная
		 аналитика» ленты инструментов.


		- Нажмите кнопку «Правила
		 валидации», расположенную
		 в группе «Валидация данных».


		- Будет отображен список, в котором выберите требуемое правило
		 валидации.


	После выполнения последовательности действий для выделенного диапазона
	 данных будет выполнена валидация.


	При работе с исключениями доступны следующие операции:


		- [перемещение](Data_validation.htm#navigate) по
		 исключениям;


		- отображение/скрытие [легенды
		 валидации](Data_validation.htm#validation_legend);


		- [очистка](Data_validation.htm#hide_results) результатов
		 валидации.


	Более подробное описание работы с валидацией данных приведено в
	 разделе «[Валидация
	 данных](Data_validation.htm)».


[![](../opened.gif)![](../closed.gif)Интеллектуальный
 анализ данных](javascript:TextPopup(this))


	Интеллектуальный анализ данных
	 (Data Mining) предназначен для выявления скрытых фактов и взаимосвязей
	 в больших массивах данных. Полученные данные могут использоваться
	 для принятия решений в различных сферах человеческой жизни.


	Для выполнения интеллектуального анализа данных:


		- Выделите диапазон данных, размещенный в области данных отчета.


		- Откройте вкладку «Расширенная
		 аналитика» ленты инструментов.


		- Нажмите кнопку, соответствующую методу анализа, расположенную
		 в группе «Интеллектуальный анализ
		 данных»:


			- [Кластеризация](DataMining.chm::/Master/Methods/DataMining_CategDetect.htm).
			 Разделяет данные на заданное количество групп, используя выбранный
			 метод;


			- [Поиск
			 исключений](DataMining.chm::/Master/Methods/DataMining_ExceptionSearch.htm). Определяет «степень
			 исключительности» для каждого признака каждого объекта на
			 основании всей совокупности данных. На основе полученной характеристики
			 можно ранжировать объекты по их исключительности;


			- [Заполнение
			 по шаблону](DataMining.chm::/Master/Methods/DataMining_PatternSubst.htm). На основании имеющейся
			 классификации заполняет пропущенные значения объясняемого
			 признака в зависимости от значений объясняющих признаков;


			- [Ключевые
			 факторы](DataMining.chm::/Master/Methods/DataMining_KeyInf.htm). Оценивает степень влияния
			 каждого фактора на зависимую переменную и выявляет наиболее
			 значимые факторы;


			- [Анализ
			 ассоциаций](DataMining.chm::/Master/Methods/DataMining_AssosiationAn.htm). Определяет часто встречающиеся
			 совместно наборы элементов на основе анализа множества повторяющихся
			 транзакций;


			- [Прогнозирование](DataMining.chm::/Master/Methods/DataMining_Forecasting.htm).
			 Позволяет продолжить указанный временной ряд выбранным методом
			 прогнозирования, используя информацию о характере его периодичности.


	Для выполнения анализа будет открыт мастер интеллектуального анализа
	 данных.
