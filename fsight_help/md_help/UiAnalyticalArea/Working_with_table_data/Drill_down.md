# Детализация и обобщение данных


	Для получения более подробной или общей информации по данным используйте
	 детализацию или обобщение данных. Различают детализацию и обобщение
	 агрегированных и неагрегированных данных:


		- детализация и обобщение неагрегированных и агрегированных
		 данных. Доступна для всех измерений куба. Требуется настройка
		 [источника
		 данных](uinavobj.chm::/Cube/CreateCube/Master_Standart/UiMd_Cube_CreateCube_Master_Standart_7.htm);


		- детализация данных на основе SQL-запроса. Доступна только
		 для измерения фактов куба. Требуется настройка [источника
		 данных](uinavobj.chm::/Cube/CreateCube/Master_Standart/UiMd_Cube_CreateCube_Master_Standart_2.htm#details_sql_query);


		- детализация агрегированных на сервере данных. Доступна только
		 для измерения фактов куба. Требуется настройка [источника
		 данных](uinavobj.chm::/Cube/CreateCube/Master_Standart/UiMd_Cube_CreateCube_Master_Standart_2.htm#agregation).


	Примечание.
	 [Агрегация](uinavobj.chm::/Cube/Agregation.htm)
	 - процесс объединения данных из нескольких элементов в один. При использовании
	 агрегированных данных детализация позволяет просмотреть элементы измерений
	 и реляционные данные, из которых были сагрегированны данные.


	Детализация и обобщение данных доступны в инструментах «[Аналитические
	 запросы (OLAP)](UiExpress.chm::/purpose/UiExpress_Purpose.htm)», «[Аналитические
	 панели](UiAdhoc.chm::/UiAdhoc_Purpose.htm)» и «[Отчёты](UiReport.chm::/UiReport_purpose.htm)».


	Детализация недоступна, если в выделенный диапазон области данных
	 попадают:


		- ячейки из разных источников;


		- ячейки с [итогами](../Totals/Calculate_totals.htm);


		- ячейки «Прочее», появляющиеся при [распределении](../Analysis/distribution.htm)
		 данных.


## Детализация и обобщение неагрегированных
	 и агрегированных данных


	Детализация/обобщение данных - операция, позволяющая переходить
	 по уровням данных: от общих к частным и наоборот. Доступная глубина
	 детализации определяется иерархией измерений источника данных. При
	 детализации агрегированных фиксированных измерений можно просмотреть
	 элементы, из которых были сагрегированы данные.


	При выполнении детализации/обобщения данных отмечаются родительские/дочерние
	 элементы соответствующего уровня измерения.


	Для отображения исходной отметки, заданной на вкладке «Отметка»
	 боковой панели или, например, с помощью элементов управления, после
	 применения детализации/обобщения данных выполните команду «Вернуться
	 в исходное состояние» в контекстном меню ячейки таблицы.


	Если детализация/обобщение данных выполняется несколько раз и на
	 одной из итераций меняется отметка измерения, то при выполнении команды
	 «Вернуться в исходное состояние»,
	 после очередной детализации/обобщения данных, будет отображена не
	 исходная, а последняя заданная отметка.


	Если при детализации/обобщении попали на уровень, на котором нет
	 данных, необходимо выполнить команду «Вернуться
	 в исходное состояние» в контекстном меню ячейки таблицы.


	Примечание.
	 Команда «Вернуться в исходное состояние»
	 доступна в инструментах «[Аналитические
	 запросы (OLAP)](UiExpress.chm::/purpose/UiExpress_Purpose.htm)», «[Аналитические
	 панели](UiAdhoc.chm::/UiAdhoc_Purpose.htm)» при изменении отметки и последующей
	 детализации/обобщения данных. Если детализация/обобщение данных не
	 выполняется после изменения отметки, то команда будет недоступна.


### Детализация


	Детализация доступна для элементов иерархии источника данных, имеющих
	 дочерние элементы.


	Гиперссылки доступны, если снят флажок «Гиперссылки
	 как простой текст» на вкладке «[Параметры](../Parameters/Parameters.htm)» боковой
	 панели.


	Примечание.
	 Если настроена расшифровка измерения, то при переходе по гиперссылке
	 будет происходить [настроенное
	 действие](../Hyperlinks/Working_with_hyperlinks.htm#hyperlink_click_actions).


	В результате детализации происходит переход на более глубокий уровень
	 иерархии справочника. Отметка перестраивается: включаются все дочерние
	 элементы более глубокого уровня измерения.


#### Детализация по элементам измерения


	Для детализации данных по элементам измерения:


		- щёлкните по гиперссылке в наименовании элемента измерения
		 в боковике или шапке таблицы;


		- выделите ячейки боковика или шапки таблицы, данные которых
		 необходимо детализировать, и выполните команду «Детализировать»
		 в контекстном меню ячеек.


#### Детализация по элементам нескольких измерений


	Для детализации данных по элементам нескольких измерений:


		- выполните команду «Детализировать»
		 в контекстном меню выделенных ячеек таблицы;


	Примечание.
	 В инструменте «Аналитические панели»
	 команда называется «Таблица >
	 Детализировать».


		- щёлкните по гиперссылке в ячейке с зажатой клавишей CTRL.
		 Доступно для ячейки.


	Детализация произойдет по всем измерениям, пересекающимся в данной
	 ячейке, при наличии у них дочерних элементов. Если дочерние элементы
	 есть у одного измерения, то детализация пройдёт по этому измерению.


	При настроенной [агрегации
	 фиксированных измерений](uinavobj.chm::/Cube/Agregation.htm) возможна детализация по агрегированным
	 элементам. В результате фиксированное измерение перейдет в [измерение
	 по строкам](UiSelection.chm::/Selection/Dimension.htm#variant_of_location), отображая все элементы, участвующие
	 в получении агрегированных данных. Для детализации ячеек, данные которых
	 получены в результате агрегации фиксированных измерений:


		- Выполните команду «Детализировать»
		 в контекстном меню ячеек.


	Примечание.
	 В инструменте «Аналитические панели»
	 команда называется «Таблица >
	 Детализировать».


		- Выберите фиксированное измерение, элементы которого необходимо
		 детализировать.


	В результате выполнения операции данное [измерение
	 перейдет в строки](UiSelection.chm::/Selection/Dimension.htm#variant_of_location) и будет содержать элементы, используемые
	 для агрегации диапазона ячеек.


### Обобщение


	Обобщение доступно для элементов иерархии источника данных, имеющих
	 родительские элементы.


	Для обобщения данных по всем дочерним измерениям:


		- щёлкните по гиперссылке в боковике или шапке таблицы по
		 наименованию элемента измерения с зажатой клавишей SHIFT;


		- выполните команду «Обобщить»
		 в контекстном меню выделенной ячейки или диапазона ячеек таблицы.


	Примечание.
	 В инструменте «Аналитические панели»
	 команда называется «Таблица >
	 Обобщить».


	При обобщении по агрегированным элементам измерение из строк перейдет
	 в фиксированное. В других измерениях в строках, столбцах отметка изменится
	 в соответствии с выделенной ячейкой/диапазоном ячеек, для которых
	 происходило обобщение.


	Для обобщения ячеек, данные которых получены в результате агрегации
	 фиксированных измерений:


		- Выполните команду «Обобщить»
		 в контекстном меню ячеек.


	Примечание.
	 В инструменте «Аналитические панели»
	 команда называется «Таблица >
	 Обобщить».


		- Выберите измерение, элементы которого необходимо обобщить.


## Детализация на основе SQL-запроса


	При отсутствии настроенной [агрегации
	 фактов куба](uinavobj.chm::/Cube/CreateCube/Master_Standart/UiMd_Cube_CreateCube_Master_Standart_2.htm#agregation) используйте [детализацию
	 на основе SQL-запроса](uinavobj.chm::/Cube/CreateCube/Master_Standart/UiMd_Cube_CreateCube_Master_Standart_2.htm#details_sql_query). Детализация будет доступна,
	 если в кубе выбран [запрос](UiNavObj.chm::/Cube/CreateCube/Master_Standart/UiMd_Cube_CreateCube_Master_Standart_2.htm#details_sql_query),
	 который будет использоваться для получения исходных реляционных данных.
	 При этом основные принципы работы и ограничения будут такими же, как
	 при настроенной серверной агрегации измерения фактов.


## Детализация агрегированного на сервере
	 измерения фактов


	При настроенной [серверной
	 агрегации измерения фактов](uinavobj.chm::/Cube/CreateCube/Master_Standart/UiMd_Cube_CreateCube_Master_Standart_2.htm#agregation) на последнем уровне
	 иерархии возможна детализация в реляционные данные. В результате выполнения
	 детализации выводятся записи из таблицы фактов, по которым сформировано
	 значение в ячейке в виде плоской таблицы. В боковой панели появляется
	 дополнительное измерение «Поля источника»,
	 размещенное по столбцам и содержащее все поля из таблицы фактов, в
	 том числе поля, не привязанные к измерениям куба.


	Для детализации ячеек, данные которых получены в результате серверной
	 агрегации измерения фактов, выполните команду «Детализировать»
	 в контекстном меню ячеек.


	Примечание.
	 В инструменте «Аналитические панели»
	 команды называются «Таблица >
	 Детализировать».


	[![](../opened.gif)![](../closed.gif)Ограничения
	 аналитических функций в реляционном режиме](javascript:TextPopup(this))


		При работе в реляционном режиме недоступна часть аналитических
		 функций:


			- преобразовать данные;


			- отобразить как ранги;


			- индикаторы роста;


			- итоги по строкам;


			- повернуть таблицу;


			- спарклайны;


			- переход в режим анализа временных рядов;


			- условное форматирование;


			- настройка наименований;


			- настройка оформления боковика;


			- анализ 80/20.


### Группировка реляционных данных


	При представлении данных в реляционном виде доступна группировка
	 данных по столбцам.


	Для группировки данных выберите столбец и выполните команду «Группировать» в контекстном меню
	 столбца.


	Примечание.
	 В инструменте «Аналитические панели»
	 команда называется «Таблица >
	 Группировать».


	Для отмены группировки выберите сгруппированный столбец и выполните
	 команду «Разгруппировать»
	 в контекстном меню столбца.
