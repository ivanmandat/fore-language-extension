# Проблемы при запуске, входе и проверке
 пользователя


При запуске и входе в «Форсайт. Аналитическая платформа»
 в отдельных случаях могут возникать следующие ошибки.


## Проблемы
 при запуске


[![](../opened.gif)![](../closed.gif)Произошло
 нарушение целостности файлов ](javascript:TextPopup(this))


	Проблема


	При запуске продукта «Форсайт. Аналитическая платформа»
	 обнаружено нарушение целостности платформы, вместо окна регистрации
	 появляется следующее сообщение об ошибке:


	Нарушение целостности
	 файлов: <имя файлов>.


	Причина


	Эта проблема может возникнуть, если исполняемые файлы или библиотеки
	 отсутствуют, или не соответствуют эталону.


	Решение


	При возникновении данной ситуации следует обратиться к администратору.

	Проблема


	При запуске продукта «Форсайт. Аналитическая платформа»
	 не появляется сообщение об ошибке, но «Форсайт. Аналитическая платформа»
	 завершает свою работу.


	Если в окне авторизации нажать кнопку «Отмена»,
	 то сообщение об ошибке отображено не будет и «Форсайт. Аналитическая платформа»
	 завершит свою работу.


	Причина


	Возможные причины:


		- при работе с 64-разрядной операционной системой используется программный
		 комплекс DeviceLock версии ниже 7.1;


		- произошли ошибки в процессе установки Framework.


	Решение


	Ниже перечислены варианты решения данной проблемы. Если вариант
	 решения не помогает, перейдите к следующему варианту.


		- Проверьте соответствие версии DeviceLock типу системы, установите
		 программный комплекс DeviceLock более новой версии, например,
		 7.1 или выше.


		- Переустановите Microsoft .NET Framework, предварительно
		 удалив Microsoft .NET Framework Client 4 Profile и Microsoft
		 .NET Framework 4 Exstended.

 найден файл api-ms-win-crt-runtime-l1-1-0.dll ](javascript:TextPopup(this))


	Проблема


	При запуске продукта «Форсайт. Аналитическая платформа»
	 появляется сообщение:


	Запуск программы невозможен,
	 так как на компьютере отсутствует api-ms-win-crt-runtime-l1-1-0.dll.
	 Попробуйте переустановить программу.


	Причина


	Эта проблема может возникнуть, если на рабочей станции удалён или
	 некорректно установлен пререквизит Microsoft Visual C++ 2019 Redistributable
	 Package.


	Решение


	Запустите инсталлятор Microsoft Visual C++ 2019 Redistributable
	 Package, разрядность инсталлятора должна соответствовать разрядности
	 «Форсайт. Аналитическая платформа».
	 Инсталлятор содержится в папке ISSetupPrerequisites,
	 расположенной на одном уровне с инсталлятором продукта «Форсайт. Аналитическая платформа».


	Для установки Microsoft Visual C++ 2019 Redistributable Package
	 на Windows Server 2012 R2 требуются обновления [KB2975061](http://support.microsoft.com/KB/2975061)
	 и [KB2919355](http://support.microsoft.com/KB/2919355).

[![](../opened.gif)![](../closed.gif)Клиент
 Oracle не найден ](javascript:TextPopup(this))


	Проблема


	При запуске продукта «Форсайт. Аналитическая платформа»
	 появляется следующее сообщение с ошибкой:


	Ошибка при подключении
	 к базе данных. Клиент <Oracle 12.x\18.x> не найден.


	Причина


	Возможные причины:


		- клиентская часть СУБД Oracle не установлена;


		- несовпадение разрядностей «Форсайт. Аналитическая платформа»
		 и клиентской части СУБД Oracle.


	Решение


	Проверьте, установлена ли клиентская часть СУБД Oracle и совпадает
	 ли её разрядность с разрядностью «Форсайт. Аналитическая платформа».


[![](../opened.gif)![](../closed.gif)Ошибка
 при подключении к репозиторию на базе СУБД PostgreSQL ](javascript:TextPopup(this))


	Проблема


	При подключении к репозиторию может возникнуть ошибка:


	Ошибка загрузки библиотеки
	 функций PostgreSQL. Возможно, драйвер не установлен.


	Причина


	Отсутствует установленный драйвер PostgreSQL или разрядность драйвера
	 отличается от разрядности «Форсайт. Аналитическая платформа».
	 Также ошибка может проявляться, если на компьютере пользователя отсутствует
	 пакет Microsoft Visual C++ Runtime Library даже при корректно установленном
	 драйвере СУБД.


	Решение


	Проверьте наличие установленного драйвера и его разрядность. Если
	 необходимый драйвер установлен, то скачайте с сайта [www.microsoft.com](http://www.microsoft.com)
	 и установите файл vcredist
	 необходимой разрядности.


	Также рекомендуется в системную переменную PATH добавить путь к
	 папке, в которую установлены драйвера для СУБД PostgreSQL.


[![](../opened.gif)![](../closed.gif)Возникает
 ошибка «Connection to database failed. Oracle 12.x/18.x is not found»
 ](javascript:TextPopup(this))


	Проблема


	При подключении к репозиторию на базе СУБД Oracle через BI-сервер
	 может возникнуть ошибка:


	Connection to database
	 failed. Oracle 12.x/18.x is not found.


	Причина


	Веб-сервер IIS взаимодействует с клиентом Oracle под пользователями
	 Network Service и IUSR, а не под тем пользователем, который обращается
	 к веб-серверу. В доступе к клиенту Oracle может быть отказано, если
	 у пользователей Network Service и IUSR нет соответствующих прав доступа.


	Решение


	Для решения проблемы необходимо предоставить права на доступ к папке
	 с клиентом Oracle пользователям Network Service и IUSR.


[![](../opened.gif)![](../closed.gif)Возникает
 ошибка «The process cannot access the file because it is being used by
 another process» ](javascript:TextPopup(this))


	Проблема


	При подключении к репозиторию может возникнуть ошибка:


	The process cannot
	 access the file because it is being used by another process.


	Причина


	В ОС Linux используется [Tmpfs](https://www.kernel.org/doc/html/latest/filesystems/tmpfs.html).


	Решение


	Для решения проблемы необходимо отказаться от использования Tmpfs,
	 чтобы временные файлы располагались на жестком диске. Файловая система
	 Tmpfs не поддерживает атрибуты, необходимые для корректной работы
	 «Форсайт. Аналитическая платформа».

	Проблема


	При авторизации в «Форсайт. Аналитическая платформа»
	 в окне регистрации появляется следующее сообщение:


	Политикой безопасности
	 предписано изменить пароль.


	Если новый пароль не задать или нажать кнопку «Отмена»
	 появляется сообщение с ошибкой:


	Пользователь обязан
	 сменить пароль.


	Причина


	Эта проблема может возникнуть, если заданный в редакторе политик
	 максимальный срок действия пароля истек.


	Решение


	Необходимо сменить пароль. Для этого в окне с уведомлением о необходимости
	 смены пароля введите пароль и нажмите кнопку «Да».


	Если новый пароль не удовлетворяет каким-либо требованиям парольной
	 политики, то ниже полей ввода будет отображена соответствующая информация.
	 Задайте пароль в соответствии с правилами. После смены пароля сеанс
	 работы с веб-приложением будет завершен. Для входа в систему откройте
	 новое окно обозревателя.

[![](../opened.gif)![](../closed.gif)Сообщение
 об ошибке «Доступ запрещен» при авторизации в веб-приложении
 ](javascript:TextPopup(this))


	Проблема


	При авторизации в веб-приложении появляется сообщение:


	Доступ запрещен.

	Обратитесь к администратору.


	Регистрация в настольном приложении происходит без ошибок.


	Причина


	Эта проблема может возникнуть, если некорректно настроены параметры
	 конфигурационного файла SQLNET.ORA. Файл расположен по пути S\network\admin\,
	 где S - путь до места установки Oracle.


	Решение


	Для корректной парольной авторизации в конфигурационном файле SQLNET.ORA
	 замените SQLNET.AUTHENTICATION_SERVICES = (NTS) на SQLNET.AUTHENTICATION_SERVICES
	 = (NONE).


[![](../opened.gif)![](../closed.gif)Ненайденные
 сервисы при авторизации в веб-приложении ](javascript:TextPopup(this))


	Проблема


	После установки веб-приложения, при попытке пройти процедуру авторизации
	 система выдает ошибку о ненайденных сервисах (ServiceNotFound).


	Причина


	Адрес, по которому будет производиться обращение к веб-сервисам,
	 задается в базовых настройках веб-приложения в файле PP.xml. Адрес
	 к сервисам – регистрозависимый, поэтому при несоблюдении регистра
	 в написании адреса, возникает ошибка о ненайденных сервисах.


	Решение


	Путь, по которому расположены данные файлы, зависит от используемого
	 веб-сервера:


		- IIS: S\Config;


		- Java: S\pp\WEB-INF;


	где S
	 - путь, по которому установлено веб-приложение.


	Открыв на редактирование файл PP.xml,
	 необходимо проверить корректность адреса к веб-сервисам с учетом регистра
	 в строке «service url="S1"
	 />». Где S1 – путь,
	 по которому располагается BI-сервер.


	Пути, используемые по умолчанию, при использовании сервера IIS:


		- http://<BI-сервер>/FPBI_App_v10.x/axis2/services/PP.SOM.Som


	Где <BI-сервер>
	 - наименование или IP-адрес сервера, на котором функционирует BI-сервер.


	После сохранения изменений следует открыть веб-приложение и повторно
	 пройти процедуру авторизации.


[![](../opened.gif)![](../closed.gif)Неверный
 идентификатор репозитория при авторизации в веб-приложении
 ](javascript:TextPopup(this))


	Проблема


	После установки веб-приложения, при попытке пройти процедуру авторизации
	 система выдает ошибку о неверном идентификаторе репозитория.


	Решение


		- Закройте окно браузера и запустите настольное приложение
		 «Форсайт. Аналитическая платформа»
		 с правами Администратора на сервере с установленными веб-сервисами.


		- В открывшемся окне приветствия нажмите на кнопку «Настройка». Из списка доступных

		 в веб-приложении, и нажмите на кнопку «Редактировать».


		- В открывшемся окне установите флажок «Для
		 всех пользователей на компьютере», чтобы репозиторий стал
		 доступным для всех пользователей и веб-сервисы могли взаимодействовать
		 с репозиторием «Форсайт. Аналитическая платформа».


		- После сохранения изменений следует перезапустить веб-сервер,
		 на котором установлены веб-сервисы, открыть веб-приложение и повторно
		 пройти процедуру авторизации.


[![](../opened.gif)![](../closed.gif)Ошибка
 создания/обновления пользователя при работе в репозитории на базе СУБД
 PostgreSQL](javascript:TextPopup(this))


	Проблема


	Во время создания или обновления пользователя возникает ошибка «При
	 раздаче прав на объекты СУБД произошла ошибка. Обновите пользователей».


	Решение


	Если при работе с СУБД PostgreSQL используется схема по умолчанию
	 Public, то у всех пользователей репозитория будет возможность создания
	 таблиц. Данное поведение связано со спецификой работы СУБД PostgreSQL.
	 Это может привести к проблемам обновления пользователей и учёту прав
	 доступа у пользователей, которые являются администраторами репозитория.
	 Для избежания ошибки рекомендуется создавать все объекты только под
	 пользователями, входящими в группу Администраторы.
