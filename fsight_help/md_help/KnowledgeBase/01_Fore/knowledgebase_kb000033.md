## Проблема


При приведении мультиязычных справочников НСИ к интерфейсу [IDimInstance](KeDims.chm::/interface/IDimInstance/IDimInstance.htm)
 свойство [IDimInstance.Elements](KeDims.chm::/interface/IDimInstance/IDimInstance.Elements.htm)
 не содержит элементов, хотя они есть в справочнике.


## Причина


В справочнике НСИ присутствует нарушение [уникальных
 ключей](UiNavObj.chm::/reference_book/Master_RDS_reference_book/Unique_Keys.htm).


Подобная ситуация встречается, если уникальным ключом выбран атрибут,
 переводимый на какой-либо дополнительный язык и перевод двух или более
 элементов совпадает.


Например:


	- по умолчанию язык элементов справочника НСИ - русский;


	- элементы справочника НСИ переводятся на английский язык;


	- уникальным ключом справочника является атрибут «Наименование»;


	- наименование всех элементов на русском языке различается;


	- перевод наименований нескольких элементов на английский язык
	 совпадают.


Результат: если [языком
 репозитория](UiNav.chm::/02_Navigator/Repo_Default.htm)
 является русский, то свойство [IDimInstance.Elements](KeDims.chm::/interface/IDimInstance/IDimInstance.Elements.htm)
 вернет актуальное количество элементов в справочнике НСИ; если языком
 репозитория является английский, то свойство [IDimInstance.Elements](KeDims.chm::/interface/IDimInstance/IDimInstance.Elements.htm)
 не вернет ни одного элемента.


## Решение


Уникальный ключ должен состоять из атрибута, значения которого уникальны
 для каждого элемента справочника НСИ на любом языке. Если справочник НСИ
 не имеет единственного уникального атрибута, то ключ может быть составлен
 из нескольких атрибутов, совокупность значений которых гарантирует уникальность
 значений на всех языках.
