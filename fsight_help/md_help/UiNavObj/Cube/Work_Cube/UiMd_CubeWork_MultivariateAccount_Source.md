# Источники данных


Для определения источников данных многомерного расчета на сервере БД
 и их первичного множества перейдите на страницу «Источники
 данных»:


![](MultivariateAccount%20Source.png)


На вкладке мастера доступны следующие операции:


[![](../../Opened.gif)![](../../Closed.gif)Добавление источников](javascript:TextPopup(this))


	Для добавления источника:


		- нажмите кнопку «Добавить»;


		- выполните команду «Добавить»
		 в контекстном меню области списка «Источники»;


		- дважды щёлкните по свободной от элементов области списка.


	После выполнения команды будет открыт стандартный диалог выбора
	 источников.


	Примечание.
	 В качестве источников многомерного расчета можно использовать только
	 стандартные кубы, факты которых должны быть [привязаны](../CreateCube/Master_Standart/UiMd_Cube_CreateCube_Master_Standart_2.htm)
	 к полям только числового типа.


	При добавлении источников необходимо выполнение следующих условий:


		- источники данных должны иметь одинаковый тип СУБД;


		- источники и объекты, которые входят в источник, должны принадлежать
		 одной базе данных;


		- если источники находятся на одном сервере, но в разных схемах,
		 то для их базы данных должен быть установлен флажок «[Использовать
		 учетные данные, указанные при входе в систему](../../Link_with/UiDb_relational_LinkWith_master_property.htm)».


	Если источники не удовлетворяют данным условиям, будет выдано сообщение
	 о невозможности добавления источника.


	Примечание.
	 Если куб-источник построен на запросе/представлении, то текст SQL-запроса,
	 который используется для извлечения данных, должен начинаться с ключевого
	 слова Select.


	Важно.
	 Если источник данных не содержит данных, то [расчёт
	 на сервере БД](UiMd_CubeWork_MultivariateAccount_EditFormula.htm) не будет выполняться.


[![](../../Opened.gif)![](../../Closed.gif)Выделение первичного
 множества источников](javascript:TextPopup(this))


	Из добавленных источников следует выделить первичное множество,
	 которое влияет на извлечение данных.


	По данным кубов, входящих в первичное множество, формируется множество
	 точек, по которым будет производиться расчет. Из кубов, не входящих
	 в состав первичного множества, будут извлечены только данные, соответствующие
	 сформированному множеству точек.


	Примечание.
	 Первые добавленные источники будут включены в первичное множество
	 автоматически.


	Для включения/исключения источника из первичного множества:


		- установите/снимите флажок напротив наименования куба;


		- выполните команду «Включить
		 в первичное множество»/«Исключить
		 из первичного множества» в контекстном меню куба.


	Исключение источника из первичного множества будет недоступно, если
	 данный источник является единственным источником в первичном множестве.


[![](../../Opened.gif)![](../../Closed.gif)Определение
 порядка источников](javascript:TextPopup(this))


	Порядок расположения кубов не влияет на результаты расчетов, но
	 от него зависит порядок формирования множества точек, по которым будет
	 производиться расчет.


	Для изменения порядка расположения источников:


		- используйте кнопки «Вверх»
		 и «Вниз»;


		- выполните команды «Вверх»
		 и «Вниз» в контекстном
		 меню куба;


		- переместите источник в необходимую позицию с помощью механизма
		 Drag&Drop.


	При изменении порядка расположения источников следует пересоздать
	 [таблицу формул](UiMd_CubeWork_MultivariateAccount_Tables.htm).


[![](../../Opened.gif)![](../../Closed.gif)Переход к объекту](javascript:TextPopup(this))


	Для просмотра объекта в навигаторе объектов выполните команду «Перейти к объекту» в контекстном
	 меню источника данных. После выполнения команды фокус в окне навигатора
	 объектов будет перемещен к выбранному кубу.


[![](../../Opened.gif)![](../../Closed.gif)Удаление источников](javascript:TextPopup(this))


	Для удаления одного или нескольких выбранных источников:


		- нажмите кнопку «Удалить»;


		- выполните команду «Удалить»
		 в контекстном меню выделенных источников.


	После выполнения одного из действий выдается подтверждение выполняемого
	 действия.


## Ограничения


В связи с имеющимися ограничениями в реализации многомерного расчета
 накладываются определенные требования к структуре используемых источников:


	- При выборе источника необходимо, чтобы его структура удовлетворяла
	 следующему правилу: иерархия измерения
	 должна быть основана на том же индексе,
	 который использован в самом источнике для привязки
	 измерения. Несоблюдение этого правила приведет к некорректным
	 результатам вычисления формул.


	- Если какой-либо факт
	 источника привязан к нескольким полям
	 различных таблиц с данными, то при расчете будут использоваться данные
	 только первой привязанной
	 таблицы.
