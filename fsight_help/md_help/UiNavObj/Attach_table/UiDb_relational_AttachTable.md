# Присоединенная таблица


Присоединенные таблицы отличаются от обычных таблиц тем, что при удалении
 присоединенной таблицы физический объект базы данных, на который она ссылается,
 не удаляется. Таким образом, обычная таблица может являться источником
 для построения нескольких присоединенных таблиц.


## Создание присоединенной таблицы


Процедура создания присоединенной таблицы используется в случае, когда
 необходимо создать новые таблицы в репозитории на основе уже существующих
 таблиц на сервере.


Для создания присоединенной таблицы в [навигаторе
 объектов](GetStarted.chm::/Interface/Interface_Navigator.htm):


	- в веб-приложении нажмите кнопку ![](../Cube/CreateCube/Master_Standart/add_button.png) «Создать» в главном меню и выберите
	 тип объекта на боковой панели «[Новый объект](UiNav.chm::/02_Navigator/General_Principles_of_Work.htm#add_object)»;


	- в настольном приложении:


		- Выполните одно из действий:


			- выполните команду «Создать >
			 Таблица» в контекстном меню;


			- нажмите кнопку «Новый
			 объект», расположенную в группе «Создать»
			 на вкладке «[Главная](GetStarted.chm::/Interface/Interface_Description.htm#customize_ribbon)»
			 ленты инструментов, и выберите пункт «Таблица».


		- Выберите тип создаваемого
		 объекта на странице «Тип объекта».


После выполнения
 действий в мастере присоединенной таблицы задайте настройки на страницах:


	- [Базовые
	 настройки/Базовые свойства](Master/base_settings.htm). Задаются наименование
	 объекта, идентификатор и примечание;


	- [Параметры](Master/parameters.htm).
	 Формируется список параметров. Страница доступна только в настольном
	 приложении;


	- [Свойства/Выбор базы данных](Master/UiDb_relational_AttachTable_master_database.htm).
	 Указывается база данных, таблицы которой будут использоваться для
	 создания присоединенных таблиц;


	- [Выбор
	 таблиц](Master/UiDb_relational_AttachTable_master_table.htm). Указываются таблицы, на базе которых в
	 репозитории будут созданы присоединенные таблицы. Страница доступна
	 только в настольном приложении.


В веб-приложении при открытии мастера присоединенной таблицы
 становятся доступными дополнительные страницы:


	- [Поля](Master/fields.htm).
	 Формируется основная структура полей присоединенной таблицы.


	- [Индексы](../Table/Master/UiDb_relational_table_master_Index.htm). Формируется
	 список индексов, которые будут использоваться для проверки уникальности
	 данных. Настройка аналогична настройке таблицы;


	- [Ограничения](../Table/Master/UiDb_relational_table_master_contingencies.htm).
	 Формируется список ограничений, которые будут использоваться для автоматического
	 контроля вводимых данных. Настройка аналогична настройке таблицы;


	- [Триггеры](../Table/Master/UiDb_relational_table_master_trigger.htm). Формируется
	 список триггеров, выполняющих определенные действия при наступлении
	 заданных событий. Настройка аналогична настройке таблицы.


Для сохранения настроек в мастере присоединенной таблицы:


	- в веб-приложении:


		- нажмите кнопку «Сохранить»;


		- выполните команду «Сохранить
		 и закрыть» в раскрывающемся меню кнопки «Сохранить».
		 После этого будут сохранены заданные настройки, закрыт мастер
		 и выполнен переход в [навигатор
		 объектов](getstarted.chm::/Interface/Interface_Navigator.htm);


Примечание.
 Команда «Сохранить и закрыть»
 доступна только при [редактировании
 настроек](UiDb_relational_AttachTable.htm#edit).


	- в настольном приложении нажмите кнопку «Готово».


## Работа с присоединенной таблицей


[![](../Opened.gif)![](../Closed.gif)Редактирование
 настроек](javascript:TextPopup(this))


	Для редактирования набора данных выделите соответствующий объект
	 в [навигаторе
	 объектов](GetStarted.chm::/Interface/Interface_Navigator.htm) и выполните одно из действий:


		- в веб-приложении:


			- выполните команду «Редактировать» в контекстном
			 меню объекта;


			- выполните команду «Редактировать в новой вкладке»
			 в контекстном меню объекта;


			- выполните команду «Редактировать в новом окне»
			 в контекстном меню объекта;


			- нажмите клавишу F4.


	Затем нажмите кнопку ![](../b_settings_web.png) «Настройки»;


		- в настольном приложении:


			- нажмите кнопку ![](../reference_book/Work/b_edit.png)
			 «Редактировать» на вкладке
			 «Главная» ленты инструментов;


			- выполните команду «Редактировать»
			 в контекстном меню объекта;


			- нажмите клавишу F4.


	После выполнения одного из действий будет открыт мастер присоединенной
	 таблицы. Измените настройки на соответствующих страницах мастера.


[![](../Opened.gif)![](../Closed.gif)Просмотр данных](javascript:TextPopup(this))


Для просмотра данных выделите соответствующий объект в [навигаторе
 объектов](GetStarted.chm::/Interface/Interface_Navigator.htm) и выполните одно из действий:


	- в веб-приложении:


		- выполните команду «Открыть»
		 в контекстном меню объекта;


		- выполните команду «Открыть
		 в новой вкладке» в контекстном меню объекта;


		- выполните команду «Открыть
		 в новом окне» в контекстном меню объекта;


		- дважды щёлкните по объекту;


		- нажмите клавишу ENTER;


	- в настольном приложении:


		- нажмите кнопку ![](../table/work/b_open.png) «Открыть»
		 на вкладке «Главная»
		 ленты инструментов;


		- нажмите кнопку ![](../table/work/b_open.png) «Открыть
		 > В инструменте по умолчанию» на вкладке «Главная» ленты инструментов;


		- выполните команду «Открыть»
		 в контекстном меню объекта.


		- дважды щёлкните по объекту.


После выполнения одного из действий будет открыт соответствующий объект
 с данными. Для получения подробной информации о принципах работы с набором
 данных обратитесь к разделу «[Работа
 с набором данных](../Table/Work/UiDb_relational_table_work.htm)».


Для просмотра данных в отчете в настольном приложении:


	- нажмите кнопку «Открыть
	 > Конструктор регламентных отчётов» на вкладке «Главная» ленты инструментов;


	- выполните команду «Открыть
	 с помощью > Конструктор регламентных отчётов» в контекстном
	 меню объекта.


Для [импорта данных](../Data_import_wizard/Data_import_wizard.htm)
 в настольном приложении:


	- нажмите кнопку «Открыть >
	 Мастер импорта данных» на вкладке «Главная»
	 ленты инструментов;


	- выполните команду «Открыть
	 с помощью > Мастер импорта данных» в контекстном меню объекта.


[![](../Opened.gif)![](../Closed.gif)Обновление](javascript:TextPopup(this))


	Поскольку присоединенная таблица только ссылается на физическую
	 таблицу, то изменения, произведённые в физическом объекте, на основании
	 которого построена присоединенная таблица, не отображаются в присоединенной
	 таблице.


Для обновления информации о таблице выполните команду ![](../Table/update.png) «Обновить
 информацию о таблице» в контекстном меню таблицы в навигаторе объектов.
 После чего будет обновлена следующая информация из базы данных:


	- количество строк и столбцов в таблице;


	- тип данных полей;


	- список [индексов](../Table/Master/UiDb_relational_table_master_Index.htm);


	- список [ограничений](../Table/Master/UiDb_relational_table_master_contingencies.htm)
	 на поля таблицы;


	- список [триггеров](../Table/Master/UiDb_relational_table_master_trigger.htm);


	- время последнего обновления информации о таблице в [журнале изменений](UiNav.chm::/02_Navigator/UiNav_ObjVer.htm).


Данные, содержащиеся в таблице, автоматически обновляются в соответствии
 с полученной информацией о таблице при её открытии.


Если таблица открыта на просмотр и обновлена информация о ней, то для
 обновления данных:


	- в веб-приложении нажмите кнопку ![](../Table/update.png) «Обновить содержимое таблицы» на
	 панели инструментов;


	- в настольном приложении:


		- нажмите кнопку ![](../Table/Work/View_Panel_Button_12.gif) «Обновить
		 содержимое таблицы» на панели инструментов;


		- выполните команду «Таблица >
		 Обновить» в главном меню;


		- нажмите клавишу F5.


После выполнения действий содержимое таблицы будет обновлено.
