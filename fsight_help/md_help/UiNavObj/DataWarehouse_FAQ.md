# Вопросы и ответы


[![](Opened.gif)![](Closed.gif)Возможно
 ли создание ключевых показателей эффективности (KPI) на основе данных
 из любых приложений, указанных в заявке на коммерческое предложение (RFP)?](javascript:TextPopup(this))


	В «Форсайт. Аналитическая платформа»
	 ключевые показатели эффективности могут выступать измерениями в источнике
	 данных. Есть возможность на прикладном уровне реализовать интеграцию
	 платформы с различными прикладными системами для извлечения и загрузки
	 данных о ключевых показателях эффективности, используя встроенный
	 [ETL-инструментарий](UiETL.chm::/Desktop/01_General_Info/UiETL_General.htm),
	 веб-сервисы и другие инструменты.


[![](Opened.gif)![](Closed.gif)Сколько
 источников данных изначально поддерживаются? Поддерживаются ли: MDX, XML,
 RSS-каналы, Hadoop, неструктурированные файлы и др.?](javascript:TextPopup(this))


	«Форсайт. Аналитическая платформа»
	 поддерживает следующие источники данных:


		- ODBC;


		- OLEDB;


		- драйверы к ряду СУБД;


		- аппаратно-программный комплекс Netezza;


		- СУБД с поколоночным хранением Sybase IQ;


		- многомерные источники: SQL Server Analysis Services, SAP
		 NetWeaver BW и другие, совместимые с OLEDB for OLAP;


		- XML-файлы (поддерживаются с помощью [ETL-инструмента](UiETL.chm::/Desktop/01_General_Info/UiETL_General.htm));


		- файлы MS Excel;


		- неструктурированные файлы: vpf, dbase, Paradox, csv, txt.


	Выполнение MDX-запросов к источникам данных поддерживается на OLAP-серверах:
	 SQL Server Analysis Services, SAP NetWeaver BW, Oracle Essbase и других,
	 совместимых с OLEDB for OLAP.


	Поддержка других источников, таких как RSS, Hadoop, возможна на
	 прикладном уровне.


[![](Opened.gif)![](Closed.gif)Поддерживает
 ли «Форсайт. Аналитическая платформа»
 источники данных: XML, RSS-каналы, плоские файлы и др.?](javascript:TextPopup(this))


	Помимо реляционных и mdx-источников данных «Форсайт. Аналитическая платформа»
	 поддерживает в качестве источников данных плоские текстовые файлы
	 (txt, csv), плоские файлы dbase, Paradox, excel-файлы, mdb-файлы,
	 xml-файлы с возможностью указать xpath-запрос для выборки, json-файлы
	 и rest-сервисы.


	Поддержка других источников, таких как RSS-каналы, возможна на прикладном
	 уровне.


[![](Opened.gif)![](Closed.gif)Возможно
 ли отслеживать время ответа на запрос?](javascript:TextPopup(this))


	«Форсайт. Аналитическая платформа»
	 формирует журнал выполнения запросов, но он недоступен в пользовательском
	 интерфейсе. Его визуализация доступна на прикладном уровне.


[![](Opened.gif)![](Closed.gif)Могут
 ли пользователи соединять источники данных без программирования?](javascript:TextPopup(this))


	В «Форсайт. Аналитическая платформа»
	 бизнес-пользователи могут соединять источники данных на уровне создания
	 куба, на уровне [OLAP-отчета](UiExpress.chm::/purpose/UiExpress_Tree_Multivariate_Object.htm)
	 по общим измерениям, на уровне определения виртуального куба. Использование
	 программирования не требуется.


[![](Opened.gif)![](Closed.gif)Могут
 ли пользователи создавать расчёты и фильтровать данные нескольких источников
 без программирования?](javascript:TextPopup(this))


	В «Форсайт. Аналитическая платформа»
	 пользователи могут создавать расчёты и фильтровать данные одновременно
	 на основе нескольких источников на уровне определения куба данных,
	 связывающего источники, на уровне [OLAP-отчета](UiExpress.chm::/purpose/UiExpress_Tree_Multivariate_Object.htm)
	 (multi-OLAP), а также путем настройки [виртуального
	 куба](Cube/CreateCube/Master_Virtual_Cube/UiMd_Cube_CreateCube_Master_Virtual.htm) из нескольких источников и определения вычислений. Использование
	 программирования не требуется.


[![](Opened.gif)![](Closed.gif)Могут ли пользователи создать вычисляемые элементы?](javascript:TextPopup(this))


	В «Форсайт. Аналитическая платформа»
	 поддерживается возможность создания вычисляемых элементов данных на
	 уровне [таблиц](Table/Master/UiDb_relational_table_master_field.htm),
	 [справочников](reference_book/Master_Table_reference_book/UiMd_reference_book_Master_Table_page3.htm),
	 [кубов](equation_editor/UiMd_Equation_editor_CalculationCube.htm),
	 [отчетов](UiExpress.chm::/Express/Data_sources_parameters.htm)
	 через визуальные мастера. При этом используется единый механизм трансформаций.
	 Пользователи могут создавать вычисляемые кубы для разнообразных расчетов
	 (например, методик оценки и т.п.) без использования программирования.


[![](Opened.gif)![](Closed.gif)Возможна
 ли интеграция с другими ETL или инструментами моделирования?](javascript:TextPopup(this))


	В «Форсайт. Аналитическая платформа»
	 специальных механизмов интеграции не предусмотрено. Интеграция с ETL-инструментами
	 и инструментами моделирования данных возможна через общие БД, файлы,
	 веб-сервисы и т.п. и может потребовать кастомизации платформы.


[![](Opened.gif)![](Closed.gif)Есть
 ли поддержка Open Database Connectivity (ODBC), Java Database Connectivity
 (JDBC)?](javascript:TextPopup(this))


	«Форсайт. Аналитическая платформа»
	 поддерживает стандарты ODBC, OLEDB, OLEDB for OLAP.


[![](Opened.gif)![](Closed.gif)Необходимо
 ли программирование для добавления или загрузки источника данных?](javascript:TextPopup(this))


	В «Форсайт. Аналитическая платформа»
	 добавление источников данных не требует программирования и выполняется
	 через визуальный интерфейс.


	Для выполнения базовых операций по загрузке данных над типовыми
	 источниками данных со стандартной структурой программирование не требуется
	 (любые ODBC и OLEDB-совместимые источники). Источники на основе Excel
	 могут быть загружены непосредственно из BI-инструментов при помощи
	 простой команды импорта с возможностью привязки к уже созданным в
	 платформе измерениям. Для обработки источников данных с нестандартной
	 структурой может потребоваться программирование.


[![](Opened.gif)![](Closed.gif)Предоставляет
 ли доступ к по меньшей мере 50 нативным источникам данных, включая доступ
 по уровню API и метаданным?](javascript:TextPopup(this))


	«Форсайт. Аналитическая платформа»
	 использует нативные адаптеры для Oracle, MS SQL Server, PostgreSQL,
	 Excel, XML, плоских файлов vpf, dbase, Paradox, csv, txt. Для доступа
	 к другим СУБД возможно использовать ODBC, OLEDB, OLEDB for OLAP. Также возможна реализация специфических адаптеров
	 на прикладном уровне в процессе кастомизации платформы.


[![](Opened.gif)![](Closed.gif)Поддерживает
 ли «Форсайт. Аналитическая платформа»
 прямой доступ через SQL запросы и/ или MDX](javascript:TextPopup(this))?


	«Форсайт. Аналитическая платформа»
	 поддерживает возможность создания в репозитории «виртуальных» таблиц
	 на основе SQL-запросов к источникам данных с возможностью использования
	 в отчетах, кубах и т.д. Возможность отправки пользовательских MDX-запросов
	 возможна на прикладном уровне. Данные могут извлекаться из источников
	 «на лету» без необходимости перезагрузки в хранилище данных платформы.


[![](Opened.gif)![](Closed.gif)Как настроить подключение к базе данных при доменной авторизации?](javascript:TextPopup(this))


	При необходимости одновременной работы доменных и не доменных пользователей,
	 необходимо [настроить
	 подключение к базе данных](database/UiDb_database_master_property.htm) следующим образом:


		- Задайте использование парольной авторизации.


		- Снимите флажок «Использовать
		 учетные данные, указанные при входе в систему» и введите
		 логин и пароль текущего пользователя.


		- Установите флажок «Управление
		 правами на объекты СУБД».


		- При использовании СУБД Oracle в [дополнительных
		 настройках](database/database_additional_adjustment.htm) укажите префикс с сервера Oracle.
