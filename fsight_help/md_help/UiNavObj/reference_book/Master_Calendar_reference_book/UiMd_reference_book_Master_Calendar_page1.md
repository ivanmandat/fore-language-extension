# Тип/актуальность


На странице «Тип/актуальность»
 задаётся тип иерархии календаря и календарный период, который будет формироваться
 в справочнике.


	 Веб-приложение Настольное приложение


		![](Type_web.png)


		![](Calendar_directory_type.png)


На данной странице определяются следующие атрибуты:


	- Тип календаря. Следует
	 выбрать тип иерархии календаря:


		- Годы - Полугодия - Кварталы - Месяцы - Дни;


		- Годы - (Полугодия - Кварталы - 9 месяцев) - Месяцы - Дни;


		- Годы - (Полугодия - 9 месяцев) - Кварталы - Месяцы - Дни;


		- Годы - Полугодия - Кварталы - Месяцы - Недели - Дни.


В зависимости от выбранного типа календаря
 формируются уровни календарного справочника;


	- Год начала и окончания.
	 Для создания динамического справочника годы начала и окончания можно
	 не определять. В этом случае при открытии справочника эти данные будут
	 запрошены. Для фиксации даты начала или окончания периода необходимо
	 указать требуемые значения в соответствующих полях «Фиксировать
	 как»;


	- Обратный
	 порядок элементов в справочнике. Флажок устанавливается в том
	 случае, если элементы справочников необходимо расположить в обратном
	 порядке;


	- Сдвиг последнего
	 уровня на точку вперед. Установка данного флажка позволяет
	 сдвигать наименования элементов справочника на месяц вперед. При установке
	 флажка «Коррекция иерархии»
	 также меняется структура справочника таким образом, что элементы с
	 новым названием находятся в нужном году, т.е. если Декабрь 2000
	 имел родительский элемент 2000, то в перестроенном справочнике родительский
	 элемент будет 2001;


	- Замена
	 наименования последнего уровня на день. Установка флажка преобразует
	 наименования элементов справочника в вид dd.mm.yyyy:


Январь 2000 => 01.01.2000


Февраль 2000 => 01.02.2000


.......


Декабрь 2000 => 01.12.2000
