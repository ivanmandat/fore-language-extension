# Представление


Представление строится на основе таблиц и является логической таблицей.
 Содержание данной таблицы выбирается из базовых таблиц путём выполнения
 SQL-запросов. Представление только отображает данные базовых таблиц без
 возможности их редактирования, может являться источником данных для измерений
 и кубов.


## Создание представления


Для создания представления в [навигаторе
 объектов](GetStarted.chm::/Interface/Interface_Navigator.htm):


	- в веб-приложении нажмите кнопку ![](../Cube/CreateCube/Master_Standart/add_button.png) «Создать» в главном меню и выберите
	 тип объекта на боковой панели «[Новый объект](UiNav.chm::/02_Navigator/General_Principles_of_Work.htm#add_object)»;


	- в настольном приложении:


		- Выполните одно из действий:


			- выполните команду «Создать >
			 Таблица» в контекстном меню;


			- нажмите кнопку «Новый
			 объект», расположенную в группе «Создать»
			 на вкладке «[Главная](GetStarted.chm::/Interface/Interface_Description.htm#customize_ribbon)»
			 ленты инструментов, и выберите пункт «Таблица».


		- Выберите тип создаваемого
		 объекта на странице «Тип объекта».


После выполнения
 действий в мастере представления задайте настройки на страницах:


	- [Базовые
	 настройки/Базовые свойства](Master/base_settings.htm). Задаются наименование
	 объекта, идентификатор и примечание;


	- [Параметры](Master/parameters.htm).
	 Формируется список параметров. Страница доступна только в настольном
	 приложении;


	- [Свойства](Master/UiDb_relational_conception_master_Property.htm).
	 Указывается база данных, физическое имя и текст запроса для представления;


	- [Индексы](Master/UiDb_relational_conception_master_index.htm).
	 Формируется список индексов, которые будут использоваться для проверки
	 уникальности данных.


Для сохранения настроек в мастере представления:


	- в веб-приложении:


		- нажмите кнопку «Сохранить»;


		- выполните команду «Сохранить
		 и закрыть» в раскрывающемся меню кнопки «Сохранить».
		 После этого будут сохранены заданные настройки, закрыт мастер
		 и выполнен переход в [режим
		 редактирования](../Table/Work/Changing_Database_Query.htm) данных;


Примечание.
 Команда «Сохранить и закрыть»
 доступна только при [редактировании
 настроек](UiDb_relational_conception.htm#edit).


	- в настольном приложении нажмите кнопку «Готово».


## Работа с представлением


[![](../Opened.gif)![](../Closed.gif)Редактирование
 настроек](javascript:TextPopup(this))


	Для редактирования выбранного в навигаторе объектов представления:


		- в веб-приложении:


			- выполните команду «Редактировать»
			 в контекстном меню объекта;


			- выполните команду «Редактировать
			 в новой вкладке» в контекстном меню объекта;


			- выполните команду «Редактировать в новом окне»
			 в контекстном меню объекта;


			- нажмите клавишу F4.


	Затем нажмите кнопку ![](../b_settings_web.png) «Настройки»;


		- в настольном приложении:


			- нажмите кнопку ![](../reference_book/Work/b_edit.png)
			 «Редактировать» на
			 вкладке «Главная»
			 ленты инструментов;


			- выполните команду «Редактировать»
			 в контекстном меню объекта;


			- нажмите клавишу F4.


	После чего будет открыт мастер представления. Измените настройки
	 на соответствующих страницах мастера.


[![](../Opened.gif)![](../Closed.gif)Просмотр данных](javascript:TextPopup(this))


Для просмотра данных выделите соответствующий объект в [навигаторе
 объектов](GetStarted.chm::/Interface/Interface_Navigator.htm) и выполните одно из действий:


	- в веб-приложении:


		- выполните команду «Открыть»
		 в контекстном меню объекта;


		- выполните команду «Открыть
		 в новой вкладке» в контекстном меню объекта;


		- выполните команду «Открыть
		 в новом окне» в контекстном меню объекта;


		- дважды щёлкните по объекту;


		- нажмите клавишу ENTER;


	- в настольном приложении:


		- нажмите кнопку ![](../table/work/b_open.png) «Открыть»
		 на вкладке «Главная»
		 ленты инструментов;


		- нажмите кнопку ![](../table/work/b_open.png) «Открыть
		 > В инструменте по умолчанию» на вкладке «Главная» ленты инструментов;


		- выполните команду «Открыть»
		 в контекстном меню объекта.


		- дважды щёлкните по объекту.


После выполнения одного из действий будет открыт соответствующий объект
 с данными. Для получения подробной информации о принципах работы с набором
 данных обратитесь к разделу «[Работа
 с набором данных](../Table/Work/UiDb_relational_table_work.htm)».


Для просмотра данных в отчете в настольном приложении:


	- нажмите кнопку «Открыть
	 > Конструктор регламентных отчётов» на вкладке «Главная» ленты инструментов;


	- выполните команду «Открыть
	 с помощью > Конструктор регламентных отчётов» в контекстном
	 меню объекта.


Для [импорта данных](../Data_import_wizard/Data_import_wizard.htm)
 в настольном приложении:


	- нажмите кнопку «Открыть >
	 Мастер импорта данных» на вкладке «Главная»
	 ленты инструментов;


	- выполните команду «Открыть
	 с помощью > Мастер импорта данных» в контекстном меню объекта.


[![](../Opened.gif)![](../Closed.gif)Обновление](javascript:TextPopup(this))


	Поскольку присоединенная таблица только ссылается на физическую
	 таблицу, то изменения, произведённые в физическом объекте, на основании
	 которого построена присоединенная таблица, не отображаются в присоединенной
	 таблице.


	Для обновления информации о таблице, на основе которой построено
	 представление, выполните команду ![](../Table/update.png) «Обновить информацию о таблице» в
	 контекстном меню представления в навигаторе объектов. После чего будет
	 обновлена следующая информация из базы данных:


		- количество строк и столбцов в таблице;


		- тип данных полей;


		- список [индексов](../Table/Master/UiDb_relational_table_master_Index.htm);


		- время последнего обновления информации о таблице в [журнале изменений](UiNav.chm::/02_Navigator/UiNav_ObjVer.htm).


	Данные, содержащиеся в представлении, автоматически обновляются
	 в соответствии с полученной информацией о таблице при открытии представления.


	Если представление открыто на просмотр и обновлена информация о
	 таблице, то для обновления данных:


		- в веб-приложении нажмите кнопку ![](../Table/update.png) «Обновить содержимое таблицы»
		 на панели инструментов;


		- в настольном приложении:


			- нажмите кнопку ![](../Table/Work/View_Panel_Button_12.gif) «Обновить содержимое таблицы»
			 на панели инструментов;


			- выполните команду «Таблица >
			 Обновить» в главном меню;


			- нажмите клавишу F5.


	После выполнения действий содержимое представления будет обновлено.


	Если при создании представления был установлен флажок «[Сохраняемое представление](Master/UiDb_relational_conception_master_Property.htm)», то обновление
	 данных в соответствии с базовыми таблицами выполняется с помощью команды
	 «Обновить данные» в контекстном
	 меню представления. Если флажок снят, то данные обновляются автоматически
	 при открытии представления.


	Примечание.
	 Обновление данных сохраняемого представления доступно только в настольном
	 приложении.


[![](../Opened.gif)![](../Closed.gif)Пересоздание](javascript:TextPopup(this))


	Примечание.
	 Доступно только в настольном приложении.


	Пересоздание представления необходимо для обновления метаданных
	 представления на сервере БД.


	Для пересоздания представления выполните команду «Пересоздать
	 представление» в контекстном меню представления.


[![](../Opened.gif)![](../Closed.gif)Управление версиями](javascript:TextPopup(this))


	Примечание.
	 Доступно только в настольном приложении.


	Если репозиторий [подключен
	 к системе управления версиями](uidevenv.chm::/04_NavigatorSetting/VCS/Add_In_VCS.htm), то представление
	 может быть добавлено в систему для контроля изменений его версии.
	 Для этого в навигаторе объектов в контекстном меню для представления
	 выполните команду «Добавить в VCS».
	 После отправки объекта на сервер VCS будет возможность отслеживать
	 его изменение. В системе управления версиями сохраняются текст представления
	 и настройки параметров. Более подробно о работе с объектами, добавленными
	 в систему управления версиями, читайте в подразделе «[Разработка
	 приложений в команде](UiDevEnv.chm::/01_Development_Environment/06_Version_Control_System/VSC_Introduction.htm)».
