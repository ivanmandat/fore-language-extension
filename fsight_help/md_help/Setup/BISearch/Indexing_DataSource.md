# Индексация источников


BI-поиск работает только по тем источникам репозитория (стандартным
 кубам и базам данных временных рядов), которые были проиндексированы.
 При индексации в хранилище Apache Solr
 создаются все необходимые файлы для полнотекстного поиска. Для индексации
 источников репозитория предназначена задача обновления поискового индекса
 в [контейнере
 запланированных задач](UiAppSrv.chm::/Getting_Started.htm). Более подробно о подготовке задачи читайте в
 подразделе: «[Особенности
 при обновлении поискового индекса](UiAppSrv.chm::/3_Work_Tasks/Updating_the_search_index_task.htm)».


Если планировщик задач [настроен
 на работу с контейнером](UiAppSrv.chm::/1_Work_AppSrv/UiAppSrv_Work_AppSrvXml.htm), в котором создана задача обновления
 поискового индекса, то индексация будет произведена в соответствии с установленным
 для задачи [расписанием](UiAppSrv.chm::/3_Work_Tasks/UiAppSrv_Work_Tasks_CreateTask_TimeTable.htm).
 При необходимости индексацию можно произвести самостоятельно, выполнив
 задачу принудительно. Для запуска задачи используется команда контекстного
 меню «[Выполнить](UiAppSrv.chm::/3_Work_Tasks/Work_Task.htm#starttask)».
