# Активация продукта на базе RMS


Для активации продукта на базе программного обеспечения RMS:


	ОС
	 Linux ОС Windows

		При использовании [сетевой
		 лицензии](../Admin_Licensing_Variants.htm#network) для работы веб-приложения в ОС Linux:


			- Установите и настройте [сервер
			 лицензий](LicenseServer.htm).


			- Установите [сервис
			 лицензирования](../Service/Licensing_Service.htm) для ускорения работы с сервером лицензий
			 при необходимости.


			- Разместите полученный файл лицензии lservrc без расширения
			 в папку установки сервера лицензий.


			- Создайте системную переменную [LSFORCEHOST/LSHOST](LicenseServer.htm#lshost)
			 с указанием имени сервера лицензий.


		При использовании [временной
		 автономной лицензии](../Admin_Licensing_Variants.htm#temporary) для работы веб-приложения в ОС Linux:


			- Разместите полученный файл лицензии lservrc без расширения
			 в папку установки BI-сервера /opt/foresight/fp10.x-biserver/bin.


			- Создайте системную переменную [LSFORCEHOST/LSHOST](LicenseServer.htm#lshost)
			 со значением «NO-NET».


		При использовании [сетевой
		 лицензии](../Admin_Licensing_Variants.htm#network) для работы веб-приложения и настольного приложения
		 в ОС Windows:


			- Установите и настройте [сервер
			 лицензий](LicenseServer.htm).


			- Установите [сервис
			 лицензирования](../Service/Licensing_Service.htm) для ускорения работы с сервером лицензий
			 при необходимости.


			- Разместите полученный файл лицензии lservrc без расширения
			 в папку установки сервера лицензий.


			- Активируйте лицензию одним из способов:


				- используйте [менеджер
				 лицензий](Intro_LicenseManager.htm) настольного приложения;


				- создайте системную переменную [LSFORCEHOST/LSHOST](LicenseServer.htm#lshost)
				 с указанием имени сервера лицензий;


				- укажите имя сервера лицензий в [реестре](LicenseServer.htm#lshost).


		При использовании [временной
		 автономной лицензии](../Admin_Licensing_Variants.htm#temporary) для работы веб-приложения и [автономной
		 лицензии](../Admin_Licensing_Variants.htm#standalone) для работы настольного приложения в ОС Windows:


			- Разместите полученный файл лицензии lservrc без расширения
			 в любом месте.


			- Активируйте лицензию одним из способов:


				- используйте [менеджер
				 лицензий](Intro_LicenseManager.htm) настольного приложения;


				- создайте системную переменную [LSFORCEHOST/LSHOST](LicenseServer.htm#lshost)
				 со значением «NO-NET».


Совет. Начиная с версии 10.8,
 рекомендуется использовать программное обеспечение Guardant. В следующих
 версиях «Форсайт. Аналитическая платформа»
 поддержка RMS будет прекращена.


Для получения подробной информации о программном обеспечении RMS обратитесь
 к [официальной документации](https://docs.sentinel.thalesgroup.com/softwareandservices/rms/RMSDocumentation/Vendor/Content/DevGuide/Chapter 2_Components/License Manager.htm).


«Форсайт. Аналитическая платформа»
 использует предопределенный набор [единиц
 лицензирования](../AboutFeatures.htm) и позволяет использовать прикладные единицы лицензирования
 для защиты прикладных систем, разработанных на основе продукта. Для получения
 подробной информации о прикладных единицах лицензирования обратитесь к
 разделу «[Защита прикладных
 систем](../ApplicationSystemLicensing.htm)».


Примечание.
 Защита прикладных систем доступна только при активации продукта на базе
 RMS без установки [сервиса лицензирования](../Service/Licensing_Service.htm).
