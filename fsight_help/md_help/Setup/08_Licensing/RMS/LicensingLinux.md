# Работа с лицензией в ОС Linux


В ОС Linux для работы с [лицензией](../Admin_Licensing.htm)
 используются специальные настройки. Актуально при активации продукта на
 базе [RMS](Activation_Product_RMS.htm).


Раздел описывает активацию продукта «Форсайт. Аналитическая платформа»
 на BI-сервере в ОС Linux.


Примечание.
 Продукт «Форсайт. Аналитическая платформа»
 активируется на BI-сервере только с помощью [сетевой](../Admin_Licensing_Variants.htm)
 или [временной автономной лицензии](../Admin_Licensing_Variants.htm).


## Настройки поиска лицензии


Настройки поиска лицензии определяют, каким образом будет искаться лицензия.


Для управления поиском лицензии создайте системную переменную среды
 [LSFORCEHOST](LicenseServer.htm#lshost):


	- для обращения к [серверу лицензий](LicenseServer.htm)
	 в значении переменной укажите имя сервера лицензий;


	- для поиска [автономной
	 лицензии](../Admin_Licensing_Variants.htm#standalone) в значении переменной укажите «NO-NET». Для успешной
	 активации приложения автономной лицензией выполните шаги, описанные
	 ниже в подразделе «Активация автономной
	 лицензией».


Примечание.
 Для указания нескольких серверов лицензий используйте системную переменную
 среды [LSHOST или настройки, заданные
 через реестр](LicenseServer.htm#lshost).


## Активация автономной лицензией


Для поиска автономной лицензии:


	- Создайте системную переменную [LSFORCEHOST](LicenseServer.htm#lshost)
	 со значением «NO-NET».


	- [Получите цифровой отпечаток компьютера](LockingCode.htm)
	 и отправьте запрос на получение лицензии
	 по адресу [support@fsight.ru](mailto:support@fsight.ru).
	 Лицензия будет выслана в ответном письме на электронную почту.


	- Расположите полученный файл лицензии lservrc в папке /opt/foresight/fp10.x-biserver/bin.


Примечание.
 Файл lservrc должен быть без расширения. Если файл имеет расширение «.dat»,
 то уберите его.


	- Пункт актуален, если BI-сервер был установлен из архива. Если
	 использовался установочный пакет, то переходите к следующему шагу.

	Запустите утилиту lsinit, которая
	 входит в комплект поставки BI-сервера продукта «Форсайт. Аналитическая платформа»
	 под ОС Linux. Для запуска утилиты выполните в терминале команды:


cd /opt/foresight/fp10.x-biserver/bin


sudo ./lsinit


В случае успешной инициализации выводится
 сообщение «Successful to initialize the
 system». Если появилось другое сообщение, то проверьте, правильно
 ли выполнены предыдущие шаги, правильно ли вводились указанные команды.
 Повторите шаги корректно. Если повторно выводится сообщение об ошибке,
 то обратитесь в техническую поддержку по адресу [support@fsight.ru](mailto:support@fsight.ru).


	- Раздайте права доступа на файл лицензии для пользователя Apache2:


sudo chown www-data:www-data
 /opt/foresight/fp10.x-biserver/bin/lservrc


В команде указывайте путь до файла лицензии,
 который был получен и размещен на шаге 3.


Указанный пример применим для раздачи прав
 в Debian-подобных дистрибутивах.
 При работе в других дистрибутивах ОС Linux замените «www-data:www-data»
 на «apache:apache» для RedHat-подобных дистрибутивов или на «apache2:apache2» для ALT Linux.


## Просмотр используемой лицензии


Чтобы понять какая лицензия используется, посмотрите значение переменной:
 LSFORCEHOST/LSHOST или посмотрите лог файлы с помощью стандартных текстовых
 редакторов или их аналогов. Например, чтобы посмотреть лог файл Axis на
 ОС Astra Linux, выполните команду:


vi "/opt/foresight/fp10.x-biserver/var/log/axis2.log
