# Настройка доменной/интегрированной доменной аутентификации на веб-сервере
 Java


Данная статья предназначена для настройки доменной/интегрированной доменной
 [аутентификации](Authentication.htm#web) в веб-приложении,
 развёрнутом на базе веб-сервера Java (Apache Tomcat). Перед настройкой
 веб-приложения необходимо [добавить](Admin.chm::/03_Admin/Users/Admin_UserCreate_Domain.htm)
 доменных пользователей в менеджере безопасности настольного приложения
 и раздать им необходимые права доступа.


Для настройки аутентификации в веб-приложении «Форсайт. Аналитическая платформа»
 выполните следующие шаги:


	- [Установите BI-сервер](../Install_Web_on_Windows.htm),
	 [Apache
	 Tomcat и веб-приложение](../03_Setup_Web/Windows/Installing_Web_Application_Back_End_on_Windows_OS.htm).


	- Проверьте, чтобы Apache Tomcat выполнялся как служба. Для этого
	 откройте «Панель управления»,
	 перейдите в раздел «Администрирование
	 > службы» и проверьте наличие запущенной службы «Apache
	 Tomcat 7.0 Tomcat7».


	- Настройте веб-приложение. В зависимости от выбранного типа аутентификации
	 в файле [PP.xml](../03_Setup_Web/PP_config_Java.htm) (файл
	 располагается в папке C:\Program Files\Apache
	 Software Foundation\Tomcat 7.0\webapps\<веб-приложение>\config\)
	 в теге metabase добавьте атрибут
	 authentication со следующими
	 значениями:


		- для доменной аутентификации: authentication=”Domain”;


		- для интегрированной доменной аутентификации:
		 authentication=”IntegratedDomain”.


	- Перезапустите Apache Tomcat.


	- В [настройках
	 подключения к репозиторию](../../06_AK_Client_Config/UiNav_RepoConfig_repo1.htm) задайте выбранный тип аутентификации.


	- [Проверьте](../03_Setup_Web/Opening_Web_Application.htm)
	 работу веб-приложения.
