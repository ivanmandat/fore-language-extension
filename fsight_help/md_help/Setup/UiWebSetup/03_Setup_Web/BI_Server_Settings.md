# Системные настройки продукта «Форсайт. Аналитическая платформа»


Системные настройки продукта «Форсайт. Аналитическая платформа»
 формируются:


	- в файле [settings.xml](BI_Server_Settings_XML.htm);


	- в [системном реестре](BI_Server_Registry_Key.htm).


Наименования и назначения подразделов/параметров в реестре совпадают
 с соответствующими элементами/атрибутами файла settings.xml. Для получения
 подробной информации о настройках продукта обратитесь к разделу «[Настройки
 в системном реестре](BI_Server_Registry_Key.htm)».


При запуске веб- или настольного приложения выполняется поиск файла
 settings.xml. Если файл отсутствует или в нём нет необходимых настроек,
 то они берутся из реестра. Если настройки отсутствуют в реестре, то используются
 их значения по умолчанию.


Примечание.
 Файл settings.xml имеет более высокий приоритет, чем системный реестр.


При установке продукта «Форсайт. Аналитическая платформа»
 файл settings.xml и разделы/параметры в реестре не создаются. При необходимости
 использования каких-либо настроек соответствующие параметры необходимо
 создать вручную.


Все настраиваемые параметры выводятся в отладочной информации. Отследить
 раздел файла settings.xml или ветку реестра, из которых загружаются параметры,
 можно с помощью программ, предназначенных для получения отладочной информации,
 например, DebugView - входит в пакет системных программ [Windows Sysinternals](https://docs.microsoft.com/ru-ru/sysinternals/).


В зависимости от специфики использования веб-приложения и BI-сервера
 может потребоваться [настройка таймаутов](setting_the_timeout.htm)
 различных узлов разрабатываемой системы.


## Дополнительные настройки BI-сервера


Во время разработки проекта на базе BI-сервера продукта «Форсайт. Аналитическая платформа»
 могут использоваться дополнительные возможности, разработанные для расширения
 функциональности будущих систем:


	- Использование [сервера
	 состояний](../01_BIServer/StateServer/State_Server.htm) для создания отказоустойчивых систем;


	- Создание [отказоустойчивого
	 кластера](../01_BIServer/Cluster/Cluster_Of_BI_Server.htm) на базе нескольких BI-серверов;


	- Использование [пула
	 соединений](../01_BIServer/PoolConnections.htm).


Примечание.
 При установке BI-сервера в ОС Windows 7, Windows Server 2008 или Windows
 Server 2008 R2 для корректной работы с библиотеками OpenSSL может потребоваться
 установка обновления [KB2533623](https://support.microsoft.com/ru-ru/help/2533623/microsoft-security-advisory-insecure-library-loading-could-allow-remote-code-execution).
