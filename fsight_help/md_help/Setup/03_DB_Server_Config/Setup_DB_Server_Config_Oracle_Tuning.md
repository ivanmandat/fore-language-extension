# Дополнительные настройки Oracle


## Настройка параметра OS_AUTHENT_PREFIX


Данный параметр отвечает за префикс перед именами пользователей. Если
 данный параметр задан на сервере ORACLE, то необходимо его задать в [параметрах
 репозитория](../06_AK_Client_Config/UiNav_RepoConfig_repo1.htm) в поле «Формат имени
 пользователей ОС». Параметру OS_AUTHENT_PREFIX на сервере задано
 значение по умолчанию.


Примечание.
 При задании параметра необходимо помнить, что регистр имеет значение.


Параметр OS_AUTHENT_PREFIX задается в файле:


S\oracle\admin\orсl\pfile\initorcl.ora,


где:


S - путь до места установки ORACLE;


orcl - экземпляр базы данных (SERVICE_NAME).


В файле необходимо прописать следующую строку:


OS_AUTHENT_PREFIX = "VALUE"


где:


VALUE - значение параметра.


Примечание.
 Если на сервере параметр OS_AUTHENT_PREFIX прописан как пустой, то в настройках
 репозитория его следует оставить незаполненным.


## Настройка при доменной аутентификации


При использовании доменной аутентификации определите формат имени пользователя
 в [настройках
 соединения базы данных](UiNavObj.chm::/database/UiDb_database_master_property.htm). Формат следует определять,
 если он отличен от используемого по умолчанию и/или на сервере ORACLE
 задан префикс перед именами пользователей (параметр OS_AUTHENT_PREFIX).
 По умолчанию при доменной аутентификации «Форсайт. Аналитическая платформа»
 использует следующий формат имен: «ДОМЕН\ИМЯ_ПОЛЬЗОВАТЕЛЯ».


	- Определите на сервере доменную аутентификацию, которая задается
	 в файле:


S\oracle\ora92\network\admin\sqlnet.ora,


где:


S - путь до места установки ORACLE на сервере.


В файле пропишите следующую строку:


SQLNET.AUTHENTICATION_SERVICES
 = (NTS)


После указания строки перезагрузите службы
 на сервере в следующем порядке:


	-


		- TNSListener;


		- OracleService.


После перезагрузки сервера перезагрузите
 указанные выше службы. Иначе могут возникнуть проблемы при подключении
 под пользователем СУБД или под доменным пользователем.


	- Для определения доменной аутентификации на клиентском компьютере
	 пропишите строку, приведенную для сервера, в файле sqlnet.ora:


S\oracle\ora92\network\admin\sqlnet.ora,


где:


S - путь до места установки ORACLE на клиентском
 компьютере.


Перезапустите «Форсайт. Аналитическая платформа».
