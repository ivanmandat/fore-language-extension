# Подготовка серверной части СУБД


Перед созданием репозитория метаданных платформы необходимо произвести
 подготовку серверной части СУБД. До начала работ рекомендуется ознакомиться
 с [требованиями к серверу](Setup_DB_ServerSysReq.htm). Подготовка
 заключается в создании отдельной базы данных и пользователя, который будет
 являться администратором этой базы данных.


Для различных СУБД процедура подготовки
 проводится по-разному:


	- [Oracle](Setup_DB_Server_Config_Oracle.htm);


	- [Microsoft SQL Server](Setup_DB_Server_Config_MSQL.htm);


	- [PostgreSQL](Postgres_server_preparation.htm);


	- [SQLite](Config_SQLite.htm).


После выполнения всех действий по подготовке СУБД рекомендуется проверить
 подключение пользователя к созданной базе данных средствами самой СУБД.
 Если в дальнейшем возникнут проблемы подключения к репозиторию, то также
 может потребоваться проверка прав пользователя и дополнительная настройка
 на уровне СУБД.


## Комментарии


Во время работы «Форсайт. Аналитическая платформа»
 формирует все необходимые запросы для работы с СУБД, инициализирует и
 отслеживает работу транзакций. Однако важным звеном, влияющим на производительность
 всей системы, являются настройки СУБД, которые задаются на сервере СУБД
 и которыми платформа не управляет. В большинстве случаев СУБД устанавливаются
 с конфигурационными файлами, содержащими настройки по умолчанию. Такие
 настройки часто могут иметь минимальные допустимые значения. В зависимости
 от предполагаемой нагрузки может потребоваться изменение объема используемой
 памяти, количества одновременных подключений, количества одновременно
 открытых курсоров и других важных настроек. В высоконагруженных многопользовательских
 системах необходимо учитывать уровень изоляции транзакций. «Форсайт. Аналитическая платформа»
 работает с уровнем read committed, корректная работа с другими уровнями
 не гарантирована.


Более подробную информацию, а также рекомендации по настройке, можно
 найти в специализированной документации на используемую СУБД.
