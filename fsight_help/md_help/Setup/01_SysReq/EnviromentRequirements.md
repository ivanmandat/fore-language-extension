# Требования к настройкам операционной системы


В данном разделе приведены требования к настройкам операционной системы,
 предъявляемые продуктом «Форсайт. Аналитическая платформа».
 Выполнение данных требований обеспечит корректную установку и работу «Форсайт. Аналитическая платформа».


## Требования к настройкам перед установкой «Форсайт. Аналитическая платформа»


Перед установкой «Форсайт. Аналитическая платформа»
 проверьте:


	- корректность системного времени на клиентском компьютере. При
	 необходимости скорректируйте его до установки «Форсайт. Аналитическая платформа»;


Важно.
 Нельзя переводить системное время на клиентском компьютере или сервере,
 если на них хотя бы раз был установлен Prognoz Platform
 или «Форсайт. Аналитическая платформа».
 В случае перевода системного времени не будут работать никакие лицензии.


	- права администратора. Права администратора требуются для установки
	 [настольного приложения
	 «Форсайт. Аналитическая платформа»](../02_AK_Install/Setup_AK_Install.htm),
	 [BI-сервера](../UiWebSetup/UiWebSetup_TitlePage.htm) и
	 [веб-приложения](../UiWebSetup/03_Setup_Web/Web_server_setup.htm),
	 [серверной
	 части СУБД](../03_DB_Server_Config/Setup_DB_Server_Config.htm) и [сервера
	 лицензий](../08_Licensing/Admin_Licensing.htm);


	- в операционной системе Linux:


		- временное файловое хранилище [Tmpfs](https://www.kernel.org/doc/html/latest/filesystems/tmpfs.html).
		 Если в операционной системе используется Tmpfs, то рассмотрите
		 возможность отказа от его использования, чтобы временные файлы
		 располагались на жестком диске. Это связано с тем, что файловая
		 система Tmpfs не поддерживает атрибуты, необходимые для корректной
		 работы «Форсайт. Аналитическая платформа»;


	- в зависимости от типа операционной системы Windows:


		- серверная операционная система Windows: роль «Веб-сервер».
		 При установке BI-сервера и веб-приложения на серверную операционную
		 систему Windows на серверах требуется [добавить
		 роль «Веб-сервер»](../UiWebSetup/03_Setup_Web/Add_WebServer_Role.htm);


		- клиентская операционная система Windows: службы Internet
		 Information Services. При установке BI-сервера и веб-приложения
		 на клиентскую операционную систему Windows требуется [подключить
		 службы Internet Information Services](../UiWebSetup/03_Setup_Web/IIS_turn_on.htm);


	- настройки мест хранение временных файлов. Во время работы BI-сервер
	 может создавать и использовать временные файлы. Временные файлы удаляются
	 после завершения их использования. Для настройки мест хранение временных
	 файлов используются штатные механизмы ОС. Для ОС Linux расположение
	 временных файлов контролируется переменной окружения TMPDIR, для ОС
	 Windows - переменной среды TMP.


Примечание.
 Дополнительные требования, предъявляемые при установке серверной части
 СУБД, следует уточнить в документации по СУБД.


## Требования к настройкам для работы с «Форсайт. Аналитическая платформа»


Для работы с «Форсайт. Аналитическая платформа»
 требуются:


	ОС
	 Linux ОС Windows


			- Открытый доступ к портам, на работу с которыми настроена
			 СУБД.


			- Запуск служб СУБД.


			- Права группы «Пользователи».


			- Открытый доступ к портам, на работу с которыми настроена
			 СУБД.


			- Запуск служб СУБД.


			- Запуск Microsoft .NET Framework 4.5.2 и выше.


			- Для корректной работы клиента Oracle и «Форсайт. Аналитическая платформа»
			 необходимо, чтобы в локальной политике безопасности Windows
			 пользователь (группа) были включены в политику «Создавать
			 глобальные объекты».


Примечание.
 Для уточнения номеров портов и необходимых служб следует обратиться к
 документации по СУБД.


Для работы «Форсайт. Аналитическая платформа»
 как части DHTML-приложения требуются:


	ОС
	 Linux ОС Windows


			- Полные права доступа на папку, в которую установлен
			 «Форсайт. Аналитическая платформа».


			- Открытый доступ к порту, через который осуществляется
			 работа веб-сервера (по умолчанию порт 8110
			 для Apache2).


			- Права группы «Пользователи».


			- Полные права доступа на папку, в которую установлен
			 «Форсайт. Аналитическая платформа».


			- Права на создание виртуальных каталогов.


			- Запуск Microsoft .NET Framework 4.5.2 и выше.


			- Запуск служб Internet Information Services. При установке BI-сервера
			 и веб-приложения инсталлятор создает необходимые пулы приложений
			 и виртуальные каталоги.


			- Открытый доступ к порту, через который осуществляется
			 работа веб-сервера (по умолчанию порт 80 для IIS).


Для взаимодействия «Форсайт. Аналитическая платформа»
 и сервера лицензий в ОС Linux и ОС Windows:


	- права привилегированного пользователя для запуска/остановки
	 и конфигурирования сервера лицензий;


	- открытый доступ к портам:


		- при использовании сервера лицензий на базе [Guardant](../08_Licensing/Guardant/Activation_Product_Guardant.htm)
		 должен быть открыт доступ к порту сервиса лицензирования (по умолчанию
		 8510), с которым взаимодействует «Форсайт. Аналитическая платформа».
		 Сервис лицензирования взаимодействует с сервером лицензий через
		 порт 3189;


		- при использовании сервера лицензий на базе [RMS](../08_Licensing/RMS/Activation_Product_RMS.htm)
		 должен быть открыт доступ к порту сервиса лицензирования (по умолчанию
		 8510), с которым взаимодействует «Форсайт. Аналитическая платформа».
		 Сервис лицензирования взаимодействует с сервером лицензий через
		 порт 5093. Если сервис лицензирования не используется, то достаточно
		 открытого доступа к порту сервера лицензий 5093.


Для корректной печати отчётов, построенных с помощью инструментов «Форсайт. Аналитическая платформа»,
 при использовании ALT Linux 10:


	- измените в файле сервиса Apache2 /etc/systemd/system/httpd2-fp10.x.service
	 значение параметра PrivateTmp, который управляет созданием каталога
	 на каждый процесс apache2-fp10.x (в каталоге tmp):


PrivateTmp=false


	- создайте каталог /tmp/apache2-fp10.x для хранения временных
	 файлов, которые формируются при работе с библиотекой Qt. Расположение
	 временных файлов контролируется переменной окружения TMPDIR.
