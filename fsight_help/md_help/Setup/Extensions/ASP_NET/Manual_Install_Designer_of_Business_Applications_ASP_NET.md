# Ручная установка и настройка IIS для доступа к конструктору бизнес-приложений


Ниже приведён пример ручной установки и настройки IIS для доступа
 к ресурсам конструктора бизнес-приложений в ОС Windows.


Совет. Для установки
 серверной части конструктора бизнес-приложений в ОС Windows рекомендуется
 использовать дистрибутив, входящий в [комплект
 поставки продукта «Форсайт. Аналитическая платформа»](../../02_AK_Install/DistributionKit.htm#linux).


Для установки серверной части конструктора бизнес-приложений вручную:


	- Убедитесь, что установлен [BI-сервер](../../UiWebSetup/Install_Web_on_Linux.htm).


	- Создайте папку, в котором будут располагаться файлы серверной
	 части конструктора бизнес-приложений, например: C:\Program Files (x86)\Foresight\DBA_10.x.


	- Скопируйте архив dba.zip.


Примечание.
 Архив dba.zip не входит в [комплект
 поставки продукта «Форсайт. Аналитическая платформа»](../../02_AK_Install/DistributionKit.htm).
 Для получения архива обратитесь в техническую поддержку по адресу [support@fsight.ru](mailto:support@fsight.ru)
 или через [сервисы
 технической поддержки](https://www.fsight.ru/support/), которые доступны после регистрации на сайте.


	- Вставьте архив dba.zip в папку C:\Program Files (x86)\Foresight\DBA_10.x
	 и распакуйте его.


	- Добавьте новое приложение в диспетчере служб IIS. В окне «Добавление приложения» задайте параметры:


		- Псевдоним. Введите
		 наименование приложения, например: «DBA_10.x»;


		- Физический путь.
		 Укажите путь расположения файлов серверной части конструктора
		 бизнес-приложений, например: C:\Program Files (x86)\Foresight\DBA_10.x.


	- [Настройте
	 конфигурацию конструктора бизнес-приложений](../Configuration_Designer_of_Business_Applications.htm).


	- [Перезапустите
	 серверную часть конструктора бизнес-приложений](../Restart_Backend_Designer_of_Business_Applications.htm).


После выполнения действий будет установлена серверная часть конструктора
 бизнес-приложений.
