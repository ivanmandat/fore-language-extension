# Как выполнить импорт, если возникает ошибка «Вносимое значение слишком
 велико для столбца»?


Если при импорте данных в базу данных временных рядов возникает ошибка
 «Вносимое значение слишком велико для
 столбца», то это значит, что превышена максимальная длина мнемоники.
 По умолчанию мнемоника не может быть длиннее 255 символов и если импортируются
 данные, содержащие мнемонику большей длины, то возникает указанная ошибка.


Выполнить импорт можно следующими способами:


[![](../Opened.gif)![](../Closed.gif)Вручную увеличьте
 длину поля «Мнемоника» в таблице фактов базы данных временных рядов](javascript:TextPopup(this))


		- В дереве папок навигатора объектов выделите требуемую базу
		 данных временных рядов. В списке объектов будут отображены её
		 внутренние объекты.


		- Откройте папку «Данные» (идентификатор FOLDER_DATA). Будет
		 отображён список метасправочников базы данных временных рядов.


		- Дважды щелкните по метасправочнику фактов. Он имеет наименование
		 вида «Факты_N» и идентификатор вида METAFACTS_K где N - это
		 наименование базы данных временных рядов, а K - её ключ.
		 Будет отображена таблица фактов.


		- Откройте мастер редактирования таблицы:


			- выделите таблицу фактов и нажмите клавишу F4;


			- выполните команду «Редактировать»
			 в контекстном меню.


		- Перейдите на страницу «Поля»
		 в мастере редактирования таблицы.


		- Выделите поле «Мнемоника»
		 (идентификатор MNEMO) и дважды по нему щёлкните или нажмите кнопку
		 «Редактировать». Будет
		 открыт диалог «[Свойства поля таблицы](UiNavObj.chm::/table/master/UiDb_relational_table_master_field.htm#add)».


		- В поле «Общая длина»
		 введите требуемую длину мнемоники.


		- Нажмите кнопку «ОК»
		 в диалоге «[Свойства поля таблицы](UiNavObj.chm::/table/master/UiDb_relational_table_master_field.htm#add)»
		 и затем кнопку «Готово»
		 в мастере редактирования таблицы.


[![](../Opened.gif)![](../Closed.gif)Увеличьте длину
 поля для атрибута «MNEMO» с помощью языка Fore](javascript:TextPopup(this))


	Приведён пример кода для увеличения длины поля для атрибута «MNEMO»
	 в базе данных временных рядов с идентификатором TSDB_MNEMO.


	Добавьте ссылки на системные сборки: Cubes, Db, Metabase, Rds.


	Пример:


		Sub UserProc;

		Var

		    mb: IMetabase;

		    TSDB: IRubricator;

		    FactDic: IMetaDictionary;

		    MnemoAttr: IMetaAttribute;

		    MnemoTable: ITable;

		    MnemoField: ITableField;

		Begin

		    mb := MetabaseClass.Active;

		    // Получаем базу данных временных рядов

		    TSDB := mb.ItemById("TSDB_MNEMO").Edit As IRubricator;

		    // Получаем метасправочник фактов для редактирования

		    FactDic := TSDB.EditFacts;

		    MnemoTable := FactDic.Attributes.Table;

		    // Получаем поле MNEMO

		    MnemoField := MnemoTable.Fields.FindById("MNEMO");

		    // Увеличиваем длину поля

		    MnemoField.Size := 300;

		    // Сохраняем изменения

		    (TSDB As IMetabaseObject).Save;

		End Sub UserProc;


[![](../Opened.gif)![](../Closed.gif)Удалите мнемоники
 из базы данных временных рядов](javascript:TextPopup(this))


		- В навигаторе объектов выберите требуемую базу данных временных
		 рядов.


		- Нажмите клавишу F4 или выполните команду «Редактировать»
		 в контекстном меню. Будет открыт мастер редактирования базы данных
		 временных рядов.


		- Перейдите на страницу «[Атрибуты](UiNavObj.chm::/TimeSeriesDatabase/TS_Attributes.htm)».


		- Снимите отметку с флажка «Ряды
		 имеют идентификатор».


		- Нажмите кнопку «Готово».


[![](../Opened.gif)![](../Closed.gif)Измените правила
 формирования мнемоник таким образом, чтобы сгенерированная мнемоника всегда
 была меньше 255 символов.](javascript:TextPopup(this))


	Правила генерации мнемоник задаются с помощью языка Fore. Для этого
	 используйте свойство [IMetaDictionary.AttributeMnemoNamings](KeRds.chm::/Interface/IMetaDictionary/IMetaDictionary.AttributeMnemoNamings.htm).
	 Пример задания правил приведён в описании свойства.
