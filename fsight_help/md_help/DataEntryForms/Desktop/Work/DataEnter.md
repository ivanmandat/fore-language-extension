# Ввод и сохранение данных


Для ввода данных или редактирования значений ячеек:


	- выделите ячейку и введите необходимые данные;


	- дважды щёлкните по области выбранной ячейки;


	- выделите ячейку и нажмите клавишу F2, после чего введите данные.


Для измененных значений применяется жирное начертание шрифта. Оформление
 может быть перекрыто другим оформлением.


При просмотре [готовой формы ввода](FinishForm.htm) доступно
 сохранение измененных значений параметров в табличной области с помощью
 [пользовательских настроек](Custom_settings.htm).


Для завершения ввода данных:


	- нажмите клавишу ENTER;


	- нажмите клавишу TAB;


	- нажмите клавишу перемещения курсора, при этом осуществится переход
	 на соседнюю ячейку в заданном направлении. Данный метод не работает,
	 если ячейка редактируется с помощью клавиши F2 или из строки формул.


Если был настроен [контроль](../Table/Calculation_Practices.htm#modelproperties)
 данных и выбран вариант «При изменении
 данных», то при вводе некорректных значений, они будут выделены.


При открытии формы ввода на просмотр или редактирование значения
 в ячейках, рассчитанные при помощи [подключенного
 алгоритма расчета](../Table/Calculation_Practices.htm#addalgorithm), отображаются как изменённые данные, если не были
 сохранены ранее. При изменении значений в ячейках, использующихся в формулах,
 происходит расчёт новых значений, если алгоритм подключен на изменение,
 и рассчитанные значения снова отображаются как изменённые данные.


Примечание.
 Для запуска перерасчёта данных установите флажок «[Объединять запросы при
 вычислении](../Starting/Starting.htm#multiexecutor)» на вкладке «Общие»
 диалога «Параметры». Перерасчёт
 происходит, если изменяемые данные являются входными данными для другой
 табличной области, значения которой рассчитываются с помощью алгоритма,
 выполняемого при изменении. Если флажок снят, то перерасчёт данных выполняется
 только в одной табличной области на текущем листе формы ввода. В других
 табличных областях перерасчёт не выполняется.


Для внесения изменений данных в форму сохраните
 изменения, нажав кнопку ![](../Starting/saveData.png) «Сохранить
 данные».


Если была настроена [блокировка
 сохранения измененных данных](../Table/Calculation_Practices.htm#block) табличной области при сработавшем контроле,
 то при вводе некорректных значений будет выведено информационное сообщение,
 что требуется корректировка введенных данных.


Если данные были изменены, но не были сохранены, то при [сохранении](../Starting/ConstructForm.htm#save),
 [создании копии](../Common/Save.htm), [обновлении](../Common/Refresh.htm),
 [смене значений параметров](UseControls.htm), закрытии, [вводе данных при согласовании](Agreement.htm#enter_data)
 формы ввода; [транспонировании](Transpose.htm#transpose) и
 [редактировании](../Table/AreaTable.htm) табличной области
 будет предложено сохранить их. Если была настроена [блокировка
 сохранения измененных данных](../Table/Calculation_Practices.htm#block) табличной области при сработавшем контроле,
 то будет выведено предупреждение, что измененные данные не будут сохранены,
 так как не удовлетворяют настройкам контролей. При согласии продолжить
 действие измененные данные будут сброшены и действие выполнено. При отказе
 продолжить действие измененные данные останутся в табличной области.


Для отмены изменений нажмите кнопку ![](../Starting/undo.png) «Отменить изменения» на вкладке «Главная» ленты инструментов.


Для наполнения табличной области данными можно использовать [буфер
 обмена](CopyPaste.htm).


Если [ввод или редактирование](Agreement.htm) данных в форме
 производится при выполнении шагов процесса:


	- Сохраните данные.


	- Завершите шаг, для этого нажмите кнопку ![](statusOK.png) «Завершить».


Закончив ввод или редактирование данных, перейдите к шагу, если требуется:

	 для управления отметкой](UseControls.htm);

	- [Сортировка и фильтрация](FilterSearch.htm);


	- [Поиск и замена](Find_and_Replace.htm);
