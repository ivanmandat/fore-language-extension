# Запуск и отладка


	Запуск - это основная операция,
	 используемая для проверки прикладного кода и функциональности работы
	 форм/веб-форм.


	В среде разработки доступны два режима запуска объектов:


		- запуск без отладки;


		- запуск с отладкой.


## Компиляция


	Перед запуском осуществляется синтаксическая проверка кода и компиляция
	 текущего объекта среды разработки, а также при необходимости всех
	 связанных объектов. Перекомпиляция объектов осуществляется, если изменяется
	 время последней компиляции и сохранения объекта. Скомпилированные
	 объекты сохраняются в кеш сборок и в дальнейшем запускаются оттуда.
	 В настольном приложении учитывается выбранный [способ
	 компиляции](Way_to_compiling.htm): стандартное приложение или веб-приложение. При наличии
	 каких-либо ошибок или несоответствий генерируются соответствующие
	 сообщение компилятора.


	При разработке может возникнуть ситуация, когда под одним пользователем
	 в разных сессиях осуществляется одновременная компиляция и запуск
	 какого-либо главного модуля, имеющего связь с другими модулями. Если
	 в связанных модулях изменялся код, то при компиляции в кеш сборок
	 попадет версия из той сессии, которая скомпилируется быстрее других
	 версий модуля. При этом в другой сессии будут обнаружены и использованы
	 обновлённые файлы из кеша сборок. Их  перекомпиляция не будет
	 осуществлена. При несовместимости кода модулей в разных сессиях возникнет
	 ошибка компиляции.


## Запуск без отладки


	Запуск без отладки используется
	 для быстрой проверки функциональности приложения без возможности отладки
	 отдельных блоков кода. Запуск без отладки приводит к запуску объекта
	 в рамках текущего приложения и по скорости соответствует работе в
	 промышленных условиях.


	Для запуска без отладки открытого в среде разработки объекта необходимо
	 выполнить одно из следующих действий:


		- выполнить команду главного меню «Отладка
		 > Выполнить без отладки»;


		- нажать сочетание клавиш CTRL+SHIFT+F9;


		- в настольном приложении нажать кнопку ![](../03_Windows_of_Development_Environment/Button/Run_nodebug.png)
		 «Выполнить без отладки»
		 на панели инструментов «Отладка»;


		- в веб-приложении нажать кнопку ![](Run_web.png) «Выполнить без отладки» на панели
		 инструментов.


	При проверке работы объекта без отладки доступен вывод отладочной
	 информации в консоль среды разработки. Для вывода информации используются
	 статические свойства и методы класса [Debug](ForeSys.chm::/Class/Debug/Debug.htm).


	Если во время выполнения возникает исключительная ситуация, то на
	 экран будет выведено информационное сообщение. После закрытия окна
	 с информацией об ошибке возможна дальнейшая работа с запущенным объектом.


## Запуск с отладкой


	Запуск с отладкой предназначен
	 для проверки работы отдельных блоков кода. Запуск объектов на отладку
	 производится в отдельном процессе, поэтому занимает больше времени
	 чем запуск без отладки. Также доступна [отладка
	 объекта среды разработки](../03_Windows_of_Development_Environment/debug_object.htm), который может использоваться каким-либо
	 объектом[
	 репозитория](../03_Windows_of_Development_Environment/debug_object.htm).


	Для отладки кода необходимо установить [точки
	 останова](../03_Windows_of_Development_Environment/Parameters_point_stop.htm). Точка останова указывает отладчику временно приостановить
	 выполнение кода в определенном месте.


	Для установки точки останова выполните одно из следующих действий:


		- установите курсор в соответствующую строку и нажмите клавишу
		 F5;


		- установите курсор в соответствующую строку и вызовите контекстное
		 меню, в котором выберите команду «Установить/Снять
		 точку останова»;


		- установите курсор в соответствующую строку и выполните команду
		 главного меню «Отладка > Установить/Снять
		 точку останова»;


		- щёлкните по круглому красному маркеру точки останова напротив
		 необходимой строки в области закладок и точек останова (область
		 слева от рабочей области). При щелчке дополнительной кнопкой мыши
		 по маркеру будет открыто окно [параметров
		 точки останова](../03_Windows_of_Development_Environment/Parameters_point_stop.htm).


	Для запуска с отладкой открытого в среде разработки объекта, необходимо
	 выполнить одно из следующих действий:


		- выполнить команду главного меню «Отладка
		 > Выполнить»;


		- нажать клавишу F9;


		- в настольном приложении нажать кнопку ![](../03_Windows_of_Development_Environment/Button/Run.gif)
		 «Начать выполнение кода»
		 на панели инструментов «Отладка»;


		- в веб-приложении нажать кнопку ![](Debug_web.png) «Начать выполнение кода» на панели
		 инструментов;


		- в навигаторе проекта для соответствующего объекта выполнить
		 команду контекстного меню «Выполнить».


	При достижении точки останова выполнение кода будет приостановлено
	 и выполняемая строка будет подсвечена зеленым цветом. В этот момент
	 выполнение объекта переходит в режим отладки.


	Поведение веб-форм зависит от значения свойства modal,
	 которое устанавливается в режиме дизайнера формы. При значении True форма запускается модально
	 относительно текущей вкладки браузера. Индикация режима отладки осуществляется
	 в заголовке формы в виде пиктограммы зеленого цвета, а также в заголовке
	 вкладки браузера.


	Заголовок веб-формы:


	![](debug_webform.png)


	Заголовок вкладки браузера:


	![](devenv_browser_header.png)


	Кнопка ![](b_rullup.png)
	 позволяет свернуть содержимое веб-формы для просмотра кода. Кнопка
	 доступна как при выполнении веб-формы, так и после перехода в режим
	 отладки.


	Если свойству modal установлено
	 значение False, то веб-форма
	 открывается в отдельной вкладке, а индикация режима отладки осуществляется
	 только в заголовке вкладки со средой разработки.


	При достижении точки останова и прерывания выполнения кода пиктограмма
	 в заголовке формы и браузера изменит цвет на красный. Если веб-форма
	 открыта в отдельной вкладке, то для отладки перейдите на вкладку со
	 средой разработки.

		- [использование
		 инспектора значений](../03_Windows_of_Development_Environment/Inspector_Value.htm);


		- [использование
		 стека вызовов](../03_Windows_of_Development_Environment/Stack_of_the_call.htm);


		- [отслеживание

		- [параметры

	Для управления ходом выполнения кода в режиме отладки используются
	 следующие команды:


		- ![](../03_Windows_of_Development_Environment/button/overstep.gif) / ![](overstep_web.png)
		 - Перешагнуть. Выполнение
		 текущей строки без входа во вложенные пользовательские методы.
		 Доступно также по нажатию клавиши F8;


		- ![](../03_Windows_of_Development_Environment/button/Run_Step.gif) / ![](runstep_web.png)
		 - Выполнить. Выполнение
		 текущей строки с входом и пошаговым выполнением вложенных пользовательских
		 методов. Доступно также по нажатию клавиши F7;


		- ![](../03_Windows_of_Development_Environment/button/execut_bef_return.gif) / ![](executebeforereturn_web.png) - Выполнять до
		 возврата. Выполнение дальнейшего кода до возврата к точке
		 останова. Доступно также по нажатию сочетания клавиш SHIFT+F11.


	Данные команды доступны в главном меню «Отладка»,
	 а также на панели инструментов. В главном меню в настольном приложении
	 также доступна команда «Отладка >
	 Выполнить до позиции курсора», позволяющая продолжить выполнение
	 кода до той строки, в которой установлен курсор.


	Примечание.
	 Если в настройках среды разработки выставлен флаг «[Приостановить выполнение при исключительной
	 ситуации](Developer.chm::/About_Developing/Development_Environment_Options.htm)», то при возникновении ошибок работа
	 объекта будет переведена в режим отладки с фокусировкой на строке,
	 в которой возникла ошибка.

## Диагностика работы кода


	При разработке прикладного приложения на Fore разработчик может
	 выводить отладочную информацию в [консоль
	 среды разработки](../03_Windows_of_Development_Environment/Console_window.htm), журнал событий операционной системы, файлы или
	 куда-либо ещё в соответствии с собственными задачами. В определённых
	 случаях выводимой информации может нехватить для выяснения причин
	 неправильного поведения кода или возникновения каких-либо ошибок.
	 В этом случае могут потребоваться дополнительные исследования происходящих
	 процессов. Ядром платформы также формируется отладочная информация,
	 но данная информация не будет доступна без использования дополнительных
	 программных средств или изменения определённых настроек, которые также
	 могут повлиять работу и производительность системы.
