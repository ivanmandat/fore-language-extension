# Создание формы и размещение компонентов


Форма является основным инструментом для создания визуального интерфейса
 приложения. Форма является контейнером для любых компонентов, доступных
 на [панели
 компонентов](../../03_Windows_of_Development_Environment/Panels_component.htm). Каждой форме также соответствует модуль, содержащий описание
 класса формы. В рамках данного класса в дальнейшем будет осуществляться
 написание кода обработки событий компонентов, а также событий самой формы.


Примечание.
 Работа с формами доступна только в настольном приложении.


[![](../../../Opened.gif)![](../../../Closed.gif)Создание новой
 формы](javascript:TextPopup(this))


	Для создания новой формы:


		- в [навигаторе
		 объектов](GetStarted.chm::/Interface/Interface_Navigator.htm):


			- нажмите кнопку «Новый
			 объект > Среда
			 разработки > Форма» в группе «Создать»
			 на вкладке «Главная»
			 ленты инструментов;


			- выполните команду «Создать
			 > Среда разработки > Форма» в контекстном меню
			 навигатора объектов;


		- в [среде
		 разработки](Developer.chm::/About_Developing/Start_the_Development_Environment.htm):


			- выполните команду «Файл
			 > Создать > Форма» в главном меню;


			- выполните команду «Создать
			 > Форма» в контекстном меню навигатора проекта;


			- выполните команду «Создать
			 > Форма» в контекстном меню сборки в навигаторе
			 проекта.


[![](../../../Opened.gif)![](../../../Closed.gif)Редактирование
 готовой формы](javascript:TextPopup(this))


	Для открытия существующей формы на редактирование:


		- в [навигаторе
		 объектов](GetStarted.chm::/Interface/Interface_Navigator.htm):


			- нажмите кнопку ![](b_edit.png)
			 «Редактировать» в
			 группе «Открыть» на
			 вкладке «Главная»
			 ленты инструментов;


			- выполните команду «Редактировать»
			 в контекстном меню формы;


			- нажмите клавишу F4;


		- в [среде
		 разработки](Developer.chm::/About_Developing/Start_the_Development_Environment.htm):


			- выполните команду «Редактировать»
			 в контекстном меню формы в навигаторе проекта;


			- выполните команду «Файл
			 > Открыть» в главном меню.


	При обращении к форме осуществляется проверка прав доступа к ней.
	 Если и пользователя нет прав на редактирование, но есть права на просмотр,
	 то команда «Редактировать»
	 редактировать в навигаторе объектов будет недоступна. По команде «Редактировать» в навигаторе проекта
	 среды разработки форма будет открываться только в режиме просмотра.


	Для открытия готовой формы на просмотр в [навигаторе
	 объектов](GetStarted.chm::/Interface/Interface_Navigator.htm) выделите форму и выполните одно из действий:


		- нажмите кнопку ![](b_open.png) «Открыть»
		 в группе «Открыть» на
		 вкладке «Главная» ленты
		 инструментов;


		- выполните команду «Открыть»
		 в контекстном меню формы;


		- дважды щёлкните по форме.


	После выполнения одного из действий будет открыта форма.


Более подробно о свойствах формы описано в разделе «[Форма](../../../02_Components_constructor_forms/Form.htm)»
 компонентов дизайнера форм. Для перехода между формой и окном макроса
 используйте команду главного меню «Показать
 форму/Скрыть форму» или горячую клавишу F11.


![](form.gif)


Для размещения компонента на форме щёлкните его на панели компонентов,
 а затем щёлкните в том месте формы, куда его необходимо добавить. Если
 при вставке компонента на форму удерживать кнопку мыши, то можно сразу
 изменить размеры компонента. Более точно задать место расположение компонента
 можно с клавиатуры с помощью сочетаний клавиш CTRL+UP/DOWN/LEFT/RIGHT,
 SHIFT+UP/DOWN/LEFT/RIGHT или установить необходимые значения свойств Left,
 Top, Height, Width в [инспекторе
 объектов](../../03_Windows_of_Development_Environment/Object_Inspector.htm). В инспекторе объектов задаются основные свойства компонентов
 и обработчики для событий. События наступают при выполнении определённых
 действий пользователем, также могут быть [сгенерированы](event_generation.htm)
 из прикладного кода. Для компонентов, которые могут принимать фокус, можно
 настроить [порядок
 обхода](../../03_Windows_of_Development_Environment/Order_avoid_component.htm).


Если какие-либо компоненты должны присутствовать на форме только в определённый
 момент, то можно их скрывать или отображать, меняя значение свойства Visible.
 Также можно использовать [динамическое](Dynamic_component_creation.htm)
 создание и уничтожение компонентов из прикладного кода во время работы
 формы. Динамическое создание позволяет экономить ресурсы операционной
 системы и ускоряет работу самой формы.
