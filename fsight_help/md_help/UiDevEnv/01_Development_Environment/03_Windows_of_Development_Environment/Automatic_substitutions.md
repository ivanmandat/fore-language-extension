# Автоматические подстановки


При разработке модулей/форм пользователю довольно часто приходится набирать
 однотипные блоки программы (процедуры, функции, циклы и т.п.) в теле макроса,
 состоящие в основном из ключевых слов языка и различных наименований.
 Для уменьшения объема кода, набираемого вручную, в языке Fore предусмотрена
 автоматическая подстановка таких структур. Ниже представлено описание
 действий, в результате которых происходят автоматические подстановки.


[![](../../Opened.gif)![](../../Closed.gif)Пространства
 имён](javascript:TextPopup(this))


	При описании пространства имен в строке "Namespace <Наименование>"
	 после нажатия клавиш ПРОБЕЛ или ENTER структура принимает вид:


	Namespace <Наименование>


	End Namespace <Наименование>;


[![](../../Opened.gif)![](../../Closed.gif)Классы](javascript:TextPopup(this))


	При описании классов в строке "Class <Наименование>"
	 после нажатия клавиш ПРОБЕЛ структура принимает вид:


	Class <Наименование>


	End Class <Наименование>;


	Либо после написания знака ":" структура принимает вид:


	Class <Наименование>:


	End Class <Наименование>;

[![](../../Opened.gif)![](../../Closed.gif)XML-комментарий](javascript:TextPopup(this))


	При написании трёх прямых слешей (///) в код будет добавлен XML-комментарий,
	 имеющий следующий вид:


	/// <summary>

	///

	/// </summary>


	По умолчанию формируется многострочный комментарий, но при необходимости
	 его можно записать в одну строку:


	/// <summary></summary>


	Если комментарий формируется для блока программы, имеющего параметры,
	 то для каждого параметра будет добавлена строка следующего вида:


	/// <param name="<наименование
	 параметра>"></param>


	Если комментарий добавляется для функции, которая возвращает какое-либо
	 значение, то будет добавлена строка:


	/// <returns></returns>


	Внутри комментария поддерживается автоматическая подстановка для
	 формирования правильного синтаксиса тегов. Например, если ввести открывающий
	 тег <b>, то автоматически будет добавлен закрывающий тег </b>.
	 Имя закрывающего тега будет совпадать с тем именем, которое было указано
	 в открывающем теге.


	XML-комментарии, формируемые для какого-либо блока программы, в
	 дальнейшем можно получить в прикладном коде, используя свойство XMLDocumentation
	 интерфейсов [IForeClass](KeFore.chm::/Interface/IForeClass/IForeClass.htm),
	 [IForeProperty](KeFore.chm::/Interface/IForeProperty/IForeProperty.htm),
	 [IForeSub](KeFore.chm::/Interface/IForeSub/IForeSub.htm).


	/// <summary>Комментарий для процедуры <b>Test</b></summary>

	Sub Test;

	Begin


	End Sub Test;


	/// <summary>

	/// Проверка работы операции Mod, которая возвращает остаток от деления двух чисел

	/// </summary>

	/// <param name="a">Делимое</param>

	/// <param name="b">Делитель</param>

	/// <returns>Остаток от деления</returns>

	Function TestMod(a: Double; b: Double): Double;

	Begin

	    Return a Mod b;

	End Function TestMod;


Также в языке Fore организована автозамена всех ключевых слов языка:
 ключевые слова пишутся прописными буквами с заглавной первой буквой и
 по умолчанию подсвечиваются синим цветом.
