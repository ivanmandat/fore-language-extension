## С чего начать


		 Сборка ABAC
		 предназначена для работы с [атрибутным
		 методом](Admin.chm::/04_SecurityPolicy/Admin_ABAC.htm) разграничения доступа на языке Fore.


		Изучив данный раздел, Вы сможете:


			- добавлять и редактировать свойства атрибутов субъектов
			 и объектов;


			- формировать структуру атрибутного доступа;


			- проверять права доступа субъекта к объекту.


		Более подробное описание возможностей блока приведено в разделе
		 «[Введение в сборку ABAC](KeABAC_Introduction.htm)».


		Описание ресурсов сборки представлено в следующих подразделах:


			- [интерфейсы
			 сборки ABAC](../Interface/KeABAC_Interface.htm);


			- [перечисления сборки
			 ABAC](../Enums/KeABAC_Enums.htm);


			- [классы сборки
			 ABAC](../Class/KeABAC_Classes.htm).


			- [примеры](../Examples/KeABAC_Example.htm).


		 Перед изучением сборки ABAC рекомендуется ознакомиться
		 с разделами: «[Среда разработки](UiDevEnv.chm::/01_Development_Environment/01_Purpose_of_the_constructor/Purpose_of_the_constructor.htm)»,
		 «[Руководство
		 по языку Fore](Fore.chm::/Fore_Title.htm)».


		Для работы с атрибутным методом разграничения доступа ознакомьтесь
		 с разделами: «[Администрирование и контроль доступа](Admin.chm::/Admin_Title.htm)», «[Выбор
		 методов разграничения доступа и их настройка](Admin.chm::/04_SecurityPolicy/Admin_SecPolicy.htm)»,
		 «[Настройка
		 атрибутного метода](Admin.chm::/04_SecurityPolicy/Admin_ABAC.htm)», «[Добавление
		 правил и политик проверки доступа](Admin.chm::/04_SecurityPolicy/Admin_AttributeAccess.htm)».


		Настройки атрибутного доступа, которые задаются через интерфейс
		 инструмента «[Менеджер
		 безопасности](Admin.chm::/Admin_Title.htm)», можно задать, используя
		 сборку ABAC. Также можно
		 создать отдельную структуру атрибутного доступа для определения
		 прав доступа субъекта к объекту, которая не будет отображаться
		 в интерфейсе инструмента.
