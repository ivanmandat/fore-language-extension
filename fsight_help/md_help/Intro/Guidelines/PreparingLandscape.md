# Подготовка ландшафта


Для подготовки ландшафта выполните шаги:


	- Определение количества контуров.


	- Выбор конфигурации системы.


	- Проверка системных требований.


	- Установка продукта.


	- Лицензирование.


После выполнения шагов по подготовке ландшафта перейдите к [разработке
 системы](SystemDevelopment.htm).


## Определение количества контуров


Систему рекомендуется разрабатывать как последовательность контуров,
 которые всё более полно удовлетворяют потребности заказчика. Виды:


	- Демонстрационный контур.
	 Создается на этапе сбора информации и показывает, можно ли решить
	 задачу заказчика. Далее не используется;


	- Исследовательский контур.
	 Создается на этапе планирования для проверки технологий и функциональности
	 будущей системы и уточнения требований. Далее не используется;


	- Контур разработки (DEV).
	 Создается одновременно с контуром тестирования для разработки функциональности
	 системы;


	- Контур тестирования (TEST).
	 Создается одновременно с контуром разработки для тестирования разрабатываемой
	 функциональности системы. Разработанная функциональность переносится
	 в контур тестирования из контура разработки;


	- Контур промышленной эксплуатации
	 (PROD). В контуре используются реальные данные заказчика. Разработанная
	 функциональность переносится на контур промышленной эксплуатации только
	 после успешного прохождения тестирования в контуре тестирования.


Количество контуров определяется спецификой системы. Обязательно должно
 быть выделено минимум три контура: разработки, тестирования и промышленной
 эксплуатации.


Для каждого контура подготовьте серверы и ПО.


## Выбор конфигурации системы


Продукты компании «Форсайт» могут работать в различных конфигурациях,
 используя отдельные серверы для пользовательского интерфейса и функций
 бизнес-логики, администрирования и информационной безопасности.


Одной из особенностей работы с продуктами является использование BI-сервера.
 BI-сервер включает в себя ядро бизнес-логики, которое взаимодействует
 с базой данных через драйвер СУБД, и ядро безопасности.


Для получения подробной информации о типах и особенностях конфигураций
 обратитесь к статье «[Конфигурации
 систем и принцип их работы](Setup.chm::/GeneralInfo.htm)».
