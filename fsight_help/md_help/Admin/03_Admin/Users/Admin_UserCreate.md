# Создание и редактирование учетной
 записи пользователя


Для создания и редактирования учетной записи пользователя используйте
 раздел «[Пользователи](Admin_Users.htm)».


[![](../../Opened.gif)![](../../Closed.gif)Создание
 учетной записи пользователя](javascript:TextPopup(this))


	Для создания учетной записи пользователя:


		- в веб-приложении:


			- нажмите кнопку ![](../../01_RunSecManager/create.png)
			 «Пользователь» на
			 [панели
			 инструментов](../../01_RunSecManager/Admin_Organizational_Starting.htm);


			- выполните команду «Создать
			 пользователя» в раскрывающемся меню кнопки ![](../../01_RunSecManager/create.png) «Пользователь»
			 на [панели
			 инструментов](../../01_RunSecManager/Admin_Organizational_Starting.htm);


		- в настольном приложении:


			- выполните команду контекстного меню «Создать
			 пользователя»;


			- выполните команду [главного
			 меню](../../01_RunSecManager/Admin_Organizational_Starting.htm) «Пользователь >
			 Создать пользователя».


	После выполнения одного из действий будет открыта боковая панель
	 «Свойства» в веб-приложении
	 или окно «Свойства пользователя»
	 в настольном приложении. Задайте [свойства](Admin_UserCreate.htm#properties)
	 создаваемого пользователя.


	При создании пользователя могут быть запрошены [учётные
	 данные пользователя](UiNavObj.chm::/database/UiDb_database_authorization.htm), обладающего правами администрирования
	 пользователей на сервере СУБД, а также учётные данные, необходимые
	 для раздачи прав на объекты репозитория. Если создание пользователя
	 выполняется в репозитории на базе СУБД PostgreSQL, то администратор
	 должен обладать привилегией CREATEROLE на уровне СУБД.


	Примечание.
	 Создание пользователей недоступно в репозиториях на базе СУБД SQLite.
	 В таких репозиториях вся работа осуществляется под одним фиксированным
	 пользователем ADMIN.

	При разделении ролей между администратором информационной безопасности
	 и прикладным администратором создание новых пользователей доступно
	 только прикладному администратору. В диалоге «Свойства
	 пользователя» будет отображена только вкладка «Общие
	 свойства», все остальные вкладки доступны только администратору
	 информационной безопасности.


[![](../../Opened.gif)![](../../Closed.gif)Редактирование
 учетной записи пользователя](javascript:TextPopup(this))


	Для редактирования учетной записи пользователя:


		- в веб-приложении выберите пользователя;


		- в настольном приложении:


			- выполните команду контекстного меню «Свойства»
			 для выбранного пользователя;


			- выполните команду [главного
			 меню](../../01_RunSecManager/Admin_Organizational_Starting.htm) «Пользователь >
			 Свойства» для выбранного пользователя;


			- дважды щелкните по выбранному пользователю.


	После выполнения одного из действий будет открыта боковая панель
	 «Свойства» в веб-приложении
	 или окно «Свойства пользователя»
	 в настольном приложении. Измените [свойства](Admin_UserCreate.htm#properties)
	 пользователя.


	Примечание.
	 При [разделении
	 ролей](../../04_SecurityPolicy/Editor_of_Politicy/Security_EditorPoliticy_Adm.htm) между администратором информационной безопасности и прикладным
	 администратором настройки [членства
	 в группах](Admin_UserProp_Groups.htm), времени доступа и [мандатного
	 доступа](Admin_UserProp_MC.htm) доступны только администратору информационной безопасности,
	 а прикладному администратору доступны только [общие
	 свойства](Admin_UserProp_Common.htm).


Для изменения свойств при [создании](Admin_UserCreate.htm#create)
 и [редактировании](Admin_UserCreate.htm#edit) учетной
 записи пользователя используйте вкладку «Общие
 свойства» на боковой панели «Свойства»
 в веб-приложении и в окне «Свойства пользователя»
 в настольном приложении.

## Особенности при разделении ролей между администраторами


При [разделении
 ролей](../../04_SecurityPolicy/Editor_of_Politicy/Security_EditorPoliticy_Adm.htm) между администратором информационной безопасности (АИБ) и прикладным
 администратором вкладки «Членство в группах»,
 «Мандатный доступ», «Уровни безопасности», «Управление
 доступом» и «Доступные принтеры»
 будут доступны только АИБ.


Если АИБ не имеет прав на обновление пользователей, то после изменения
 членства пользователя в группе или изменения состава обладателей привилегий
 будет выдано сообщение о том, что изменения вступят в силу после обновления
 пользователя. Для обновления пользователей обратитесь к прикладному администратору.


Примечание.
 Определение обладателей [привилегии](../../04_SecurityPolicy/Admin_Priv.htm)
 «Применение прав пользователей на уровне
 СУБД» задается при разделении ролей.


Если АИБ имеет право на обновление прав пользователей, то будет запущена
 процедура [обновления прав пользователей](Admin_User_Update.htm).
