# Удаление пользователя


Для удаления выбранного пользователя в разделе «[Пользователи](Admin_Users.htm)»:


	- в веб-приложении нажмите кнопку ![](../../01_RunSecManager/delete.png) «Удалить» на
	 [панели
	 инструментов](../../01_RunSecManager/Admin_Organizational_Starting.htm);


	- в настольном приложении:


		- выполните команду «Удалить»
		 в контекстном меню;


		- выполните команду «Пользователь
		 > Удалить» в [главном
		 меню](../../01_RunSecManager/Admin_Organizational_Starting.htm).


При выполнении одного из действий появится диалог подтверждения выполняемых
 действий. При выборе положительного ответа пользователь будет удален.


Важно. Удаление
 пользователя на сервере базы данных выполняется вручную.


При удалении могут быть запрошены [учетные
 данные пользователя](UiNavObj.chm::/database/UiDb_database_authorization.htm), обладающего правами администрирования
 пользователей на сервере СУБД.


Примечание.
 При [разделении
 ролей](../../04_SecurityPolicy/Editor_of_Politicy/Security_EditorPoliticy_Adm.htm) между администратором информационной безопасности и прикладным
 администратором удалять пользователей может только прикладной администратор.


Если при [разделении
 ролей администраторов](../../04_SecurityPolicy/Editor_of_Politicy/Security_EditorPoliticy_Adm.htm) установлен [запрет
 на удаление пользователей](../../04_SecurityPolicy/Editor_of_Politicy/Security_EditorPoliticy_General.htm), обладающих какими-либо правами доступа
 на объекты репозитория, то при попытке удаления такого пользователя прикладному
 администратору (или пользователю с [привилегией](../../04_SecurityPolicy/Admin_Priv.htm)
 «Создание, удаление пользователей»)
 будет выдано сообщение о том, что у пользователя есть права доступа. Для
 удаления такого пользователя потребуется снятие всех имеющихся прав доступа.
