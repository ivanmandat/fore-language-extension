 доступа](../04_SecurityPolicy/Admin_SecPolicy.htm#access_control) используются дополнительные настройки.

	 приложение

[![](../Opened.gif)![](../Closed.gif)Использовать
 механизм личных папок пользователей](javascript:TextPopup(this))


	При установленном флажке становится возможным создание для пользователей
	 [личных папок](Users/Admin_UserFolder.htm), доступ к которым
	 по умолчанию имеет только пользователь, за которым закреплена личная
	 папка.


[![](../Opened.gif)![](../Closed.gif)Разрешить
 отображение объектов в виде значков без учета прав доступа на объекты](javascript:TextPopup(this))


	По умолчанию флажок снят. После установки флажка появляется возможность
	 предварительного просмотра отчётов, созданных с помощью инструментов
	 «[Аналитические
	 панели](UIAdhoc.chm::/UiAdhoc_Purpose.htm)», «[Аналитические
	 запросы (OLAP)](UIExpress.chm::/purpose/UiExpress_Purpose.htm)», «[Отчеты](UIReport.chm::/UiReport_purpose.htm)»
	 и «[Анализ
	 временных рядов](UiDw.chm::/UiDw_Title.htm)», в виде [значков](UiNav.chm::/GUI/View.htm#extralarge_icons)
	 в навигаторе объектов.


	Если данный флажок установлен, то возможность предварительного просмотра
	 доступна даже для тех объектов, на которые у пользователя нет прав
	 на открытие (то есть чтение данных), но есть права на [чтение
	 дескриптора](../04_SecurityPolicy/Admin_Priv.htm) (данное право необходимо, чтобы объект отобразился
	 в навигаторе). При снятии флажка для таких объектов будет отображаться
	 пиктограмма класса объекта.


	Если установлены флажки «Использовать
	 мандатный доступ» или «Использовать
	 уровни безопасности», то возможность отображения объектов в
	 виде значков недоступна.


	Примечание.
	 Установка/снятие флажка протоколируется.


[![](../Opened.gif)![](../Closed.gif)Разрешить
 копирование в буфер обмена и копирование экрана](javascript:TextPopup(this))


	При установке флажка становится возможным копирование в буфер обмена
	 и копирование экрана. Снятие флажка блокирует данные возможности.
	 По умолчанию флажок установлен.


[![](../Opened.gif)![](../Closed.gif)Производить
 протоколирование операций с буфером обмена](javascript:TextPopup(this))


	При установленном флажке включается [аудит](../04_SecurityPolicy/Admin_Object_Classes.htm)
	 копирования информации в буфер обмена. Запись фиксируется в [протоколе
	 доступа](../05_AccessProtocol/Admin_AccessProtocol.htm) как операция «[Экспорт](../05_AccessProtocol/Admin_AccessProtocol_EvetsType.htm)».
	 При снятом флажке «Разрешить копирование
	 в буфер обмена и копирование экрана» будет осуществлен аудит
	 неудачных операций по работе с буфером обмена.


	Примечание.
	 Копирование экрана не протоколируется.


[![](../Opened.gif)![](../Closed.gif)Производить
 протоколирование информации о User Agent](javascript:TextPopup(this))


	По умолчанию флажок снят, информация о браузере, в котором открыто
	 веб-приложение (User-Agent), не отображается в [протоколе
	 доступа](../05_AccessProtocol/Admin_AccessProtocol.htm).


	Для отображения информации о браузере установите флажок. После чего
	 при каждом входе в систему будет отображаться информация о браузере
	 в столбце «Примечание» протокола
	 доступа.


[![](../Opened.gif)![](../Closed.gif)Разрешить
 расширенные возможности работы с БД и внешними файлами](javascript:TextPopup(this))


	По умолчанию флажок установлен. Снятие данного флажка позволяет
	 блокировать средства среды разработки в «Форсайт. Аналитическая платформа»,
	 при помощи которых опытные пользователи могут производить экспорт
	 данных из репозитория. Будет заблокировано выполнение системной сборки
	 Dal, класса File
	 в системной сборке IO, компонента
	 UIQuery. Также не будут выполнены
	 методы:


		- для интерфейса [IFile](ModIo.chm::/Interface/IFile/IFile.htm):


			- Open;


			- AppendBinary;


			- AppendText;


			- OpenBinaryReader;


			- OpenTextReader;


			- OpenBinaryWriter;


			- OpenTextWriter;


		- для интерфейса [IFileStream](ModIo.chm::/Interface/IFileStream/IFileStream.htm):


			- Create;


		- для интерфейса [IFileInfo](ModIo.chm::/Interface/IFileInfo/IFileInfo.htm):


			- Open;


			- AppendBinary;


			- AppendText;


			- OpenBinaryReader;


			- OpenTextReader;


			- OpenBinaryWriter;


			- OpenTextWriter.


[![](../Opened.gif)![](../Closed.gif)Очистка
 памяти](javascript:TextPopup(this))


	Установка флажка позволяет освобождаемую память перезаписывать нулями.


[![](../Opened.gif)![](../Closed.gif)Использовать
 роль приложения](javascript:TextPopup(this))


	Флажок «Использовать роль приложения»
	 доступен, только если репозиторий создан на базе [Microsoft SQL Server](Setup.chm::/01_SysReq/database_Support.htm).
	 Установите флажок для создания специальной роли на сервере СУБД. Специальная
	 роль создается под владельцем схемы и получает полные права доступа
	 к объектам БД.


	При открытии соединения с БД через «Форсайт. Аналитическая платформа»
	 выполняется активация специальной роли и текущая сессия получает полные
	 права доступа к объектам БД. Прямое обращение к объектам БД без специальной
	 роли недоступно.


	Для управления правами доступа субъектов к объектам на уровне платформы
	 [настройте дискреционный
	 метод](../04_SecurityPolicy/Admin_PermSep_D.htm).


	Примечание.
	 Использование роли приложения доступно, если создан [служебный
	 пользователь](Setup.chm::/05_RepoMngr/Setup_RepoMngr_Service_User.htm) и не используется [встроенная
	 авторизация](Access_control_settings.htm#buildin).


[![](../Opened.gif)![](../Closed.gif)Использовать
 хеширование паролей](javascript:TextPopup(this))


	Установка флажка позволяет заменять пароли пользователей на их хеши
	 и передавать полученные хеши для подключения к СУБД и входа в систему.
	 Также при подключении к базе данных будет хешироваться пароль, указанный
	 в настройках базы данных. По умолчанию флажок снят, при подключении
	 к базе данных пароли передаются в неизменном виде.


	Важно.
	 Если в СУБД используется несколько репозиториев, то при хешировании
	 паролей должны содержаться уникальные пользователи, которые не пересекаются
	 между репозиториями, и установлен флажок «Не
	 хешировать пароль администратора».


	При установке данного флажка становится доступным флажок «Не хешировать пароль администратора»,
	 который позволяет отменить хеширование пароля только для администратора.
	 Под администратором ADMIN понимается владелец схемы. По умолчанию
	 флажок установлен.


	Отмена хеширования пароля администратора актуальна:


		- для использования пароля администратора при подключении
		 к базе данных с помощью TOAD, PlsqlDev. Например, для создания
		 пользователя на уровне СУБД;


		- при указании пароля администратора в соединении веб-приложения
		 (web.config).


	После включения/отключения хеширования пароля при последующем подключении
	 к базе данных будет проверено, захеширован ли пароль пользователя,
	 если нет/да, то будет открыт диалог [смены
	 пароля](Users/Admin_User_EditPass.htm).


	Примечание.
	 Флажок «Использовать хеширование
	 паролей» доступен, если не используется [встроенная
	 авторизация](Access_control_settings.htm#buildin).


[![](../Opened.gif)![](../Closed.gif)Использовать
 встроенную авторизацию](javascript:TextPopup(this))


	По умолчанию флажок снят, аутентификация пользователей выполняется
	 только средствами СУБД. При установке флажка будет использоваться
	 авторизация средствами «Форсайт. Аналитическая платформа»:
	 для доступа к репозиторию метаданных в СУБД происходит соединение
	 под встроенным администратором (владельцем схемы в СУБД). Проверка
	 прав и паролей осуществляется на уровне «Форсайт. Аналитическая платформа»,
	 не используется механизм раздачи грантов через сервер БД. Также выполняется
	 хеширование паролей для сравнения хеша, сгенерированного при открытии
	 репозитория, с хешем, сохранённым в специальной таблице репозитория,
	 на уровне СУБД.


	При установке данного флажка становятся недоступными флажки «[Использовать
	 роль приложения](Access_control_settings.htm#role)», «[Использовать хеширование паролей](Access_control_settings.htm#hash)».


	Для сохранения логина и пароля владельца схемы СУБД в зашифрованном
	 виде используйте утилиту [PP.Util](setup.chm::/05_RepoMngr/Service_Applications/PP_Util.htm).


	Для настройки встроенной авторизации обратитесь к разделу «[Настройка
	 встроенной авторизации](Setup.chm::/UiWebSetup/Authentication/Integrated_Authorization.htm)».


	Важно.
	 Настройка встроенной авторизации выполняется один раз при первичной
	 настройке системы.


[![](../Opened.gif)![](../Closed.gif)Запретить доступ
 к объектам репозитория пользователям с привилегией «Создание, удаление
 пользователей»](javascript:TextPopup(this))


	Флажок доступен при активации режима [разделения
	 ролей между АИБ и ПА](../04_SecurityPolicy/Editor_of_Politicy/Security_EditorPoliticy_Adm.htm).


	Флажок определяет, будет ли у пользователей, имеющих привилегию
	 как у прикладного администратора, возможность открывать объекты репозитория.
	 При установленном флажке игнорируется наличие у пользователя привилегии «Право чтения и открытия всех объектов»,
	 и при попытке открыть объект репозитория будет выдано сообщение о
	 том, что недостаточно прав для выполнения операции.


[![](../Opened.gif)![](../Closed.gif)Запретить удаление
 пользователей с правами доступа на объекты репозитория пользователям с
 привилегией «Создание, удаление пользователей»](javascript:TextPopup(this))


	Флажок становится доступным при [активации
	 режима разделения ролей между АИБ и ПА](../04_SecurityPolicy/Editor_of_Politicy/Security_EditorPoliticy_Adm.htm).


	Если флажок установлен, то прикладной администратор (или пользователь
	 с привилегией «Создание, удаление
	 пользователей») не сможет удалить пользователя, который обладает
	 какими-либо правами доступа на объекты репозитория. При попытке удаления
	 будет выдано сообщение о том, что у пользователя есть права доступа.
	 Для удаления такого пользователя потребуется снятие всех имеющихся
	 прав доступа. При удалении пользователя проверяются только разрешения,
	 запреты игнорируются, то есть пользователь, у которого есть только
	 запреты, будет удалён.


[![](../Opened.gif)![](../Closed.gif)Запретить удаление
 групп с правами доступа на объекты репозитория](javascript:TextPopup(this))


	Флажок доступен при отключенном [режиме
	 разделения ролей между АИБ и ПА](../04_SecurityPolicy/Editor_of_Politicy/Security_EditorPoliticy_Adm.htm).


	После установки флажка и включении режима разделения ролей администратору
	 информационной безопасности будет запрещено удалять группы с правами
	 доступа на объекты репозитория.


[![](../Opened.gif)![](../Closed.gif)Использовать
 роли с правами Create, Drop и Alter User](javascript:TextPopup(this))


	По умолчанию флажок не установлен и становится доступным только
	 при активированном [режиме
	 разделения ролей между АИБ и ПА](../04_SecurityPolicy/Editor_of_Politicy/Security_EditorPoliticy_Adm.htm).


	Флажок отображается только для репозиториев, подключенных к Oracle.


	При установке флажка происходит создание двух ролей, содержащих
	 права на «Create user», «Drop User» и «Alter
	 user»:


		- первая роль включает в себя системные привилегии «Create user» и «Drop
		 user»;


		- вторая роль включает в себя системную привилегию «Alter user».


	При снятии флажка происходит удаление обеих ролей. Также удаление
	 ролей происходит при деактивации режима разделения ролей между прикладным
	 администратором и администратором информационной безопасности.


[![](../Opened.gif)![](../Closed.gif)Запретить
 повторное использование идентификаторов удаленных пользователей](javascript:TextPopup(this))


	Установка флажка позволяет задавать период, в течение которого запрещено
	 повторное использование идентификаторов удаленных пользователей.


[![](../Opened.gif)![](../Closed.gif)Заблокировать
 учетную запись пользователя, если пользователь неактивен](javascript:TextPopup(this))


	Установка флажка позволяет задавать период неактивности пользователя,
	 по истечению которого пользователь блокируется.


[![](../Opened.gif)![](../Closed.gif)Создавать
 субъекты с отложенной загрузкой](javascript:TextPopup(this))


	Установка флажка актуальна, если в системе содержится более двухсот
	 субъектов безопасности. Субъекты с признаком отложенной загрузки загружаются
	 после окончания загрузки обычных субъектов.


	При установленном флажке новые субъекты безопасности создаются с
	 признаком отложенной загрузки.


	По умолчанию флажок снят, субъекты безопасности не имеют признака
	 отложенной загрузки и загружаются одновременно.


Для применения заданных настроек контроля доступа:


	- в веб-приложении нажмите кнопку ![](../01_RunSecManager/SavePolicy.png) «Сохранить»
	 на [панели
	 инструментов](../01_RunSecManager/Admin_Organizational_Starting.htm) или на [боковой
	 панели](../01_RunSecManager/Admin_Organizational_Starting.htm);


	- в настольном приложении выполните
	 одно из действий:


		- выполните команду «Репозиторий
		 > Применить политику безопасности» в [главном
		 меню](../01_RunSecManager/Admin_Organizational_Starting.htm);


		- нажмите кнопку ![](../01_RunSecManager/SavePolicy.png)
		 «Применить политику безопасности»
		 на [панели
		 инструментов](../01_RunSecManager/Admin_Organizational_Starting.htm).


Примечание.
 Если параметры раздела были изменены, то при попытке перехода на другой
 раздел или при закрытии менеджера безопасности будет выдан запрос о применении
 измененных настроек.
