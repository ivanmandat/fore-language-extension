# Классы объектов


	Классы объектов объединяют инструменты аналитической обработки и
	 представления информации, разработки и администрирования, интеграционные
	 компоненты.


	В таблицах представлено описание системных объектов по классам.


## Средства бизнес-аналитики


			 Тип объектов
			 Описание


			 Аналитическая панель
			 Объект предназначен для конструирования интерактивных
			 панелей на данных из различных источников с помощью визуальных
			 блоков и отчетов.


			Подробное описание приведено в разделе «[Построение аналитических
			 панелей](UIAdhoc.chm::/UiAdhoc_Purpose.htm)».


			 Регламентный отчет
			 Объект предназначен для конструирования, экспорта и
			 печати отчетности с произвольной структурой, содержащей таблицы,
			 диаграммы, карты, основанные на многомерных и реляционных
			 источниках данных, и другие объекты. Доступна тонкая настройка
			 внешнего вида отчетов.


			Подробное описание приведено в разделе «[Построение отчёта
			 для печати](UIReport.chm::/UiReport_purpose.htm)».


			 Экспресс-отчет
			 Объект предназначен для оперирования данными из различных
			 источников и проведения экспресс-анализа с помощью аналитических
			 функций и визуализаторов.


			Подробное описание приведено в разделе «[Оперативный
			 просмотр и анализ данных (OLAP)](UIExpress.chm::/purpose/UiExpress_Purpose.htm)».


			 Рабочая книга
			 Объект предназначен для исследования данных, изменяющихся
			 во времени.


			Подробное описание приведено в разделе «[Анализ временных рядов](UiDw.chm::/UiDw_Title.htm)».


## Моделирование и прогнозирование


			 Тип объектов
			 Описание


			 Сценарий моделирования
			 Объект предназначен для рассмотрения альтернативного
			 варианта развития модели. Несколько сценариев позволяют рассматривать
			 несколько альтернативных вариантов развития модели. Является
			 одним из наиболее эффективных инструментов стратегического
			 анализа.


			Подробное описание приведено в разделе «[Сценарии](UiModelling.chm::/2_Container_of_Modeling/2_3_Work_object/2_3_4_Problem/uimodelling_problem_3.htm)».


			 Группа правил валидации
			 Несколько правил валидации, объединенных в группу для
			 последовательного выполнения.


			Подробное описание приведено в разделе «[Настройка группы
			 правил валидации](UiValidation.chm::/Grouping_types.htm)».


			 Задача моделирования
			 Объект, выполняющий последовательный расчет моделей
			 из указанной метамодели с целью получения выходных данных
			 в результирующих переменных.


			Подробное описание приведено в разделе «[Задача
			 моделирования](UiModelling.chm::/2_Container_of_Modeling/2_3_Work_object/2_3_4_Problem/uimodelling_problem.htm)».


			 Контейнер моделирования
			 Объект предназначен для конструирования моделей произвольной
			 сложности и проведения расчетов на их основе. Доступно применение
			 более 300 методов и функций.


			Подробное описание приведено в разделе «[Моделирование
			 в настольном приложении](UiModelling.chm::/1_Modelling/UiModelling_First.htm)».


			 Метамодель
			 Объект, объединяющий модели в последовательность, предназначенную
			 для расчета с помощью задачи моделирования.


			Подробное описание приведено в разделе «[Метамодель](UiModelling.chm::/2_Container_of_Modeling/2_3_Work_object/2_3_3_MetaModel/uimodelling_metamodel.htm)».


			 Модель
			 Объект предназначен для расчета результирующих переменных
			 путем преобразования исходных переменных статистическими и
			 математическими методами.


			Подробное описание приведено в разделе «[Модель](UiModelling.chm::/2_Container_of_Modeling/2_3_Work_object/2_3_2_Model/UiModelling_Model.htm)».


			 Переменная моделирования
			 Объект предназначен для получения исходных данных и
			 выгрузки результатов выполнения расчета.


			Подробное описание приведено в разделе «[Переменная
			 моделирования](UiModelling.chm::/2_Container_of_Modeling/2_3_Work_object/2_3_1_Value/uimodelling_work_object_value.htm)».


			 Правило валидации


			 Объект предназначен для проверки данных временных рядов
			 на соответствие заданным условиям и ограничениям.


			Подробное описание приведено в разделе «[Валидация данных](UiDw.chm::/Workbook/Data_validation.htm)».


## Многомерные объекты


			 Тип объектов
			 Описание


			 Объект кеша


			 Объект кеширования данных, предназначенный для хранения
			 данных в глобальном кеше. Данные, с которыми в текущий момент
			 осуществляется работа, проецируются в память компьютера и
			 используются инструментами «Форсайт. Аналитическая платформа».


			Подробное описание приведено в разделе «[Кеширование](UiNav.chm::/02_Navigator/UiNav_Cache.htm)».


			 Сегмент куба


			 Объект предназначен для хранения настроек измерений
			 и отметки элементов куба.


			Подробное описание приведено в разделе «[Настройка
			 измерений и отметка элементов](UiSelection.chm::/Selection/Dimension.htm)».


			 Контейнер сегментов куба


			 Объект предназначен для размещения сегментов куба и
			 таблиц, которые соответствуют отметкам измерений. Настроить
			 контейнер можно только на языке программирования [Fore](Fore.chm::/Fore_Title.htm).


			 Стандартный куб
			 Объект предназначен для логической организации многомерных
			 данных. Структура куба определяется набором справочников и
			 составом фактов. Особенность данного вида куба в том, что
			 помимо фактов, в которых хранятся данные, в кубе можно создавать
			 вычислимые факты и отношения.


			Подробное описание приведено в разделе «[Стандартный
			 куб](UiNavObj.chm::/Cube/CreateCube/Master_Standart/UiMd_Cube_CreateCube_Master_Standart.htm)».


			 Представление-куб
			 Куб, построенный на основе другого куба, но с уменьшенной
			 размерностью. Уменьшение размерности происходит за счет фиксации
			 измерений.


			Подробное описание приведено в разделе «[Представление-куб](UiNavObj.chm::/Cube/CreateCube/Master_Conception/UiMd_Cube_CreateCube_Master_Conception.htm)».


			 Виртуальный куб
			 Куб предназначен для объединения данных из двух и более
			 кубов. Для создания виртуального куба требуется несколько
			 кубов любого вида. Одинаковые измерения кубов-источников будут
			 преобразованы в измерения виртуального куба. Измерение фактов
			 в виртуальном кубе будет сформировано из элементов зафиксированных
			 измерений кубов-источников.


			Подробное описание приведено в разделе «[Виртуальный
			 куб](UiNavObj.chm::/Cube/CreateCube/Master_Virtual_Cube/UiMd_Cube_CreateCube_Master_Virtual.htm)».


			 Вычисляемый куб
			 Куб предназначен для получения новых данных, вычисленных
			 на основе данных из других кубов. Для создания вычисляемого
			 куба требуется один или несколько кубов любого вида. Измерения
			 в вычисляемом кубе могут быть скопированы из кубов-источников
			 или созданы на основе произвольных справочников. Факты могут
			 быть сформированы пользователем или также скопированы из куба-источника.
			 Все формулы для расчета данных куба задаются пользователем.


			Подробное описание приведено в разделе «[Вычисляемый
			 куб](UiNavObj.chm::/Cube/CreateCube/Master_Calculation/UiMd_Cube_CreateCube_Master_Calculation.htm)».


			 Автоматический куб
			 Куб предназначен для формирования источника данных по
			 принципу «сверху-вниз». Для создания автоматического куба
			 требуются база данных и справочники. На основе справочников
			 будут созданы измерения, а для хранения данных будет использоваться
			 указанная БД. Данные в куб могут быть внесены пользователем
			 или загружены из указанного куба.


			Подробное описание приведено в разделе «[Автоматический
			 куб](UiNavObj.chm::/Cube/CreateCube/Master_Auto/UiMd_Cube_CreateCube_Master_Auto.htm)».


			 База данных временных рядов
			 Объект предназначен для хранения временных рядов, которые
			 используются для анализа и статистической обработки данных.


			Подробное описание приведено в разделе «[Загрузка
			 данных в существующий куб/БД временных рядов](KeCubes.chm::/Samples/LoadDataInExistingObject.htm)».


			 Куб ADOMD
			 Объект предназначен для получения доступа к многомерным
			 источникам данных, которые созданы на основе [Microsoft
			 Analysis Services](UiNavObj.chm::/ADOMD/Connect_to_Microsoft_Analysis_Services.htm), [SAP
			 NetWeaver BW](UiNavObj.chm::/ADOMD/UiDb_relational_ADOMD.htm), либо произвольных провайдеров,
			 установленных в операционной системе.


			Подробное описание приведено в разделе «[Подключение
			 к многомерным базам данных ADOMD](UiNavObj.chm::/ADOMD/UiDb_relational_ADOMD.htm)».


			 Загрузчик в куб
			 Объект предназначен для заполнения куба данными из другого
			 куба.


			Подробное описание приведено в разделе «[Загрузчик
			 данных](UiNavObj.chm::/Cube/Work_Cube/Loader/UiMd_Cube_Work_Cube_Loader.htm)».


## Справочники


			 Тип объектов
			 Описание


			 Измерение атрибутов


			 Объект базы данных временных рядов, в результате работы
			 с временными рядами. Измерение атрибутов содержит атрибуты
			 показателей, которые являются ссылками на справочник, а также
			 атрибут «Значение».


			Подробное описание приведено в разделе «[IRubricator.FactAndValuesDimension](KeCubes.chm::/Interface/IRubricator/IRubricator.FactAndValuesDimension.htm)».


			 Виртуальное измерение


			 Объект, предназначенный для хранения частного измерения
			 в отчёте, который содержит [несколько
			 источников](UIExpress.chm::/ChangeNote/UiExpress_Tabl_ChangeNote_PrivDim.htm).


			Подробное описание приведено в разделе «[Использование
			 частных измерений](UiSelection.chm::/Selection/Working_with_private_dimensions.htm)».


			 Составной справочник боковика


			 Справочник, который создается автоматически в виде скрытого
			 дочернего объекта в отчёте, содержащий свойства области данных.


			Подробное описание приведено в разделе «[Свойства
			 области данных](UiReport.chm::/desktop/AreaData/Param/UiReport_AreaData_Param.htm)».


			 Справочник НСИ
			 Совокупность элементов, содержащая сведения научного,
			 производственного и прикладного характера, объединенных единой
			 тематикой.


			Подробное описание приведено в разделе «[Работа
			 со справочником](UiNavObj.chm::/reference_book/Work/Work_Dictionary.htm)».


			 Составной справочник НСИ
			 Справочник, который включает, как свои собственные элементы,
			 так и заимствованные элементы из других справочников НСИ.


			Подробное описание приведено в разделе «[Работа
			 со справочником](UiNavObj.chm::/reference_book/Work/Work_Dictionary.htm)».


			 Группа элементов справочника
			 Объект репозитория, используемый в справочниках для
			 ограничения списка доступных элементов. Группы элементов создаются
			 как дочерние объекты справочника и не отображаются в навигаторе
			 объектов.


			Подробное описание приведено в разделе «[Группы
			 элементов](UiNavObj.chm::/reference_book/look-and-feel_Reference_book/UiMd_reference_book_look-and-feel_Group.htm)».


			 Схема отметки элементов справочника
			 Объект репозитория, используемый в справочниках для
			 автоматизации выделения элементов по заданным правилам. Схемы
			 отметки создаются как дочерние объекты справочника и не отображаются
			 в навигаторе объектов, однако могут быть использованы везде,
			 где используется справочник.


			Подробное описание приведено в разделе «[Схемы
			 отметки](UiNavObj.chm::/reference_book/look-and-feel_Reference_book/UiMd_reference_book_look-and-feel_Scheme.htm)».


			 Составной справочник
			 Справочник, который создается автоматически в виде скрытого
			 дочернего объекта в стандартном и вычисляемом кубе, лежащий
			 в основе локального измерения фактов.


			Подробное описание приведено в разделе «[Структура
			 составного справочника](KeDims.chm::/Intro/Dimensions/KeDims_StructureCompound.htm)».


			 Справочник сценариев
			 Справочник, который создается автоматически в виде скрытого
			 дочернего объекта в стандартном и вычисляемом кубе, лежащий
			 в основе локального измерения фактов.


			Подробное описание приведено в разделе «[Общие принципы
			 программирования с использованием сборки Ms](KeMs.chm::/Intro/KeMs_Programming.htm)».


			 Объект импорта в справочник НСИ
			 Объект репозитория для загрузки данных из произвольных
			 источников в справочник НСИ.


			Подробное описание приведено в разделе «[Создание
			 схем импорта/экспорта](UiNavObj.chm::/reference_book/Master_RDS_reference_book/Import.htm#import)».


			 Объект экспорта из справочника НСИ
			 Объект репозитория для экспорта в файлы-приёмники.


			Подробное описание приведено в разделе «[Создание
			 схем импорта/экспорта](UiNavObj.chm::/reference_book/Master_RDS_reference_book/Import.htm#export)».


			 Составной календарный справочник
			 Составной календарный справочник представляет собой
			 объединение нескольких календарных справочников с различной
			 динамикой.


			Подробное описание приведено в разделе «[Календарный
			 справочник](UiNavObj.chm::/reference_book/Master_Calendar_reference_book/Master_Calendar.htm)».


			 Табличный справочник
			 Справочник, основанный на таблице данных определенного
			 формата, каждая запись которой представляет собой элемент
			 справочника.


			Подробное описание приведено в разделе «[Табличный
			 справочник](UiNavObj.chm::/reference_book/Master_Table_reference_book/Master_Table.htm)».


			 Календарный справочник
			 Справочник, реализующий иерархическое представление
			 временной шкалы с заданной детализацией. Содержимое календарного
			 справочника генерируется автоматически.


			Подробное описание приведено в разделе «[Календарный
			 справочник](UiNavObj.chm::/reference_book/Master_Calendar_reference_book/Master_Calendar.htm)».


			 Вычисляемый справочник
			 Справочник, наполнение данными которого осуществляется
			 по алгоритму пользователя, реализованного в виде процедуры
			 на внутреннем языке программирования.


			Подробное описание приведено в разделе «[Вычисляемый
			 справочник](UiNavObj.chm::/reference_book/Master_Calculation_reference_book/Master_Calculation.htm)».


			 Метасправочник


			 Объект базы данных временных рядов, который создается
			 автоматически. Метасправочник фактов содержит данные о временных
			 рядах, присутствующих в базе данных временных рядов.


			Подробное описание приведено в разделе «[Особенности
			 хранения данных в базе данных временных рядов](KnowledgeBase.chm::/01_fore/knowledgebase_kb000036.htm)».


			 Справочник ADOMD
			 Объект репозитория, предназначенный для получения доступа
			 к многомерным источникам данных, которые созданы на основе
			 [Microsoft
			 Analysis Services](UiNavObj.chm::/ADOMD/Connect_to_Microsoft_Analysis_Services.htm), либо [SAP
			 NetWeaver BW](UiNavObj.chm::/ADOMD/UiDb_relational_ADOMD.htm).


			Подробное описание приведено в разделе «[Подключение
			 к многомерным базам данных ADOMD](UiNavObj.chm::/ADOMD/UiDb_relational_ADOMD.htm)».


			 Справочник правил валидации


			 Справочник, основанный на правилах [валидации
			 данных](UiAnalyticalArea.chm::/Analysis/Data_validation.htm).


			 Конструируемый справочник


			 Объект [стандартного](UiNavObj.chm::/Cube/CreateCube/Master_Standart/UiMd_Cube_CreateCube_Master_Standart.htm)
			 или [вычисляемого](UiNavObj.chm::/Cube/CreateCube/Master_Calculation/UiMd_Cube_CreateCube_Master_Calculation.htm)
			 куба, в результате [локального
			 измерения фактов](UiNavObj.chm::/Cube/CreateCube/Master_Standart/UiMd_Cube_CreateCube_Master_Standart_1.htm) или [сохранения](UiSelection.chm::/Selection/Hierarchies.htm#save_hierarchy)
			 пользовательской иерархии как альтернативной.


## Объекты базы данных


			 Тип объектов
			 Описание


			 База данных
			 Объект репозитория, содержащий основные настройки для
			 подключения к существующей базе данных, созданной на сервере
			 СУБД.


			Подробное описание приведено в разделе «[Подключение
			 к реляционным базам данных](UiNavObj.chm::/database/UiDb_database.htm)».


			 Репозиторий НСИ
			 Централизованное хранилище нормативно-справочной информации,
			 которое обеспечивает согласованность и консолидацию данных,
			 устраняет избыточность информации и оптимизирует поиск нужных
			 сведений.


			Подробное описание приведено в разделе «[Репозиторий
			 НСИ](UiNavObj.chm::/repository_RDS/RepositoryRDS.htm)».


			 Каталог ADOMD
			 Объект репозитория, предназначенный для получения доступа
			 к многомерным источникам данных, которые созданы на основе
			 [Microsoft
			 Analysis Services](UiNavObj.chm::/ADOMD/Connect_to_Microsoft_Analysis_Services.htm), либо [SAP
			 NetWeaver BW](UiNavObj.chm::/ADOMD/UiDb_relational_ADOMD.htm).


			Подробное описание приведено в разделе «[Подключение
			 к многомерным базам данных ADOMD](UiNavObj.chm::/ADOMD/UiDb_relational_ADOMD.htm)».


			 Объект импорта
			 Дочерний объект БД временных рядов, предназначенный
			 для импорта данных из внешних источников в базу данных временных
			 рядов.


			Подробное описание приведено в разделе «[Импорт данных](UiDw.chm::/Import/Import.htm)».


			 Объект репликации
			 Объект предназначен для репликации базы данных временных
			 рядов.
