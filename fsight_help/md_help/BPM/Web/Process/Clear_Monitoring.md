# Очистка мониторинга процессов


Доступно для администратора ADMIN, который является владельцем схемы.


Остановка всех активных экземпляров процессов, валидация сегментов данных,
 инициализации контейнера задач и очищение кеша репозитория осуществляется
 с помощью модуля «Очистка мониторинга».
 Для этого:


	- Выполните остановку запущенных задач, если [планировщик задач](uiappsrv.chm::/UiAppSrv_purpose.htm)
	 был запущен. Для этого:


		- в ОС Linux остановите службу scheduler.service с помощью
		 команды:


sudo systemctl stop scheduler.service


		- в ОС Windows выполните команду «Закрыть»
		 в контекстном меню значка ![](appsrv.png).


	- Откройте на редактирование модуль «Очистка
	 мониторинга» с идентификатором BP_UTILS_MONITOR_CLEAN в [навигаторе
	 объектов](GetStarted.chm::/Interface/Interface_Navigator.htm).


Примечание.
 При необходимости очистки кеша репозитория раскомментируйте строку cleanCache(mb).


	- [Выполните
	 модуль](uidevenv.chm::/01_Development_Environment/02_Work_in_Development_Environment/Run_Debug.htm).


Важно. Досрочное
 прерывание выполнения модуля «Очистка
 мониторинга» может привести к сбою в работе бизнес-процессов.
