# Настройка ролевой модели


Ролевая модель задаёт права пользователей на доступ к данным, используемым
 при построении процесса. Для определения прав доступа пользователей к
 данным используйте сегменты данных или [объекты
 полномочий](../../Desktop/Facility_Agreement/Create_Authorization_Object.htm), с помощью которых задаётся связь субъектов безопасности
 (пользователей/групп пользователей) со срезами данных куба.


Права доступа к информационным объектам могут быть двух типов:


	- Статические. Действуют
	 постоянно на протяжении всего времени работы пользователя с источником
	 данных;


	- Динамические. Раздаются
	 только во время выполнения отдельных шагов, к которым привязаны [сегменты данных](Setting_Segments.htm)/объекты полномочий.


Для настройки прав доступа пользователей к данным, используемым при
 построении процесса:


	- [Создайте группы
	 пользователей](Admin.chm::/03_Admin/Groups/Admin_Groups.htm), учитывая [роль
	 пользователя](Role_model.htm#role) при работе с процессом. Роли пользователей определяют
	 доступ к [мониторингу
	 процесса](../Starting/Monitoring_process_execution.htm).


	- [Включите
	 пользователей в группы](Admin.chm::/03_Admin/Users/Admin_UserProp_Groups.htm).


	- Выполните одно из действий:


		- при использовании сегментов данных:


			- Создайте [сегменты
			 данных](uinavobj.chm::/Cube/segments/segments.htm)


			- [Настройте взаимодействие
			 процесса с сегментами данных](Setting_Segments.htm).


		- при использовании объектов полномочий
		 выполните их [создание](../../Desktop/Facility_Agreement/Create_Authorization_Object.htm)
		 и [настройку](../../Desktop/Facility_Agreement/Configuring_the_Authorization_Object.htm).


Примечание.
 В веб-приложении для [разграничения
 прав доступа](../Process/Basic_properties.htm#control_mode) субъектов безопасности рекомендуется использовать сегменты
 данных.


При совместной работе над процессом выделяются следующие
 роли пользователей:


[![](../../opened.gif)![](../../closed.gif)Администратор](javascript:TextPopup(this))


		- владелец схемы ADMIN.


		- пользователь, входящий во встроенную группу «[Администраторы](Admin.chm::/03_Admin/Groups/Admin_Groups.htm)».


	Администратору, с настроенным  [полным
	 доступом к процессу](../Process/Basic_properties.htm#role_of_the_curator), доступны:


		- [полный
		 мониторинг процесса](../Starting/Monitoring_process_execution.htm#monitoring) с отображением экземпляров всех активных
		 процессов;


		- [запуск](../Process/Start_Process.htm) новых
		 экземпляров процесса;


		- [выполнение](../MonitoringProcess/Manual_mode.htm)/[отклонение](../MonitoringProcess/Reject_step.htm) шагов
		 во всех экземплярах соответствующего процесса;


		- [управление
		 экземплярами процесса](../MonitoringProcess/process_instance_management.htm).


[![](../../opened.gif)![](../../closed.gif)Ответственный
 за процесс](javascript:TextPopup(this))


	Ответственному за процесс доступны:


		- [полный
		 мониторинг процесса](../Starting/Monitoring_process_execution.htm#monitoring) с отображением всех экземпляров процесса;


		- [запуск](../Process/Start_Process.htm) новых
		 экземпляров процесса;


		- [выполнение](../MonitoringProcess/Manual_mode.htm)/[отклонение](../MonitoringProcess/Reject_step.htm) шагов
		 во всех экземплярах процесса;


		- [управление
		 экземплярами процесса](../MonitoringProcess/process_instance_management.htm).


	Для назначения пользователя, ответственным за процесс, используйте
	 вкладку «[Базовые
	 свойства](../Process/Basic_properties.htm)» окна «Настройки».


	Примечание.
	 На роль ответственного за процесс может быть назначен пользователь/группа
	 пользователей.


[![](../../opened.gif)![](../../closed.gif)Ответственный
 за выполнение шага](javascript:TextPopup(this))


	Ответственному за выполнение шага доступны:


		- [пользовательский
		 мониторинг процесса](../Starting/Monitoring_process_execution.htm#monitoring_simple) со всеми шагами, за выполнение которых
		 он назначен ответственным;


		- [выполнение](../MonitoringProcess/Manual_mode.htm)/[отклонение](../MonitoringProcess/Reject_step.htm) назначенных
		 шагов.


	Для назначения ответственного за выполнение шага используйте вкладку
	 «[Настройки](../Process/StepsProcess/CreateStepsProcess.htm#steps_process_firs_page)»
	 панели параметров шага. Для него доступен [пользовательский
	 мониторинг процесса](../Starting/Monitoring_process_execution.htm#monitoring_simple).


	Примечание.
	 На роль ответственного за выполнение шага может быть назначен пользователь/группа
	 пользователей.


	При совместной работе пользователей учитывайте следующие особенности:


		- если на роль ответственного за выполнение шага назначена
		 группа пользователей, то в мониторинге процесса ответственный
		 за процесс/администратор может выбрать исполнителя для выполнения
		 шага. Исполнителем является [назначенный
		 пользователь](../MonitoringProcess/user_on_task.htm) из группы ответственных за выполнение определенного
		 шага экземпляра процесса. После выбора исполнителя шаг экземпляра
		 процесса доступен только исполнителю;


		- если на роль ответственного за выполнение шага назначена
		 группа пользователей, которая содержит другие группы пользователей,
		 то их выбор в качестве исполнителя будет недоступен;


		- если на роль ответственного за выполнение шага назначена
		 группа пользователей, а исполнитель не задан, то для всех пользователей
		 группы экземпляр процесса отображается  в мониторинге процесса
		 и соответствующий шаг доступен для выполнения.
