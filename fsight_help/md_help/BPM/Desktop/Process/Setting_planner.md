# Настройка планировщика для бизнес-процессов


Настройка планировщика для бизнес-процессов включает определение типа
 исполнения бизнес-процессов, настройку параметров для серверного исполнения
 процессов и указание адреса BI-сервера. Для настройки планировщика используйте
 страницу «Планировщик» в окне
 «[Настройка
 параметров и администрирование бизнес-процессов](Setting_and_administrating_process_parameters.htm)».


[![](../../opened.gif)![](../../closed.gif)Для открытия
 страницы «Планировщик»](javascript:TextPopup(this))


	- Откройте на просмотр утилиту «Настройка
	 параметров и администрирование бизнес-процессов» с идентификатором
	 BP_UTIL, которая входит в состав компонента «Бизнес-процессы».
	 Будет открыто окно «Настройка параметров
	 и администрирование бизнес-процессов».


		- Перейдите на страницу «Планировщик».


![](Setting_and_administrating_planner.png)


Задайте параметры:


	- Клиентское
	 исполнение. Установите переключатель для выполнения процессов
	 на локальном компьютере пользователя. Клиентское исполнение требуется
	 только для отладки настроек бизнес-процессов. В промышленной разработке
	 используйте серверное исполнение процессов. Для запуска клиентского
	 исполнения процессов выполните [запуск
	 процессов](Start_Process.htm);


Примечание.
 Для корректной работы [объектов
 полномочий](../Facility_Agreement/Create_Authorization_Object.htm), процессов и связанных с ними [форм ввода](DataEntryForms.chm::/DataEntryForms_Purpose.htm)
 исполнение процессов должно быть серверным. Клиентское исполнение используется
 только для отладки настроек бизнес-процессов.


	- Серверное
	 исполнение. Установите переключатель для выполнения процессов
	 на сервере. Предварительно установите и настройте [сервер приложений](Setup.chm::/UiWebSetup/AppServer.htm).
	 После выбора серверного исполнения потребуются [дополнительные
	 настройки](Setting_Server_Process_Execution.htm). При установке переключателя будут доступны параметры:


		- Исполнителей. Задайте
		 количество [задач](uiappsrv.chm::/3_Work_Tasks/Work_Task.htm),
		 которые будут созданы в [контейнере запланированных
		 задач](uiappsrv.chm::/Getting_Started.htm) с идентификатором BPM_EXECUTOR_CONTAINER,
		 и интервал между запусками задач в поле «Интервал,
		 мин». Интервал задаётся в минутах и применяется к каждому
		 исполнителю. После [запуска процесса](Start_Process.htm)
		 один исполнитель выполняет один шаг и меняет его [статус](../Starting/Monitoring_process_execution.htm#status)
		 в зависимости от результата выполнения. По умолчанию указывается
		 пять исполнителей и пять минут. Увеличьте количество исполнителей
		 и интервал между запусками задач при необходимости выполнения
		 большого количества шагов в нескольких процессах одновременно;


Примечание.
 [Параллельное выполнение](Setting_up_steps_conditions.htm#gateway)
 шагов в рамках одного процесса выполняется разными исполнителями, если
 есть свободные. Если свободных исполнителей нет, то параллельные шаги
 выполняются последовательно одним и тем же исполнителем.


		- Проверять
		 срыв сроков исполнения шагов. Установите флажок для проверки
		 нарушения сроков выполнения шагов и задайте интервал в поле «Интервал, мин», с которым будет
		 производиться проверка. При нарушении сроков выполнения шагов
		 будет отправлено e-mail оповещение и/или выполнен Fore-метод,
		 в зависимости от настроек на странице «[События](StepsProcess/CreateStepsProcess.htm)»;


Примечание.
 Для проверки нарушения сроков выполнения шагов при выборе клиентского
 исполнения процессов выполните операцию «Проверить
 срыв сроков» на странице «[Администрирование](Administrating.htm)».
 Доступно только в настольном приложении.


	- URL BI-сервиса.
	 Установите флажок и задайте адрес BI-сервера в поле «Адрес
	 BI-сервера» для выполнения ресурсоёмких шагов ![](Calculation.png) «[Расчёт](StepsProcess/Calculation.htm)»
	 и ![](fore_method_call.png) «[Вызов Fore-метода](StepsProcess/Fore.htm)» на [BI-сервере](Setup.chm::/UiWebSetup/UiWebSetup_TitlePage.htm).
	 По умолчанию флажок снят.


Примечание.
 Выполнение шагов на BI-сервере доступно, если установлен флажок «[Выполнять
 шаги расчёта и методов Fore на BI-сервере](Basic_properties.htm#calculation_bi_server)» при настройке базовых
 свойств процесса.


При настройке выполнения ресурсоёмких шагов
 убедитесь, что на [BI-сервере](Setup.chm::/UiWebSetup/UiWebSetup_TitlePage.htm)
 размещён файл [registry.reg](Setup.chm::/UiWebSetup/03_Setup_Web/config_and_setting.htm)
 или [Metabases.xml](Setup.chm::/06_AK_Client_Config/Configuring_repository_list_in_the_system_registry.htm)
 с настройками подключения к репозиториям.


После изменения настроек планировщика нажмите кнопку «Применить».
 Будет выдано информационное окно для подтверждения изменений. После подтверждения
 изменений повторно подключитесь к репозиторию.


При выборе клиентского исполнения дополнительные настройки не потребуются.


При выборе серверного исполнения перейдите к [настройке
 серверного исполнения процессов](Setting_Server_Process_Execution.htm).
