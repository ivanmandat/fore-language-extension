# Моделирование в настольном приложении


В настольном приложении моделирование и прогнозирование выполняется
 в рамках контейнера моделирования.


Контейнер моделирования - объект
 репозитория, предназначенный для формирования математических моделей процессов
 (явлений) и проведения расчетов на их основе. Результатом расчетов является
 решение различного типа задач:


	- задачи исследования данных;


	- задачи прогнозирования;


	- задачи управления;


	- задачи оптимального управления;


	- задачи имитации и другие.


Решение задач моделирования и прогнозирования выполняется путем создания
 и настройки взаимодействия следующих объектов:


	- [моделей](../2_Container_of_Modeling/2_3_Work_object/2_3_2_Model/UiModelling_Model.htm). Предназначены
	 для расчета моделируемых переменных путем преобразования исходных
	 переменных статистическими и математическими методами. Расчёты ведутся
	 с учетом календарной динамики модели (годы, полугодия, кварталы, месяцы,
	 дни);


	- [переменных](../2_Container_of_Modeling/2_3_Work_object/2_3_1_Value/uimodelling_work_object_value.htm). Предназначены
	 для получения исходных данных и выгрузки результатов выполнения расчёта;


	- [метамоделей](../2_Container_of_Modeling/2_3_Work_object/2_3_3_MetaModel/uimodelling_metamodel.htm). Предназначены
	 для организации цепочки расчета и объектов модели в требуемом порядке
	 и для описания логики моделирования произвольной задачи/процесса;


	- [задач](../2_Container_of_Modeling/2_3_Work_object/2_3_4_Problem/uimodelling_problem.htm). Предназначены для
	 выполнения последовательного расчёта метамодели с целью получения
	 выходных данных в результирующих переменных. По окончании расчета
	 результаты выгружаются в переменные;


	- [сценариев моделирования](../2_Container_of_Modeling/2_3_Work_object/2_3_4_Problem/uimodelling_problem_3.htm).
	 Предназначены для создания в моделируемой переменной ряда, в который
	 будут выгружены данные после окончания расчёта.


Кроме объектов моделирования контейнер может содержать ярлыки на любые
 другие объекты репозитория платформы, таких как отчеты, формы и т.д.


## Процесс моделирования


Процесс моделирования в настольном приложении заключается в последовательном
 создании и настройке объектов контейнера моделирования, направленном на
 решение задач математического моделирования динамических систем.


Процесс моделирования начинается с создания [моделей](../2_Container_of_Modeling/2_3_Work_object/2_3_2_Model/UiModelling_Model.htm),
 предназначенных для преобразования данных различными методами с учетом
 календарной динамики. В рамках модели добавляются [переменные](../2_Container_of_Modeling/2_3_Work_object/2_3_1_Value/uimodelling_work_object_value.htm),
 содержащие исходные данные и предназначенные для возвращения результатов
 расчета.


Количество, параметры переменных и моделей зависят от решаемой задачи
 моделирования.


Существующие модели необходимо выстроить в порядке их расчета. Для этой
 цели должна быть создана [метамодель](../2_Container_of_Modeling/2_3_Work_object/2_3_3_MetaModel/uimodelling_metamodel.htm),
 представляющая собой иерархическую последовательность различных элементов,
 образующих цепочку расчета задачи моделирования. Цепочка может содержать
 модели, другие цепочки расчета (метамодели) и папки, помогающие упорядочить
 структуру цепочки расчета.


Расчет моделей, входящих в метамодель, осуществляется с помощью [задачи
 моделирования](../2_Container_of_Modeling/2_3_Work_object/2_3_4_Problem/uimodelling_problem.htm). Расчет задачи моделирования позволяет получить выходные
 данные в результирующих переменных. Задача может быть рассчитана по различным
 [сценариям](../2_Container_of_Modeling/2_3_Work_object/2_3_4_Problem/uimodelling_problem_3.htm),
 позволяющим получить прогнозы по моделируемым переменным, например, исходя
 из оптимистичного и пессимистичного прогноза сценарных переменных.


После расчета задачи совокупность полученных данных можно проанализировать,
 открыв [отчет
 по выходным переменным](../2_Container_of_Modeling/2_3_Work_object/2_3_4_Problem/uimodelling_problem_4.htm) или посмотрев [историю
 расчета](../2_Container_of_Modeling/2_3_Work_object/2_3_6_History/uimodelling_work_object_History.htm).
