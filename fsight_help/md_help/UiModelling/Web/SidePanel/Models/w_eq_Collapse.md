# Коллапс


Метод рассчитывает данные верхнего уровня на основе данных нижнего
 уровня. Применяется в задачах, требующих агрегации данных. Например, данные,
 рассчитанные по месяцам, суммируются для получения квартальных данных.


Примечание.
 Для агрегации данных календарная динамика входной переменной должна быть
 меньше динамики моделируемой переменной. Если календарная динамика входной
 переменной больше или совпадает с динамикой моделируемой переменной, то
 создание уравнения «Коллапс»
 недоступно.


Для настройки параметров метода используйте вкладку «Уравнение»
 на боковой панели.


[![](../../../opened.gif)![](../../../closed.gif)Для отображения
 вкладки](javascript:TextPopup(this))


		- Убедитесь, что боковая панель отображается;


		- Выберите моделируемую переменную или одну из связей уравнения
		 вида «Коллапс»;


		- Перейдите на вкладку «Уравнение»
		 на боковой панели.


![](Collapse.gif)


Параметры метода:


	- Вид коллапса. Укажите
	 используемый метод агрегации:


	-


		- Сумма. Сумма значений
		 элементов динамики. Используется по умолчанию;


		- Среднее. [Среднее
		 значение](Lib.chm::/05_Statistics/UiModelling_Avg.htm) элементов динамики;


		- Минимум. Минимальное
		 значение элементов динамики;


		- Максимум. Максимальное
		 значение элементов динамики;


		- Первое значение.
		 Первое имеющееся значение элементов динамики;


		- Последнее значение.
		 Последнее имеющееся значение элементов динамики;


		- Размах. Разность
		 между максимальным и минимальным значениями элементов динамики;


		- Стандартное отклонение.
		 [Стандартное
		 отклонение](Lib.chm::/05_Statistics/UiModelling_Stddev.htm) по динамике;


	- Период действия формулы.
	 Укажите период, на котором будет выполнена агрегация данных:


	-


		- Период идентификации.
		 Агрегация рассчитывается только на данных периода идентификации;


		- Период прогнозирования.
		 Агрегация рассчитывается только на данных периода прогнозирования;


		- Весь период. Агрегация
		 рассчитывается на данных периодов идентификации и прогнозирования.


Для настройки границ периодов используйте
 вкладку «[Периоды
 расчёта](../UiModelling_w_pp_CalcPeriod.htm)» на боковой панели.
