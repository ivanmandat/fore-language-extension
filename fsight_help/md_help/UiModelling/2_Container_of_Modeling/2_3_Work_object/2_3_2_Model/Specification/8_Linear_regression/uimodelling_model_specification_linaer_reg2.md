# Линейная регрессия (оценка методом инструментальных переменных)


Для оценки коэффициентов [линейной
 регрессии](Lib.chm::/01_Regression_models/Line_Regression.htm) используется [метод
 инструментальных переменных](Lib.chm::/01_Regression_models/UiModelling_LinearRegr_Method_Instr_Var.htm).


Панель «Спецификация» для данного
 метода:


![](equation_%20linear_regression2.gif)


В строке «Уравнение» выводится
 уравнение модели, отражающее в сокращенном виде преобразование над моделируемой
 переменной и параметры метода.


[![](../../../../../opened.gif)![](../../../../../closed.gif)Для преобразования
 моделируемой/исходной переменной](javascript:TextPopup(this))


	По умолчанию над моделируемой/исходной переменной не выполняются
	 дополнительные преобразования перед расчетом модели.


	Для задания [дополнительного
	 преобразования](../../../UiModelling_work_Changes.htm) моделируемой/исходной переменной перед расчетом
	 модели используйте раскрывающий список «Преобразование
	 моделируемой/исходной переменной». По умолчанию преобразование
	 входной переменной совпадает с преобразованием моделируемой переменной.


[![](../../../../../opened.gif)![](../../../../../closed.gif)Для задания
 константы](javascript:TextPopup(this))


	Для выбора режима задания константы в модели используйте переключатели
	 в группе «Константа»:


		- Нет. Константа в
		 модели не используется;


		- Оценить. Значение
		 константы будет оценено автоматически в процессе расчета метода.
		 Полученное значение будет отображено в поле ввода справа;


		- Указание значения константы.
		 Значение константы задается пользователем в соответствующем поле,
		 которое доступно для ввода после установки данного переключателя.


[![](../../../../../opened.gif)![](../../../../../closed.gif)Для задания
 факторов](javascript:TextPopup(this))


	Для задания набора факторов (входных переменных), влияющих на моделируемую
	 переменную, используйте таблицу «Факторы».


	Совет. Для получения
	 после расчета модели оцененного значения коэффициента фактора используйте
	 столбец «Оценка» в таблице «Факторы» или панель «[Идентифицированное уравнение](../../Standart_Model/identified_equation.htm)».


	Основные принципы работы с факторами приведены в разделе «[Работа
	 с факторами (входными переменными)](../../Standart_Model/UiModelling_Model_Factor_work.htm)».


[![](../../../../../opened.gif)![](../../../../../closed.gif)Для задания
 инструментальных переменных](javascript:TextPopup(this))


	Для задания набора инструментальных переменных, применяемых для
	 оценки коэффициентов, используйте таблицу «Инструментальные
	 переменные».


	Порядок работы с инструментальными переменными совпадают с порядком
	 работы с факторами и приведен в разделе «[Работа
	 с факторами (входными переменными)](../../Standart_Model/UiModelling_Model_Factor_work.htm)».


[![](../../../../../opened.gif)![](../../../../../closed.gif)Для
 обработки пропусков в данных фактора](javascript:TextPopup(this))


	По умолчанию к фактору применяется метод обработки пропусков, установленный
	 для модели.


	Для изменения метода обработки пропусков в данных фактора:


		- Снимите флажок «Применить
		 обработку пропусков модели» в контекстном меню фактора.
		 В таблице «Факторы» в
		 столбце «Обработка пропусков»
		 отобразится используемый метод обработки пропусков.


		- Перейдите на панель «[Дополнительные
		 параметры](../../Standart_Model/UiModelling_Model_Extra.htm)» и задайте требуемый метод обработки пропусков в
		 данных фактора.


	Для фактора будет применяться выбранный метод обработки пропусков.


	Для применения к фактору метода обработки пропусков, заданного для
	 модели, установите флажок «Применить
	 обработку пропусков модели» в контекстном меню фактора.


[![](../../../../../opened.gif)![](../../../../../closed.gif)Для обработки
 пропусков в данных инструментальной переменной](javascript:TextPopup(this))


	Обработка пропусков в инструментальной переменной выполняется аналогично
	 обработке пропусков в факторе.


[![](../../../../../opened.gif)![](../../../../../closed.gif)Для задания
 порядка авторегрессии](javascript:TextPopup(this))


	Для использования в модели коэффициентов авторегрессии:


		- Установите флажок «Установить
		 порядок авторегрессии».


		- В соответствующем поле введите номера или диапазоны порядка
		 авторегрессии, разделенные запятыми. Диапазон порядка авторегрессии
		 указывается через знак «-».


	Для модели будет задан порядок авторегрессии и будут рассчитаны
	 верхняя и нижняя динамические доверительные границы прогнозного ряда.


	Для исключения из модели коэффициентов авторегрессии снимите флажок
	 «Установить порядок авторегрессии».


[![](../../../../../opened.gif)![](../../../../../closed.gif)Для задания
 порядка скользящего среднего](javascript:TextPopup(this))


	Для использования в модели скользящего среднего:


		- Установите флажок «Установить
		 порядок скользящего среднего».


		- В соответствующем поле введите номера или диапазоны порядка
		 скользящего среднего, разделенные запятыми. Диапазон порядка скользящего
		 среднего указывается через знак «-».


	Для модели будет задан порядок скользящего среднего.


	Для исключения из модели коэффициентов скользящего среднего снимите
	 флажок «Установить порядок скользящего
	 среднего».


[![](../../../../../opened.gif)![](../../../../../closed.gif)Для использования
 дополнительных инструментов](javascript:TextPopup(this))


	По умолчанию константа модели используется в качестве дополнительной
	 инструментальной переменной. Для исключения константы из инструментальных
	 переменных снимите флажок «Использовать
	 константу в качестве инструментальной переменной».


	Если заданы порядки авторегрессии и/или скользящего среднего, то
	 по умолчанию лагированные значения объясняемой и объясняющей переменных
	 используются в качестве дополнительных инструментов. Для прекращения
	 этого снимите флажок «Использовать
	 лагированные значения объясняемой и объясняющей переменных в качестве
	 дополнительных инструментов».


[![](../../../../../opened.gif)![](../../../../../closed.gif)Для корректировки
 прогноза](javascript:TextPopup(this))


		- Установите флажок «Корректировка
		 прогноза».


		- Задайте переменную, которая будет использоваться для корректировки
		 прогноза. Данная переменная не включаются в идентифицированное
		 уравнение модели.


	В модели будет применяться корректировка прогноза.


	При работе моделирования и прогнозирования в [режиме
	 на переменных](../../../../../1_Modelling/Start_Modelling.htm) учитывайте следующие особенности:


		- при выборе переменной, количество измерений которой не совпадает
		 с количеством измерений моделируемой переменной, будет открыт
		 диалог «[Изменение размерности](../../../UiModelling_ChangeDimension.htm)».
		 В данном диалоге выполните фиксацию по измерениям, отсутствующим
		 у моделируемой переменной;


		- доступны следующие кнопки для работы с переменной корректировки
		 прогноза:


			- Создать. Создает
			 переменную (без данных), используемую для корректировки прогноза. Динамика
			 переменной соответствует динамике модели. Созданная переменная
			 располагается в корне контейнера моделирования, имеет наименование
			 «Фактор корректировки прогноза»
			 и автоматически будет открыта для редактирования;


			- Зафиксировать.
			 Осуществляет фиксацию переменной. Открывает диалог «[Изменение размерности](../../../UiModelling_ChangeDimension.htm)». Если
			 размерности фактора корректировки прогноза и моделируемой
			 переменной совпадают, то кнопка недоступна.
