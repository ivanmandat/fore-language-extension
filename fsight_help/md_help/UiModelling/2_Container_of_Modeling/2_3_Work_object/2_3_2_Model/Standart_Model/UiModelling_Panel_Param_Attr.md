# Параметры модели


Для настройки параметров модели предназначена панель «Параметры»:


![](Std_Model_Panel_Param_Attr.gif)


Примечание.
 Если моделирование и прогнозирование запущено в [режиме
 на переменных](../../../../1_Modelling/Start_Modelling.htm), то работа с панелью «Параметры»
 отличается от стандартной и описана в разделе «[Параметры модели, основанной на переменных](uimodelling_model_3.htm)».


На панели «Параметры» задайте
 моделируемую, исходную переменные и метод расчета модели.


В каждом поле для выбора переменной содержится кнопка с наименованием
 источника данных и все атрибуты данного источника, необходимые для однозначного
 определения переменной. Наименования переменных отображаются в следующем
 формате: Наименование источника данных
 переменной > Атрибут 1 = Значение атрибута 1 > Атрибут
 n = Значение атрибута n. Наименование источника данных
 переменной приводится после кнопки ![](../del_value.gif).


При работе с моделью учитывайте следующие особенности:


	- если источник данных переменной не содержит календарного измерения,
	 то переменная будет рассматриваться как константа;


	- если справочник единиц измерения не входит в уникальный ключ
	 базы данных временных рядов, являющейся источником данных переменной,
	 то используемые единицы измерения отображаются серым курсивом в полях
	 моделируемой и исходной переменных;


	- если в источник данных, используемый моделью, добавляется новое
	 измерение, то для корректной работы модели задайте с помощью Fore
	 элемент по умолчанию для данного измерения. Используйте свойство IRdsDictionary.DefaultElementKey.


[![](../../../../opened.gif)![](../../../../closed.gif)Для
 выбора моделируемой переменной](javascript:TextPopup(this))


		- Выберите источник данных переменной. Если при [создании
		 контейнера моделирования](../../../2_1_Create_Cont_Modeling/UiModelling_Create_Container.htm) был задана база данных временных
		 рядов, то она по умолчанию предлагается в качестве источника данных
		 моделируемой переменной. Если база данных временных рядов не задана,
		 то в поле «Моделируемая переменная»
		 отображается кнопка «Источник».
		 Нажмите её и выберите требуемый источник данных. Для выбора доступны
		 кубы и базы данных временных рядов.

		После выбора источника данных в поле «Моделируемая
		 переменная» будут отображены атрибуты выбранного источника.


		- Задайте значения всех атрибутов источника данных. Выбор
		 значения атрибута производится из раскрывающегося списка, который
		 вызывается щелчком мыши по атрибуту. В списке доступна только
		 единичная отметка.

		Если не все атрибуты помещаются на панели, то отображаются кнопки
		 ![](b_prev.gif)
		 и ![](b_next.gif)
		 для перемещения между атрибутами.


	После выбора моделируемой переменной станет доступен выбор вида
	 модели и выбор исходной переменной.


[![](../../../../opened.gif)![](../../../../closed.gif)Для
 выбора вида модели](javascript:TextPopup(this))


		- Убедитесь, что задана моделируемая переменная.


		- В поле «Вид модели»
		 укажите метод расчета модели. Выбор производится из раскрывающегося
		 списка.


	В зависимости от [выбранного
	 метода](../Specification/UiModelling_Model_Specification.htm) будет отображен набор дополнительных панелей. Если для
	 выбранного метода доступен расчет с помощью пакета R, то будет отображен
	 дополнительный флажок «Использовать
	 при расчете подключение к R». Подробнее см. раздел «[Использование R при
	 расчете модели](R_Parms.htm)».


[![](../../../../opened.gif)![](../../../../closed.gif)Для
 выбора исходной переменной](javascript:TextPopup(this))


		- Убедитесь, что задана моделируемая переменная.


		- В поле «Исходная переменная»
		 задайте переменную, к данным которой будет применен указанный
		 метод расчета.


	Набор атрибутов исходной и моделируемой переменных должен совпадать.


	По умолчанию исходная переменная совпадает с моделируемой переменной.
	 Выбор исходной переменной аналогичен выбору моделируемой переменной.


	Примечание.
	 Параметр «Исходная переменная»
	 отсутствует в моделях «[Детерминированное
	 уравнение](../Specification/4_Deterministic_equation/uimodelling_model_specification_deter.htm)» и «[R](../Specification/R.htm)».


[![](../../../../opened.gif)![](../../../../closed.gif)Для
 настройки агрегации значений атрибутов исходной переменной](javascript:TextPopup(this))


	- Вызовите контекстное меню атрибута.


	- В группе «Агрегация»
	 выберите один из доступных методов агрегации.


	- Для расчета агрегации с учетом иерархии значений атрибута установите
	 переключатель «По иерархии».
	 По умолчанию флажок снят и расчет агрегации выполняется без учета
	 иерархии.


	- Для расчета агрегации только по тем значениям атрибута, которые
	 соответствуют определенным условиям, выполните команду «Расширенная
	 фильтрация». Будет отображен диалог «Расширенный
	 фильтр», в котором задайте требуемые условия. Порядок работы
	 с данным диалогом приведен в разделе «[Настройка фильтрации по значениям атрибутов](uinav.chm::/GUI/UFE/AttributeFiltering.htm)».


	- После закрытия контекстного меню атрибута щелкните по нему и
	 в отобразившемся списке значений выберите агрегируемые. Доступна множественная
	 отметка. Если ни одно значение не выбрано, то агрегация рассчитывается
	 по всем значениям.


В результате будет настроена агрегация значений атрибутов переменной.
 Для сброса агрегации выберите значение «Нет»
 в группе «Агрегация» в контекстном
 меню атрибута.


[![](../../../../opened.gif)![](../../../../closed.gif)Для
 изменения источника данных переменной](javascript:TextPopup(this))


		- Нажмите кнопку с наименованием источника данных переменной.
		 Будет отображен список с доступными источниками данных.


		- Выберите требуемый источник. Для выбора доступны стандартные
		 кубы, виртуальные кубы и базы данных временных рядов.


	Примечание.
	 С помощью внутреннего языка Fore в качестве источника данных можно
	 использовать представление-куб.


	Для переменной будет использоваться новый источник данных. Если
	 набор атрибутов выбранного источника совпадает с набором атрибутов
	 предыдущего источника, то значения атрибутов будут сохранены. Если
	 набор атрибутов различается, то потребуется задать значения атрибутов
	 переменной.


	Примечание.
	 Если изменен источник данных моделируемой переменной и набор атрибутов
	 выбранного источника отличается от набора атрибутов предыдущего источника,
	 то будет сброшена исходная переменная и выбранный вид модели.


[![](../../../../opened.gif)![](../../../../closed.gif)Для
 сброса переменной/вида модели](javascript:TextPopup(this))


	Нажмите кнопку ![](../del_value.gif), расположенную
	 в поле выбора переменной/вида модели.


	При сбросе:


		- моделируемой переменной.
		 Сбрасываются параметры всей модели и значения атрибутов моделируемой
		 переменной. Источник данных переменной сброшен не будет;


		- вида модели.
		 Сбрасывается вид модели, исходная переменная сбрасывается и недоступна
		 для выбора;


		- исходной переменной.
		 Сбрасываются значения атрибутов исходной переменной. Источник
		 данных переменной сброшен не будет.


[![](../../../../opened.gif)![](../../../../closed.gif)Для
 редактирования внутренних переменных](javascript:TextPopup(this))


	Внутренние переменные - это переменные, доступные только для [внутренних моделей](Internal_Model.htm), созданных в рамках
	 одной [метамодели](../../2_3_3_MetaModel/uimodelling_metamodel.htm).


	Для редактирования внутренних переменных нажмите кнопку «Редактировать
	 внутренние переменные» на панели «Параметры».
	 Будет открыт диалог «[Редактирование
	 внутренних переменных](Internal_Model.htm)», в котором задайте необходимые настройки.


[![](../../../../opened.gif)![](../../../../closed.gif)Для
 задания значений дополнительных атрибутов](javascript:TextPopup(this))


	Задание дополнительных атрибутов доступно, если источником данных
	 моделируемой переменной является база данных временных рядов, содержащая
	 дополнительные атрибуты. Дополнительные атрибуты - это атрибуты ряда,
	 присутствующие в базе данных временных рядов, но не являющиеся обязательными
	 для однозначной идентификации переменной.


	Для задания значения дополнительных атрибутов нажмите кнопку «Задать другие атрибуты» на панели
	 «Параметры». Будет открыт
	 диалог «[Значения
	 атрибутов](AdditionalAttributes.htm)», в котором задайте необходимые значения.


## Использование параметров модели


Если модель содержит параметры или входит в метамодель, содержащую [параметры](../../2_3_3_MetaModel/Metamodel_params.htm), то
 в качестве значения атрибута можно указать параметр.


[![](../../../../opened.gif)![](../../../../closed.gif)Для
 установки параметра в качестве значения атрибута](javascript:TextPopup(this))


		- Вызовите контекстное меню атрибута.


		- Выберите пункт «Установить
		 параметр» и во всплывающем меню установите переключатель,
		 соответствующий необходимому параметру.


	Выбранный параметр будет установлен в качестве значения атрибута.
	 Наименование параметра будет отображаться в фигурных скобках.


[![](../../../../opened.gif)![](../../../../closed.gif)Для
 сброса параметра, установленного для атрибута](javascript:TextPopup(this))


		- Вызовите контекстное меню атрибута.


		- Выберите пункт «Установить
		 параметр» и во всплывающем меню установите переключатель
		 «Нет».


	Параметр перестанет использоваться в качестве значения атрибута.


[![](../../../../opened.gif)![](../../../../closed.gif)Для
 задания значения параметра, установленного для атрибута](javascript:TextPopup(this))


		- Убедитесь, что заданы все параметры модели.


		- Перейдите на вкладку «[Просмотр
		 результатов](uimodelling_model_view.htm)».


		- В группе «Значения параметров»
		 введите значения требуемых параметров.


	Указанные значения будут использоваться в качестве значений атрибутов
	 при расчете модели.
