# Тип задачи


На панели редактируются основные параметры задачи моделирования.


![](problem_type.gif)


	- Метамодель. Укажите
	 метамодель, цепочка которой будет рассчитана.


	- Тип задачи. Выберите
	 тип решаемой задачи:


	-


		- Задача
		 прогнозирования. Состоит в расчете всех моделей в каждой
		 точке сценария отдельно: сначала рассчитываются все модели по
		 первой точке сценария, затем происходит расчет всех моделей по
		 второй точке сценария и так далее. Таким образом, применяется
		 поточечный режим расчета;


		- Задача
		 управления. Находится в стадии разработки;


		- Задача оптимального управления.
		 Состоит в отыскании таких значений управляющих переменных, при
		 которых будет выполнено условие оптимизации значения целевой функции.
		 Предварительно должна быть проведена [интеграция
		 с LPSolve](UiNav.chm::/02_Navigator/CommonSettings/Integration.htm#lpsolve);


		- Целевая
		 задача. Состоит в отыскании таких значений управляющих
		 переменных, при которых значение целевой функции будет максимально
		 соответствовать заданной траектории. Предварительно должна быть
		 проведена [интеграция
		 с LPSolve](UiNav.chm::/02_Navigator/CommonSettings/Integration.htm#lpsolve);


		- Имитационная
		 задача. Находится в стадии разработки;


		- Forward-Looking
		 прогнозирование. Состоит в расчете моделей, содержащих
		 переменные с лагами не только в предыдущие моменты времени (t-lag),
		 но и в последующие (t+lag);


		- Задача
		 трансформации. Состоит в расчете моделей последовательно
		 по всем точкам сценария: сначала рассчитывается первая модель
		 по всем точкам сценария, затем происходит расчет второй модели
		 по всем точкам сценария и так далее. Таким образом, применяется
		 векторный режим расчета;


		- Пользовательская
		 задача оптимизации. Состоит в расчете задачи оптимизации
		 по алгоритму пользователя. Может использоваться для интеграции
		 с различными оптимизационными пакетами;


		- Задача
		 генерации формул. Состоит в конвертации формул моделирования
		 в формулы, предназначенные для расчёта в таблице отчёта. Задача
		 применяется для подключения внешних цепочек трансформации в инструменты
		 отчётности и используется расширением «[Интерактивные
		 формы ввода данных](DataEntryForms.chm::/dataentryforms_purpose.htm)»;


		- Задача
		 многопоточной трансформации. Состоит в расчёте задачи трансформации
		 в режиме, который в следующих версиях продукта «Форсайт. Аналитическая платформа»
		 будет адаптирован под многопоточность. Это позволяет увеличить
		 скорость расчёта.


Примечание.
 Задача многопоточной трансформации поддерживает расчёт только [многомерных
 циклов](../2_3_3_MetaModel/MultiDimIterator.htm) и моделей [детерминированного
 уравнения](../2_3_2_Model/Specification/4_Deterministic_equation/uimodelling_model_specification_deter.htm) и [матричной
 агрегации](../2_3_2_Model/MatrixAggregation/MatrixAggregation.htm).

 панелей.
