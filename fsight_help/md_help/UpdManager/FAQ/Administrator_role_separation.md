# Как происходит процесс обновления при использовании разделения ролей
 администраторов?


Процесс обновления при использовании разделения ролей администраторов
 проходит в три этапа:


	- Создание обновления (в репозитории разработчиков).


	- Проведение обновления администратором информационной
	 безопасности (в клонированном репозитории заказчика).


	- Проведение обновления прикладным администратором (в
	 клонированном репозитории заказчика).


Файл обновления, который содержит данные об обновляемых объектах и правах
 доступа к ним, будет создан, открыт или установлен каждым из администраторов
 с разным набором доступных элементов.


Для администратора информационной безопасности будет доступна работа
 с:


	- [правами доступа](../CreateUpdate/AddAccessPermissions.htm)
	 к объектам репозитория и к специальным объектам;


	- [папками](../CreateUpdate/Admin_CreateUpdate_CreateFolder.htm).


Для прикладного администратора будет доступна работа с:


	- [объектами
	 репозитория](../CreateUpdate/Admin_CreateUpdate_AddObjects.htm) (новыми для базы заказчика и уже существующими в ней);


	- [субъектами
	 безопасности](../CreateUpdate/Admin_CreateUpdate_AddUser.htm);


	- [специальными
	 объектами](../CreateUpdate/Admin_CreateUpdate_SpecialObjects.htm);


	- [папками](../CreateUpdate/Admin_CreateUpdate_CreateFolder.htm);


	- [данными
	 об удалении объекта](../CreateUpdate/Admin_CreateUpdate_AddDel.htm);


	- [комментариями](../CreateUpdate/Admin_CreateUpdate_AddComent.htm);


	- [SQL-операторами](../CreateUpdate/Admin_CreateUpdate_AddSQL.htm).


Так же имеются отличия в процессе [создания](../CreateUpdate/Admin_CreateUpdate.htm),
 [открытия](../OpenUpdate/Admin_OpenUpdate.htm), [сохранения](../SaveUpdate/Admin_SaveUpdate.htm#in_file)
 и [установки обновления](../Admin_UpMBObj_DoUpdate.htm), при
 [поиске](../CreateUpdate/Admin_CreateUpdate_SerchObjects.htm)
 элемента обновления.


В соответствии с привилегиями пользователя [параметры
 обновления объекта](../Admin_UpMBObj_OBJUpdateParameters.htm) доступны не полностью. Несоответствующие привилегиям
 значения параметров не отображаются. Если параметр включает только несоответствующие
 привилегиям значения, он отображается только для чтения в текущем значении.


Для администратора информационной безопасности также доступна установка
 обновлений, содержащих [модуль обновления](../CreateUpdate/Features/Admin_CreateUpdate_Features_Module.htm),
 без возможности [просматривать содержимое
 модуля](../Admin_Update_ViewText.htm).
