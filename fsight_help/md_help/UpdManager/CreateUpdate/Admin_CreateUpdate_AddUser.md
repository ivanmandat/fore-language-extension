# Добавление субъекта безопасности


При добавлении в обновление субъекта безопасности переносятся учётные
 данные [групп](Admin.chm::/03_Admin/Groups/Admin_Groups.htm)
 или отдельных [пользователей](Admin.chm::/03_Admin/Users/Admin_Users.htm):


	- [общие
	 свойства](Admin.chm::/03_Admin/Users/Admin_UserProp_Common.htm):


		- наименование пользователя или группы;


		- полное наименование пользователя;


		- описание пользователя или группы;


		- пароль пользователя;


	- [атрибуты](Admin.chm::/03_Admin/Users/Admin_UserProp_Attributes.htm).
	 Обновление атрибутов производится по идентификатору следующим образом:


		- если атрибуты имеют одинаковые идентификаторы, то они обновляются;


		- если атрибут отсутствует, то он добавляется.


Для добавления субъекта безопасности в обновление:


	- в веб-приложении выберите тип объекта обновления «Субъект
	 безопасности» в раскрывающемся меню кнопки ![](admin_createupdate_addsql_button.gif)
	 «Добавить объект в обновление»
	 на панели инструментов;


	- в настольном приложении выполните одно из действий:


		- выполните команду «Добавить
		 субъект безопасности» в контекстном меню [структуры
		 обновления](../Admin_UpMBObj_RunManager.htm);


		- выберите тип объекта обновления «Субъект
		 безопасности» в раскрывающемся меню кнопки ![](admin_createupdate_addsql_button.gif) «Добавить
		 объект в обновление» на панели инструментов.


После выполнения одного из действий будет открыт стандартный диалог
 выбора субъекта безопасности «[Выбор
 пользователей и групп](Admin.chm::/03_Admin/Admin_UsersGroups.htm)».


Примечание.
 При установке обновления, содержащего субъекты безопасности, пользователь
 должен обладать привилегией «[Изменение
 прав пользователей, раздача ролей, изменение политики](Admin.chm::/04_SecurityPolicy/Admin_Priv.htm#changing_user_permissions)».
 При [разделении
 ролей администраторов](Admin.chm::/04_SecurityPolicy/Editor_of_Politicy/Security_EditorPoliticy_Adm.htm) и в зависимости от выбранного
 [метода
 разграничения доступа](Admin.chm::/04_SecurityPolicy/Admin_SecPolicy.htm) возможны [различия
 в доступе](../FAQ/FAQ.htm) к элементам обновления, а также к параметрам обновления
 объекта.


## Особенности добавления отдельных пользователей


Для корректного переноса отдельных пользователей на сервер базы данных
 установите флажок «[Делать
 пользователя подключаемым с сервера, если он существует на уровне СУБД](../UpdateProcess_Parameters.htm#when_creating_update)».
 В этом случае при [установке обновления](../Admin_UpMBObj_DoUpdate.htm)
 новый пользователь не создаётся, если он уже существует на сервере. Если
 флажок не установлен, то на сервере будет создаваться новый пользователь.
 В случае, если данный пользователь уже существует, появляется соответствующая
 ошибка.


## Перенос доменных субъектов безопасности


Для корректного переноса [доменных
 пользователей](Admin.chm::/03_Admin/Users/Admin_UserCreate_Domain.htm) или [групп](admin.chm::/03_Admin/Groups/Admin_GroupsCreateDomain.htm)
 рекомендуется проверять идентичность [настроек LDAP](Admin.chm::/03_Admin/communication_with_ad.htm)
 в файлах [settings.xml](setup.chm::/UiWebSetup/03_Setup_Web/BI_Server_Settings_XML.htm),
 расположенных в различных окружениях. Если настройки совпадают, то для
 корректной работы с доменными пользователями или группами переносите их
 в рамках одного приложения, в котором они были подключены: либо в веб-приложении,
 либо в настольном приложении.


## Перенос субъектов безопасности с сохранением
 членства в группах


Для переноса между репозиториями учётных данных группы вместе с её членами:


	- Добавьте [группу
	 пользователей](Admin.chm::/03_Admin/Groups/Admin_Groups.htm)
	 в обновление.


	- Добавьте [отдельных
	 пользователей](Admin.chm::/03_Admin/Users/Admin_Users.htm),
	 являющихся [членами
	 группы](Admin.chm::/03_Admin/Groups/Group_Members.htm),
	 которую необходимо перенести между репозиториями.


	- Установите для группы или каждого пользователя, состоящего в
	 ней, значение «Да» для параметра
	 «[Переносить
	 членство в группах](../Admin_UpMBObj_OBJUpdateParameters.htm#transfer_group_memberships)».


После выполнения действий при [установке
 обновления](../Admin_UpMBObj_DoUpdate.htm) будет перенесена группа и пользователи, состоящие в группе,
 с сохранением членства в группах. Если в целевом репозитории отсутствует
 пользователь в заданной группе, то он будет создан с сохранением членства
 в группе.


Для переноса между репозиториями пользователей с сохранением членства
 в группах:


	- Добавьте [отдельных
	 пользователей](Admin.chm::/03_Admin/Users/Admin_Users.htm),
	 являющихся [членами
	 групп](Admin.chm::/03_Admin/Groups/Group_Members.htm),
	 которые содержатся в целевом репозитории.


	- Установите для каждого пользователя значение «Да»
	 для параметра «[Переносить
	 членство в группах](../Admin_UpMBObj_OBJUpdateParameters.htm#transfer_group_memberships)».


После выполнения действий при [установке
 обновления](../Admin_UpMBObj_DoUpdate.htm) будут перенесены пользователи с сохранением членства в
 группах. Если в целевом репозитории отсутствует группа, в которой состоит
 пользователь, то членство в этой группе не будет сохранено. При переносе
 пользователей группа, в которой состоит пользователь, должна содержаться
 в обновлении или в целевом репозитории.


Также в целевом репозитории могут быть очищены списки групп, в которые
 входят субъекты безопасности, в процессе обновления. Например, если в
 исходном репозитории субъект входит в группу А, а в целевом репозитории
 - в группу Б, то после применения обновления в целевом репозитории субъект
 будет входить только в группу А. Для очистки списков групп установите
 флажок «[Очищать списки групп, в которые
 входят субъекты безопасности](../UpdateProcess_Parameters.htm#when_creating_update)» при настройке [параметров
 процесса обновления](../UpdateProcess_Parameters.htm).


Примечание.
 Для применения прав доступа, распространяемых на членов группы пользователей,
 дополнительно [обновите
 пользователей](admin.chm::/03_admin/users/admin_user_update.htm) в менеджере безопасности после установки
 обновления.
