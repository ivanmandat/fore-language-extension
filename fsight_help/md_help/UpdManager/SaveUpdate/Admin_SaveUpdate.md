# Сохранение обновления


Созданное обновление можно сохранить как [файл обновления](#in_file)
 и как [объект репозитория](#in_repository). После сохранения
 обновления перейдите к его [установке](../Admin_UpMBObj_DoUpdate.htm).


Также после сохранения обновления можно [экспортировать](../Admin_UpMBObj_Export.htm)
 список обновления в файл *.txt.


## Сохранение обновления в файл


Для сохранения созданного обновления в файл:


	- в веб-приложении:


		- нажмите кнопку ![](Save_as_file.png) «Сохранить
		 обновление в файл» на панели инструментов;


		- нажмите кнопку «Готово».


При выполнении одного из действий появится
 информационное окно для подтверждения [сортировки
 объектов](../Admin_UpMBObj_ObjView.htm#order). При подтверждении объекты будут автоматически отсортированы,
 в противном случае порядок объектов останется неизменным.


Обновление будет сохранено файл с наименованием
 «Обновление_<наименование репозитория_время_дата>»;


	- в настольном приложении:


		- выполните команду главного меню «Обновление
		 > Сохранить» или «Обновление
		 > Сохранить в файл»;


		- нажмите кнопку ![](../UpdateMBObjects/Admin_UpMBObj_MainToolBar_1_3.gif)
		 «Сохранить» на панели
		 инструментов.


При выполнении одного из действий появится
 информационное окно для подтверждения [сортировки
 объектов](../Admin_UpMBObj_ObjView.htm#order). При подтверждении объекты будут автоматически отсортированы,
 в противном случае порядок объектов останется неизменным.


Укажите имя и месторасположение файла в открывшемся
 стандартном окне сохранения файла. Для файла обновления используется расширение
 *.pefx. При нажатии кнопки «Сохранить»
 файл обновления будет сохранен.


При сохранении обновления, открытого из файла,
 стандартное окно не открывается. Для сохранения обновления под новым именем
 выполните команду главного меню «Обновление
 > Сохранить как». После выполнения данной команды укажите имя
 и месторасположение сохраняемого обновления в открывшемся окне.


В зависимости от того, создан файл обновления
 или открыт, доступны типы файлов при сохранении:


		- Файл обновления.pefx.
		 Файл, содержащий данные об обновляемых объектах и правах доступа
		 к ним;


		- Файл обновления для администратора.pefx.
		 Файл, содержащий данные об обновляемых объектах;


		- Файл обновления для АИБа.pefx.
		 Файл, содержащий данные о правах доступа к обновляемым объектам.


Примечание.
 При разделении [ролей
 администраторов](Admin.chm::/04_SecurityPolicy/Editor_of_Politicy/Security_EditorPoliticy_Adm.htm) при сохранении недоступен тип «Файл обновления.pefx».


## Сохранение обновления в репозиторий


Для сохранения созданного обновления в репозиторий выполните команду
 главного меню «Обновление > Сохранить
 в репозиторий». Укажите имя и месторасположение сохраняемого обновления
 в открывшемся окне с объектами репозитория.


Примечание.
 Сохранение обновления в репозиторий доступно только в настольном приложении.


Для сохранения обновления, открытого из репозитория:


	- выполните команду главного меню «Обновление
	 > Сохранить»;


	- нажмите кнопку ![](../UpdateMBObjects/Admin_UpMBObj_MainToolBar_1_3.gif)
	 «Сохранить» на панели инструментов.


Для сохранения обновления, открытого из репозитория, под новым именем
 выполните команду главного меню «Обновление
 > Сохранить как». После выполнения данной команды укажите имя
 и месторасположение сохраняемого обновления в открывшемся окне.


При выполнении одного из действий обновление будет сохранено как объект
 репозитория. Данный объект можно использовать для создания новых файлов
 обновлений с расширением *.pefx.
