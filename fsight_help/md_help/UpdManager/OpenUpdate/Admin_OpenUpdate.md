# Особенности при открытии обновления


При [открытии готового
 обновления](../Admin_UpMBObj_RunManager.htm#open_update) в менеджере обновлений учитывайте следующие особенности:


	- если в [структуре обновления](../Admin_UpMBObj_RunManager.htm)
	 содержатся объекты, то при открытии готового обновления будет открыто
	 окно «Загрузка объектов»:


![](open_update.png)


Выберите действие с помощью переключателей:


		- Очистить обновление и загрузить
		 объекты. Текущие объекты обновления будут заменены объектами,
		 содержащимися в открывающемся обновлении;


		- Добавить сохранённые объекты
		 в текущее обновление. Объекты, содержащиеся в открывающемся
		 обновлении, будут добавлены к текущим объектам обновления. Если
		 объекты совпадают, то они будут продублированы;


		- Дополнить текущее обновление.
		  Объекты, содержащиеся в открывающемся обновлении, будут
		 добавлены к текущим объектам обновления. Если объекты совпадают,
		 то текущие объекты будут заменены объектами из обновления.


	- если обновление открыто в исходном репозитории и версии/даты
	 изменения объектов обновления не соответствуют версиям/датам последнего
	 изменения объектов в навигаторе объектов, то [синхронизируйте
	 объекты обновления с репозиторием](../AdminUpMGObj_Syn.htm). Актуально при открытии обновления
	 в настольном приложении;


	- если параметры обновления конкретных объектов изменены после
	 открытия обновления в исходном репозитории, то [синхронизируйте
	 объекты обновления с репозиторием](../AdminUpMGObj_Syn.htm). Например, в файл обновления
	 была добавлена таблица, в параметрах обновления которой указано «[Метаданные
	 объекта](../CreateUpdate/Features/Admin_CreateUpdate_Features_Tables.htm)». При открытии файла обновления в исходном репозитории
	 для данной таблицы были изменены параметры обновления на «Метаданные
	 и данные объекта». Без синхронизации объектов обновления с
	 репозиторием данные объекта не будут загружены. Актуально при открытии
	 обновления в настольном приложении.


Примечание.
 При [разделении
 ролей администраторов](Admin.chm::/04_SecurityPolicy/Editor_of_Politicy/Security_EditorPoliticy_Adm.htm) и в зависимости от выбранного
 метода [разграничения
 доступа](Admin.chm::/04_SecurityPolicy/Admin_SecPolicy.htm) возможно открытие файла обновления, только
 если в нем присутствуют [доступные
 пользователю элементы](../FAQ/Administrator_role_separation.htm). Если они отсутствуют, то файл обновления не
 будет открыт.
