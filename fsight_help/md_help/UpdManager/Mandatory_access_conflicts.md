# Конфликты мандатного доступа


На вкладке «Мандатный доступ»
 страницы «[Информация](Admin_UpMBObj_DoUpdateFile.htm)»
 отображаются все категории секретности и уровни безопасности, которые
 присутствуют в репозитории разработчиков.


Вкладка отображается, если мандатный доступ используется в обеих репозиториях
 (разработчиков и заказчика).


Примечание.
 Вкладка «Мандатный доступ» доступна
 только в настольном приложении.


На вкладке содержится информация о наименовании категории секретности,
 уровень безопасности и описание конфликта.


Возможные конфликты:


	- Отсутствие категории.
	 Конфликт отображается для категории при отсутствии её в репозитории
	 заказчика. При этом все уровни данной категории выводятся с таким
	 же конфликтом;


	- Отсутствие уровня. Конфликт
	 отображается для уровня при отсутствии его в репозитории заказчика,
	 но при наличии в системе соответствующей категории для этого уровня.


![](UpdateMBObjects/mandatory_access_conflicts.png)


Установка обновления невозможна при наличии конфликтов мандатного доступа.
 Устраните конфликты в [менеджере
 безопасности](Admin.chm::/04_SecurityPolicy/Admin_MandatAccess.htm) вручную и повторите попытку.
