# Как сделать доступным создание блоков, предназначенных для работы с
 отчетами?


По умолчанию в инструменте «Аналитические
 панели» доступно создание только блоков типа «[Регламентный отчет](../Blocks/Reports/Procedural_report.htm)»
 и «[Аналитическая
 панель](../Blocks/Reports/Dashboard.htm)». Для создания блоков типа «[Экспресс-отчет](../Blocks/Reports/Express_Report.htm)»
 и «[Временные ряды](../Blocks/Reports/TimeSeries.htm)» требуются дополнительные
 настройки. Данные настройки не влияют на доступность отчётов в сохраненных
 аналитических панелях.


## Решение для веб-приложения


	- Откройте для редактирования файл «[PP.xml](Setup.chm::/UiWebSetup/03_Setup_Web/PP_config_Java.htm)».


	- В файл [PP.xml](Setup.chm::/UiWebSetup/03_Setup_Web/PP_config_Java.htm)
	 в раздел <pp> добавьте
	 следующий код:


<modules>
    <dashboard>
        <ReportBox TimeSeries="true" ExpressReport="true"/>
    </dashboard>
</modules>


	- Сохраните изменения и закройте файл [PP.xml](Setup.chm::/UiWebSetup/03_Setup_Web/PP_config_Java.htm).


	- Перезапустите BI-сервер и серверную часть веб-приложения.


Значение TimeSeries="true" отвечает за доступность блока «[Временные ряды](../Blocks/Reports/TimeSeries.htm)», ExpressReport="true"
 - блока «[Экспресс-отчет](../Blocks/Reports/Express_Report.htm)». Можно указывать
 только одно из значений, если нужно разрешить создание только одного из
 вида блоков.


### Пример файла PP.xml, содержащего настройки для доступности создания
 всех видов блоков для работы с отчетами


<pp>
    <service url="http://localhost/FPBI_App_v10.x/axis2/services/PP.SOM.Som"/>
    <metabase id="FPRepository"/>
    <cultures>
        <culture cultureName="ru"/>
        <culture cultureName="en"/>
    </cultures>
    <Modules>
        <Dashboard>
            <ReportBox TimeSeries="true" ExpressReport="true"/>
        </Dashboard>
    </Modules>
</pp>

## Решение для настольного приложения


В настольном приложении доступность создания блоков, предназначенных
 для работы с отчетами, задается системном реестре в разделе [HKEY_CURRENT_USER\SOFTWARE\Foresight\Foresight Analytics Platform\10.0\Dashboard]. Для доступности
 создания блока добавьте в раздел реестра параметр типа REG_DWORD со значением
 «1». Название параметра соответствует виду отчета:


	- ExpressAnalyze. Экспресс-отчет;


	- Workbook. Временные
	 ряды.


Для исключения блока из списка доступных для создания присвойте соответствующему
 параметру значение «0».


Примечание.
 В настольном приложении для предоставления возможности создания блоков,
 предназначенных для работы с отчетами, пользователь должен иметь права
 на редактирование системного реестра.


Порядок действий для предоставления возможности создания всех возможных
 блоков, предназначенных для работы с отчетами:


	- Откройте реестр для редактирования.


	- Перейдите в ветку реестра [HKEY_CURRENT_USER\SOFTWARE\Foresight\Foresight Analytics Platform\10.0\Dashboard].


	- Найдите параметр ExpressAnalyze
	 типа REG_DWORD. Если параметр отсутствует, то создайте его.


	- Присвойте параметру ExpressAnalyze
	 значение 1.


	- Повторите третий и четвертый шаги для параметра Workbook.


	- Закройте реестр, сохранив изменения.
