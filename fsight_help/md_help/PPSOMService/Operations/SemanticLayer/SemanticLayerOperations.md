# Работа с моделями данных


В данном разделе представлено описание операций, предназначенных для
 работы с объектом репозитория - Модель данных.


## Операции


Общие операции, используемые для открытия и работы с моделью данных:


		 Операция
		 Краткое описание


		 ![](../../Sub_Image.gif)
		 [CloseSemanticLayer](CloseSemanticLayer.htm)
		 Операция CloseSemanticLayer
		 закрывает экземпляр модели данных.


		 ![](../../Sub_Image.gif)
		 [GetSemanticLayerMeta](GetSemanticLayerMeta.htm)
		 Операция GetSemanticLayerMeta
		 получает метаданные модели данных.


		 ![](../../Sub_Image.gif)
		 [OpenSemanticLayer](OpenSemanticLayer.htm)
		 Операция OpenSemanticLayer
		 открывает модель данных.


		 ![](../../Sub_Image.gif)
		 [SetSemanticLayerMeta](SetSemanticLayerMeta.htm)
		 Операция SetSemanticLayerMeta
		 изменяет метаданные модели данных.


Операции для работы с преднастроенными шаблонами подключения к СУБД:


		 Операция
		 Краткое описание


		 ![](../../Sub_Image.gif)
		 [SemanticLayerCheckPattern](SemanticLayerCheckPattern.htm)
		 Операция SemanticLayerCheckPattern
		 проверяет подключение к СУБД, настройки которой хранятся в преднастроенном
		 шаблоне.


		 ![](../../Sub_Image.gif)
		 [SemanticLayerDeletePattern](SemanticLayerDeletePattern.htm)
		 Операция SemanticLayerDeletePattern
		 удаляет преднастроенный шаблон подключения к СУБД.


		 ![](../../Sub_Image.gif)
		 [SemanticLayerGetPatterns](SemanticLayerGetPatterns.htm)
		 Операция SemanticLayerGetPatterns
		 получает список преднастроенных шаблонов подключения к СУБД.


		 ![](../../Sub_Image.gif)
		 [SemanticLayerSetNamePattern](SemanticLayerSetNamePattern.htm)
		 Операция SemanticLayerSetNamePattern
		 переименовывает преднастроенный шаблон подключения к СУБД.


		 ![](../../Sub_Image.gif)
		 [SemanticLayerSetPattern](SemanticLayerSetPattern.htm)
		 Операция SemanticLayerSetPattern
		 создаёт/изменяет преднастроенный шаблон подключения к СУБД.
