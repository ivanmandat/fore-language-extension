# AutoCubeUpdateOption


## Описание


Перечисление AutoCubeUpdateOption
 содержит способы обновления данных.


Используется следующими свойствами и методами:


	- [IAutoCubeInstance.UpdateFromSource](../Interface/IAutoCubeInstance/IAutoCubeInstance.UpdateFromSource.htm);


	- [ICubeLoaderInstance.Execute](../Interface/ICubeLoaderInstance/ICubeLoaderInstance.Execute.htm).


## Допустимые значения


		 Значение
		 Краткое описание


		 0
		 None. Осуществляется
		 синхронизация данных источника и приёмника, в соответствии с передаваемой
		 отметкой.


		 1
		 NoCheckDuplicates.
		 Проверка на наличие дубликатов не осуществляется. Предполагается
		 наличие пустого приёмника данных, если попадается существующая
		 запись, то генерируется исключительная ситуация. Данный режим
		 ускоряет сохранение данных путем уменьшения запросов к серверу
		 БД.


		 2
		 All. Передаваемая
		 отметка не учитывается. Обновляются данные по всем координатам.


		 4
		 NoDelete.
		 Перед обновлением данных очистка приёмника, в соответствии с передаваемой
		 отметкой, не осуществляется. Если в источнике по каким-либо координатам
		 отсутствуют данные, то в приёмнике по данным координатам будут
		 оставлены уже существующие значения.


		 8
		 GetResult.
		 Формируется отчет о результатах обновления данных.


		 16
		 WithNulls.
		 В приёмнике будут установлены пустые значения только по тем координатам,
		 для которых в источнике есть пустые записи.


		 64
		 UseTempTables. Используются
		 временные таблицы. Сначала все данные вставляются во временную
		 таблицу, затем на сервере СУБД осуществляется проверка и перенос
		 недостающих данных в основной таблице.
