# IDalCommandTask


Сборка: Dal;


## Описание


Интерфейс IDalCommandTask содержит
 свойства задачи, предназначенной для работы с базой данных с помощью SQL-запросов
 в асинхронном режиме.


## Иерархия наследования


IDalCommandTask


## Комментарии


Данная задача предназначена для асинхронного выполнения SQL-запросов
 к базе данных. Асинхронное выполнение позволяет, не дожидаясь результата
 выполнения SQL-запроса, продолжить работу в приложении. Для работы с задачей
 необходимо:


	- Создать команду, описываемую интерфейсом [IDalCommand](../IDalCommand/IDalCommand.htm)
	 и настроить ее параметры.


	- Привести полученную команду к интерфейсу [IDalCommand2](../IDalCommand2/IDalCommand2.htm).


	- Выполнить метод [CreateTask](../IDalCommand2/IDalCommand2.CreateTask.htm)
	 для создания задачи.


	- Полученную задачу привести к интерфейсу [ITask](ForeSys.chm::/Interface/ITask/ITask.htm).
	 Используя свойства и методы данного интерфейса можно управлять ходом
	 выполнения задачи.


	- После выполнения задачи результирующие значения будут доступны
	 в свойстве [Result](IDalCommandTask.Result.htm).


Асинхронное выполнение SQL-запросов к базе данных допустимо при соблюдении
 следующих требований:


	- Сервер базы данных создан на базе СУБД Oracle, либо MSSQL.


	- В SQL-запросе указана только одна команда. Для выполнения блока
	 команд используйте хранимую процедуру либо создайте [очередь
	 задач](ForeSys.chm::/Interface/ITask/ITask.ContinueWith.htm).


	- Для асинхронного выполнения хранимых процедур, созданных на
	 сервере MSSQL, в начале текста процедуры после строки «AS BEGIN» должна
	 быть указана строка «SET NOCOUNT ON».


Примечание.
 В рамках одного соединения с базой данных асинхронно может выполняться
 только одна задача, выполняющая SQL-запросы. Для выполнения нескольких
 задач необходимо создавать отдельные соединения, либо организовать [очередь
 задач](ForeSys.chm::/Interface/ITask/ITask.ContinueWith.htm).
