## A


ASF


Advanced Systems Format (ранее Advanced Streaming Format, Active Streaming
 Format) - разработанный фирмой Microsoft формат файлов, содержащих потоковое
 аудио и видео. ASF является частью Windows Media.


ASMX-файл


Адресуемая точка входа веб-службы, создаваемая в управляемом коде. Тип
 получаемого при обращении к этому файлу ответа определяется порядком обращения
 к этому файлу по протоколу HTTP.


## B


BI


Инструменты, используемые для преобразования, хранения, анализа, моделирования,
 доставки и трассировки информации в ходе работы над задачами, связанными
 с принятием решений на основе фактических данных.


BI-поиск


Полнотекстовый поиск данных по ключевым словам в структуре многомерных
 источников репозитория. В «Форсайт. Аналитическая платформа»
 BI-поиск реализован на базе поисковой платформы Apache Solr.


## C


Census2


Метод сезонной декомпозиции и корректировки.


## D


DOM


Document Object Model (объектная модель документа) - это не зависящий
 от платформы и языка программный интерфейс, позволяющий программам и скриптам
 получить доступ к содержимому HTML, XHTML и XML-документов, а также изменять
 содержимое, структуру и оформление таких документов.


## E


ECMAScript


Встраиваемый расширяемый не имеющий средств ввода/вывода язык программирования,
 используемый в качестве основы для построения других скриптовых языков.
 Расширения языка, JavaScript, JScript и ActionScript, широко используются
 в вебе.


ETL


Extract, Transform, Load (дословно «извлечение, преобразование, загрузка»)
 - один из основных процессов в управлении хранилищами данных.


## G


GAC


Global Assembly Cache (глобальный кеш сборок) - это кеш команд компьютера,
 предоставляемый структурой .NET Framework и используемый для хранения
 сборок, к которым требуется предоставить общий доступ для нескольких приложений
 на компьютере. Более подробную информацию о глобальном кеше сборок можно
 найти в дополнительной документации: Справка Windows, MSDN.


GDI


Graphics Device Interface - это интерфейс Windows для представления
 графических объектов и передачи их на устройства отображения, такие как
 мониторы и принтеры. GDI отвечает за отрисовку линий и кривых, отображение
 шрифтов и обработку палитры. В GDI нет растеризации для отрисовки 3D-графики.


GDI+


После выхода Windows XP использование GDI стало нежелательным из-за
 выхода в свет его потомка: подсистемы GDI+, являющейся улучшенной средой
 для 2D-графики. В GDI+ добавлены такие возможности, как сглаживание линий,
 использование координат с плавающей точкой, градиентная заливка, внутренняя
 поддержка таких графических форматов, как JPEG и PNG, преобразования двумерных
 матриц и т. п. GDI+ использует ARGB-цвета. Эти возможности используются
 в пользовательском интерфейсе Windows XP, а их присутствие в базовом графическом
 слое облегчает использование систем векторной графики, таких, как SVG.


GUID


Статистически уникальный 128-битный идентификатор.


## I


IntelliSense


Специальная технология, позволяющая анализировать код, отображая определённые
 типы и члены типов, доступные для использования в текущем месте кода.


## J


JSON-представление


JavaScript Object Notation - текстовый формат обмена данными, основанный
 на JavaScript и обычно используемый именно с этим языком.


## L


LRX-фильтр


Метод сглаживания временного ряда, который используется для выделения
 длительных тенденций временного ряда и является более общим случаем фильтра
 Ходрика-Прескотта.


## M


MIME


Multipurpose Internet Mail Extensions - стандарт, описывающий передачу
 различных типов данных по электронной почте, а также спецификация для
 кодирования информации и форматирования сообщений таким образом, чтобы
 их можно было пересылать по Интернету.


## O


OLAP


Online analytical processing, аналитическая обработка в реальном времени
 - технология обработки информации, включающая составление и динамическую
 публикацию отчётов и документов. Используется для быстрой обработки сложных
 запросов к базе данных и подготовки бизнес-отчётов.


OLE DB


Object Linking and Embedding, Database - набор интерфейсов, основанных
 на объектной модели компонентов, которые позволяют обращаться к данным,
 хранимым в разных источниках информации или хранилищах данных с помощью
 унифицированного доступа.


## R


Reg-free
 COM registration


Технология, позволяющая устанавливать «Форсайт. Аналитическая платформа»
 без регистрации в реестре классов объектов и библиотек типов.


RIA


Rich Internet application «богатое Интернет-приложение» - это приложение,
 доступное через Интернет, богатое функциональностью традиционных настольных
 приложений, не поддерживаемой браузерами непосредственно.


## V


Value-At-Risk
 (VaR)


Стоимостная мера риска. Распространено общепринятое во всём мире обозначение
 «VaR». Это выраженная в денежных единицах оценка величины, которую не
 превысят ожидаемые в течение данного периода времени потери с заданной
 вероятностью.


## W


Windows
 Presentation Foundation


WPF (кодовое название - Avalon) - графическая (презентационная) подсистема
 в составе .NET Framework (начиная с версии 3.0), имеющая прямое отношение
 к XAML.


## X


XAML


eXtensible Application Markup Language - расширяемый язык разметки приложений,
 основанный на XML язык разметки для декларативного программирования приложений,
 разработанный Microsoft.


XML


eXtensible Markup Language - расширяемый язык разметки. Рекомендованный
 Консорциумом Всемирной паутины язык разметки, фактически представляющий
 собой свод общих синтаксических правил.


## А


Автокореляция k-ого
 порядка от временного ряда X


Мера тесноты и направления линейной стохастической зависимости между
 текущими значениями временного ряда и значениями временного ряда на k
 моментов времени назад.


Автоматический куб


Куб, построенный по принципу сверху вниз.


Автоматическое обновление


Режим работы с экспресс-отчётом, при включении которого при каждом изменении
 набора данных производится автоматическое обновление данных на сервере.


Автообновление


Обновление версии продукта «Форсайт. Аналитическая платформа»
 при входе в репозиторий.


Автоподгонка размера
 ячеек


Установка размеров ячейки таким образом, чтобы её содержимое отображалось
 наилучшим образом.


Авторегрессия


Статистический метод, в котором применяются предыдущие значения для
 предсказания будущих.


Авторизация


Регистрация пользователей в системе.


Авторизация БД


Проверка возможности подключения пользователя к базе данных.


Агрегация данных


Расчёт данных верхнего уровня на основе данных нижнего уровня.


Активная область
 листа


Область, которая содержит введенные данные.


Анализ 80/20


Функция, позволяющая отобразить минимальный набор значений, которые
 в сумме составляют 80% от общей суммы всех значений выбранного столбца,
 остальные значения (20%) отображаются общей суммой в строке внутри своего
 родителя.


Анализ временных
 рядов


Инструмент в составе «Форсайт. Аналитическая платформа»,
 предназначенный для выявления структуры временных рядов и для их прогнозирования
 с помощью математико-статистических методов анализа.


Аналитические
 запросы (OLAP)


Инструмент в составе «Форсайт. Аналитическая платформа»,
 предназначенный для формирования нерегламентированных запросов к данным
 и проведения экспресс-анализа с помощью средств деловой графики и с применением
 различных аналитических функций.


Аналитические панели


Инструмент в составе «Форсайт. Аналитическая платформа»,
 предназначенный для формирования аналитических документов в виде комбинации
 интерактивных взаимосвязанных панелей. Панели отображают данные из различных
 источников с использованием средств деловой графики, изображений и элементов
 управления.


Атрибут справочника


Характеристика, которую имеет каждый элемент справочника.


Атрибут
 справочника НСИ


Характеристика, которую имеет каждый элемент справочника НСИ.


Атрибуты наблюдения


Набор характеристик, которые есть у каждого наблюдения (точки) показателя.


Атрибуты показателей


Набор характеристик, совокупность значений которых однозначно идентифицирует
 показатель.


Атрибуты ряда


Набор характеристик, которые есть у каждого показателя.


Аудит


Запись действий субъектов в протокол доступа.


Аутентификация


Проверка принадлежности пользователю указанного им пароля.


## Б


База данных (БД)


Объект репозитория, содержащий основные настройки для подключения к
 существующей базе данных, созданной на сервере СУБД.


База данных
 временных рядов (БДВР)


Объект репозитория, предназначенный для управления и анализа статистических
 данных, хранящихся в виде показателей.


Безоконные
 компоненты


Компоненты, которое не могут получить фокус, не могут содержать другие
 элементы управления, и не имеют собственное окно. К безоконным компонентам
 относятся Label, Bevel
 и ImageBox.


Блок справочника


Основной структурный элемент справочника, представляющий собой группу
 элементов, созданных на основе данных, полученных из одного источника
 данных.


Блокировка объекта


Состояние объекта, когда он заблокирован для редактирования другим пользователем.


## В


Варианты отображения
 куба


Набор вариантов агрегации одного куба. Предназначены для выбора варианта
 агрегации в отчётах.


Векторный
 режим расчёта задачи


Режим расчёта, при котором модели в задаче рассчитываются последовательно
 по всем точкам сценария: сначала рассчитывается первая модель по всем
 точкам сценария, затем происходит расчёт второй модели по всем точкам
 сценария и т.д. Для расчёта задачи в векторном режиме предназначена задача
 трансформации: интерфейс IMsTransformationProblem.


Версионный
 справочник НСИ


Справочник НСИ, элементы которого могут изменяться во времени, т.е.
 для каждого элемента указан диапазон действия значения - дата начала действия
 и дата окончания действия.


Веса
 главных компонент


Значения собственных векторов.


Визуализатор


Объект для визуального представления данных. Например: карта, диаграмма,
 таблица.


Визуальный компонент


Компонент, отображаемый как на стадии проектирования, так и при выполнении
 формы.


Виртуальный куб


Куб, который строится на основе данных других кубов. Измерение фактов
 куба формируется на основе отмеченных элементов фиксированных измерений.
 При просмотре данных виртуального куба в экспресс-отчёте на панели измерений
 будут отображены незафиксированные общие измерения кубов-источников и
 измерение фактов виртуального куба.


Временной лаг


Показатель, отражающий отставание или опережение во времени одного явления
 по сравнению с другим, связанным с ним.


Выражение управления


Выражение на основе атрибутов управляющего измерения, по которому будет
 рассчитываться значение, передаваемое в качестве значения параметра управляемого
 измерения.


Вычисление отчёта


Действия, производимые с отчётом, которые включают в себя обращение
 к базе данных, обновление данных на листах отчёта, вычисление формул.


Вычисляемые показатели


Показатели, вычисляемые по заданной пользователем функции.


Вычисляемый куб


Данный куб позволяет, используя заданные формулы, вычислять значения
 фактов на основе значений фактов других кубов.


Вычисляемый ряд


Ряд, рассчитанный с помощью выбранного метода на основе данных других
 рядов.


Вычисляемый справочник


Справочник, наполнение данных которого осуществляется по алгоритму пользователя,
 реализованного в виде процедуры на внутреннем языке программирования.


Вычисляемый факт куба


Факт куба, вычисляемый по заданной пользователем функции.


## Г


Глобальные переменные


Переменные, значения которых доступны во всех объектах платформы в рамках
 текущего запущенного приложения «Форсайт. Аналитическая платформа».


Графические примитивы


Геометрические фигуры: прямоугольник, эллипс, скругленный прямоугольник,
 правильные многоугольники, фигурные стрелки.


Группа пользователей


Субъекты безопасности системы, объединённые под одним учетным именем
 с целью унифицированного управления правами доступа для них.


Группа элементов


Объект репозитория, используемый в справочниках для ограничения списка
 доступных элементов.


## Д


Дедупликация


От лат. deduplicatio - устранение дубликатов. Дедупликация - это специализированный
 метод сжатия массива данных, который использует для сжатия данных исключение
 дублирующих копий повторяющихся данных.


Диапазон данных


Подмножество данных базы данных временных рядов, которое определяется
 путём задания отметки для измерения или календаря базы данных временных
 рядов.


Дискреционный
 метод доступа


Метод разграничения доступа. Основан на использовании списков контроля
 доступа, назначаемых каждому объекту системы. Для каждого объекта определяются
 права.


Дистрибутив/Инсталлятор


Форма распространения программного обеспечения. «Форсайт. Аналитическая платформа»
 распространяется в виде EXE-файлов.


Доменная авторизация


Авторизация требует указания в явном виде домена, имени и пароля пользователя.


## Е


Единица измерения


Численное значение физической величины, используемое для стандартизованного
 представления результатов измерений.


Единица лицензирования


Термин используется, чтобы идентифицировать инструмент или функциональность
 продукта «Форсайт. Аналитическая платформа»,
 которые лицензируются.


Единичная отметка


Возможность отметить только один элемент списка.


## Ж


Журнал


Объект репозитория, предназначенный для протоколирования операций, производимых
 при выполнении задачи ETL.


Журнал операций


Файл, где ведется запись произведенных операций планировщиком задач.


## З


Загрузчик в куб


Объект репозитория, предназначенный для заполнения куба данными из другого
 куба.


Задача ETL


Объект репозитория, предназначенный для извлечения, преобразования и
 загрузки данных.


Задача моделирования


Объект контейнера моделирования, осуществляющий расчёт моделей на основе
 последовательности, заданной метамоделью. Предназначена для получения
 и выгрузки результатов расчёта в заданные переменные.


Запрос


Объект репозитория, представляющий выборку из таблиц на основе заданного
 условия. Запрос может являться источником данных для измерений и кубов.


## И


Идентификатор


Уникальная последовательность букв латинского алфавита, цифр и знаков
 подчёркивания, позволяющая однозначно идентифицировать объект среди подобных.
 Идентификатор должен начинаться с буквы или знака подчёркивания.


В «Форсайт. Аналитическая платформа»
 идентификаторы применяются для всех объектов репозитория и структурных
 элементов метаданных: атрибутов измерений и справочников, индексов, схем
 отметок и прочих.


Измерение


Справочник, участвующий в построении многомерного куба.


Имперсонация


Выполнение действий под одним пользователем от имени другого пользователя.
 В «Форсайт. Аналитическая платформа»
 используется при работе BI-сервисов для авторизации пользователей с помощью
 учетных записей различных социальных систем: LiveJournal, Google+.


Интегрированная доменная авторизация


Авторизация, при которой используются те же имя пользователя и пароль,
 с которыми пользователь подключился к домену.


Интеллектуальный анализ данных


Процесс выявления скрытых фактов и взаимосвязей в больших массивах данных.
 Полученные данные могут использоваться для принятия решений в различных

Медленно изменяющиеся размерности


Существуют два архитектурных направления: нормализованные хранилища
 данных и размерностные хранилища. В нормализованных хранилищах данные
 находятся в предметно-ориентированных таблицах третьей нормальной формы:
 витринах данных. Нормализованные хранилища являются простыми в создании
 и управлении. Недостатки нормализованных хранилищ - высокая избыточность
 информации, проблемы с интегрированностью данных из нескольких таблиц
 одновременно.


Размерностные хранилища используют схему «звезда» или «снежинка». В
 центре звезды находятся данные (таблица фактов), а размерности образуют
 лучи звезды. Различные таблицы фактов совместно используют таблицы размерностей,
 что значительно облегчает операции объединения данных из нескольких предметных
 таблиц фактов (пример - факты продаж и поставок товара). Таблицы данных
 и соответствующие размерности образуют архитектуру «шина». Для протоколирования
 изменения в размерностях они часто создаются в третьей нормальной форме
 (медленно изменяющиеся размерности). Основным достоинством размерностных
 хранилищ являются простота и понятность для разработчиков и пользователей.
 Благодаря более эффективному хранению данных и формализованным размерностям
 облегчается и ускоряется доступ к данным, особенно при сложных анализах.
 Основным недостатком является более сложные процедуры подготовки и загрузки
 данных, а также, управление и изменение размерностей данных.

 строк/столбцов указываются в квадратных скобках и обозначают смещений
 относительно текущей ячейки с формулой. Пример: R0C0, R1,C[-1], R[-2]C[-3].


Стратегия фильтрации


Набор параметров, в соответствии с которым будут формироваться запросы
 для извлечения данных при построении куба.


Строка формул


Поле в верхней части окна регламентного отчёта, предназначенное для
 ввода или изменения значений/формул в ячейках.


СУБД


Система управления базами данных.


Субъекты безопасности/системы


Пользователи или группы пользователей.


Схема


Название базы данных, к которой будет осуществляться подключение.


Схема импорта справочника НСИ


Набор параметров, в соответствии с которыми производится загрузка данных
 из таблицы в справочник НСИ.


Схема отметки


Объект репозитория, используемый в справочниках для автоматизации выделения
 элементов по заданным правилам.


Схема экспорта справочника НСИ


Набор параметров, в соответствии с которыми производится выгрузка данных
 из справочника НСИ в  таблицу.


Сценарии моделирования


Объект, предназначенный для создания в моделируемой переменной ряда,
 в который будут выгружены данные после окончания расчёта задачи моделирования.


Сценарий моделирования


Совокупность исходных данных и предположений, используемых при моделировании.
 Сценарий также предназначен для создания в моделируемой переменной
 ряда, в который будут выгружены данные после окончания расчёта задачи
 моделирования.


## Т


Таблица


Объект репозитория, который предоставляет доступ к физической таблице
 на сервере.


Таблица стилей


Набор стилей ячеек.


Таблица формул


Таблица, в которой хранятся формулы многомерного расчёта на сервере
 БД. Каждая формула представлена в виде отдельной записи в таблицы путём
 занесения операндов в необходимые поля таблицы. При расчёте на сервере
 значения полей будут использованы в выполняемой процедуре.


Табличный справочник


Справочник, основанный на таблице данных определённого формата, каждая
 запись которой представляет собой элемент справочника.


Терм


Выражение формального языка, которое является формальным именем объекта.


Тип атрибута


Пользовательский тип данных, предназначенный для классификации атрибутов
 по общим признакам.


Топооснова


Набор графических слоёв информации, наложенных друг на друга.


## У


Управляемое измерение


Измерение, значения параметров которого меняются динамически при изменении
 отметки управляющего измерения.


Управляющее измерение


Измерение, значения атрибутов которого используется для управления параметрами
 другого измерения. Управление осуществляется путём расчёта выражения управления
 и передачи полученного значения в качестве значения параметра управляемого
 измерения.


Учетные данные (УД)


Имя пользователя и пароль пользователя, используемые при подключении
 к серверу.


## Ф


Факторы


Входные переменные, влияющие на моделируемую переменную.


Факты куба


Внутренний справочник куба, который формирует одно из измерений куба.
 Если факт один - это случай вырожденного справочника.


Фантом


Полупрозрачное изображение с выделенной областью, следующее за курсором
 при перетаскивании объектов.


Фасетный поиск


Вид BI-поиска, при котором результаты поиска группируются по значениям
 определённого поля - фасетного поля. Данный вид поиска реализуется с помощью
 [макросов
 на Fore](KeBISearch.chm::/Interface/ISolrSearchResults/ISolrSearchResults.FacetSearchResults.htm). Фасетное поле может представлять из себя какую-либо
 категорию, по которой группируются данные. Для фасетного поиска могут
 быть созданы несколько фасетных полей.


Фиксированное измерение


Свёрнутое измерение многомерного куба.


Фильтр Бакстера-Кинга


Метод сглаживания временного ряда, который является модификацией фильтра
 Ходрика-Прескотта с более широкими возможностями исключения циклической
 составляющей во временном ряде.


Фильтр Ходрика-Прескотта


Метод сглаживания временного ряда, который используется для выделения
 длительных тенденций временного ряда.


Форма


Объект для создания интерфейса пользовательского приложения. Форма является
 контейнером для любых компонентов, доступных в среде разработки. Каждой
 форме также соответствует модуль, содержащий описание класса формы и её
 исполняемый код. Код должен быть написан на языке Fore.


Форсайт. Аналитическая платформа


Единая интегрированная платформа для создания BI-приложений.


Функциональность


Единица лицензирования на отдельные блоки или функции продукта «Форсайт. Аналитическая платформа».


## Х


Хранилище данных (ХД)


Массив данных организации, структурированный специальным образом, обеспечивающий
 быстрый доступ к информации, многомерный анализ данных, получение прогнозов
 и различных статистик в измерениях согласованной нормативно-справочной
 информации


## Ч


Частное измерение


Уникальное измерение источника данных мульти-OLAP'а.


## Э


Экзогенные переменные


Независимые переменные, которые определяются вне модели (системы).


Экспресс-отчёт


Отчёт, созданный в инструменте «[Аналитические запросы (OLAP)](#analytical_queries)».


Электронная таблица


Электронная матрица, разделённая на строки и столбцы, на пересечении
 которых образуются ячейки с уникальными именами. Ячейки являются основным
 элементом электронной таблицы, в которые могут вводиться данные и на которые
 можно ссылаться по именам ячеек. К данным относятся: числа, даты, время
 суток, текст или символьные данные и формулы.


Эндогенные переменные


Взаимозависимые переменные, которые определяются внутри модели (системы).
 Как правило, каждое уравнение модели определяет одну эндогенную переменную,
 стоящую в левой части уравнения.


## Ю


Юзабилити


Итоговый уровень удобности предмета для использования в заявленных целях.
 Международный стандарт ISO 9241-11 определяет юзабилити как «степень,
 с которой продукт может быть использован определёнными пользователями
 при определённом контексте использования для достижения определённых целей
 с должной эффективностью, продуктивностью и удовлетворённостью».


Юзабилити-тестирование


Эксперимент, выполняемый с целью определения, насколько хорошо люди
 могут использовать некоторый искусственный объект (такой как веб-страница,
 пользовательский интерфейс или устройство) для его предполагаемого применения,
 то есть юзабилити-тестирование измеряет юзабилити объекта.


## Я


Ячейка


Область, определяемая пересечением столбца и строки электронной таблицы,
 имеющая свой уникальный адрес.
