# Работа с буфером обмена


«Форсайт. Аналитическая платформа»
 поддерживает использование буфера обмена.


[![](../Opened.gif)![](../Closed.gif)Копирование
 объекта в буфер обмена](javascript:TextPopup(this))


	Для копирования объекта в буфер обмена:


		- в веб-приложении:


			- выполните команду ![](b_copy_web.png) «Копировать» в контекстном
			 меню объекта;


			- нажмите сочетание клавиш CTRL+С;


		- в настольном приложении:


			- нажмите кнопку ![](Copy_Button.png) «Копировать»,
			 расположенную в группе «Буфер
			 обмена» вкладки «Главная»
			 ленты инструментов в настольном приложении;


			- выполните команду «Копировать»
			 в контекстном меню объекта;


			- нажмите сочетание клавиш CTRL+С или CTRL+INSERT.


	Примечание.
	 При копировании объекта права на объект не копируются.


	Для копирования [алгоритмов
	 расчёта](calculationalgorithm.chm::/Purpose.htm)
	 пользователь должен обладать привилегией «[Доступ
	 к менеджеру обновлений](Admin.chm::/04_SecurityPolicy/Admin_Priv.htm#upd_manager)».


	Для объектов, которые представляют собой ссылки на таблицы с данными
	 (представления, справочники НСИ, контейнеры моделирования, каталог
	 показателей) возможно копирование только метаданных или метаданных
	 и данных одновременно:


		- для копирования метаданных выполните команду ![](b_copy_web.png) «Копировать >
		 Без данных» в контекстном меню объекта;


		- для копирования метаданных и данных выполните команду ![](b_copy_web.png) «Копировать > С данными»
		 в контекстном меню объекта.


	Примечание.
	 При копировании [ярлыка](#shortcut) на любой [справочник НСИ](UiNavObj.chm::/reference_book/Master_RDS_reference_book/Master_RDS.htm)
	 создаётся пустой объект.


[![](../Opened.gif)![](../Closed.gif)Перемещение
 объекта в буфер обмена или объект-контейнер](javascript:TextPopup(this))


	Для перемещения объекта в буфер обмена:


		- в веб-приложении:


			- выполните команду ![](Cut_Button_web.png) «Вырезать» в контекстном
			 меню объекта;


			- нажмите сочетание клавиш CTRL+X;


		- в настольном приложении:


			- нажмите кнопку ![](Cut_Button.png) «Вырезать»,
			 расположенную в группе «Буфер
			 обмена» вкладки «Главная»
			 ленты инструментов;


			- выполните команду «Вырезать»
			 в контекстном меню объекта;


			- нажмите сочетание клавиш CTRL+X или SHIFT+DELETE.


	После выполнения одного из действий объект будет перемещён в буфер
	 обмена.


	Для перемещения объекта в объект-контейнер используйте механизм
	 Drag&Drop.


[![](../Opened.gif)![](../Closed.gif)Вставка
 объекта из буфера обмена](javascript:TextPopup(this))


	Для вставки объекта из буфера обмена:


		- в веб-приложении:


			- выполните команду ![](b_paste_web.png) «Вставить» в контекстном
			 меню навигатора объектов;


			- нажмите сочетание клавиш CTRL+V;


		- в настольном приложении:


			- нажмите кнопку ![](Insert_Button.png) «Вставить», расположенную в
			 группе «Буфер обмена»
			 вкладки «Главная»
			 ленты инструментов;


			- выполните команду «Вставить»
			 в контекстном меню навигатора объектов;


			- нажмите сочетание клавиш CTRL+V или SHIFT+INSERT.


	После выполнения одного из действий при копировании объекта в настольном
	 приложении будет открыто окно «Вставка
	 объекта - параметры». Задайте в нём, если требуется, новое
	 наименование и новый уникальный идентификатор объекта. Идентификатор
	 может состоять только из букв латинского алфавита, цифр и символа
	 подчеркивания. В веб-приложении объекту автоматически присваивается
	 новый идентификатор, наименование не меняется.


[![](../Opened.gif)![](../Closed.gif)Создание
 ярлыка на объект репозитория](javascript:TextPopup(this))


	Примечание.
	 Доступно только в настольном приложении.


	Для создания ярлыка на объект:


		- Скопируйте необходимый объект в буфер обмена.


		- Выполните команду контекстного меню «Вставить
		 ярлык».


		- Задайте, если требуется, новое наименование и новый уникальный
		 идентификатор объекта. Идентификатор может состоять только из
		 букв латинского алфавита, цифр и символа подчеркивания.


	В навигаторе объектов будет создан объект, данные для которого берутся
	 из копируемого объекта.


Примечание.
 Некоторые команды могут быть недоступны из-за отсутствия соответствующих
 прав доступа.
