# Особенности ввода и вставки значений в таблицу


## Особенности ввода значений в ячейку


При вводе значений в ячейку учитывайте следующие особенности:


	- значение будет распознано как текст, если оно содержит пробел,
	 расположенный менее чем за три символа до конца. В ином случае значение
	 будет распознано как число. Например:


	-


		- введено значение «178 7». Оно будет распознано как
		 текст и к нему будет применено форматирование, предусмотренное
		 для текста;


		- введено значение «178 749». Оно будет распознано как
		 число и к нему будет применено форматирование, предусмотренное
		 для чисел. Например: «178 749,00»;


	- если для ячейки задан формат «Дата», то пробел не будет распознан
	 как разделитель. В качестве разделителя используйте символ, заданный
	 в настройках языка и региональных стандартов для операционной системы.
	 Например:


	-


		- введено значение «05 07 2013». Оно не будет распознано,
		 возникнет ошибка ввода данных;


		- введено значение «05.07.2013». Оно будет распознано как
		 дата и к нему будет применено форматирование, предусмотренное
		 для дат. Например: «5 июля 2013г»;


	- если в ячейке задано числовое значение, то изменить настройки
	 [шрифта](Format/UiReport_Table_Attribute_Type.htm) для
	 отдельной цифры нельзя. Настроить шрифт можно только для всей ячейки.


## Особенности вставки в таблицу скопированного диапазона


При вставке в таблицу скопированного диапазона в выделенный диапазон
 учитывайте следующие особенности:


	- если выделенный диапазон больше
	 скопированного диапазона, то:


		- если был скопирован диапазон N
		 ячеек и вставка идет в диапазон кратный N,
		 то скопированный диапазон будет вставлен столько раз, сколько
		 позволяет выделенный диапазон. Например, скопирован диапазон из
		 двух ячеек и вставляем его в диапазон из восьми ячеек, то диапазон
		 из двух ячеек будет вставлен четыре раза;


		- если был скопирован диапазон из N
		 ячеек и вставка идет в диапазон M,
		 который не кратен N,
		 то скопированный диапазон будет вставлен X
		 раз, где X - это
		 результат целочисленного деления M
		 на N. Например, скопирован
		 диапазон из трех ячеек и вставляем его в диапазон из восьми ячеек,
		 то диапазон из трех ячеек будут вставлен два раза;


	- если выделенный диапазон меньше
	 скопированного диапазона, то выделенный диапазон будет автоматически
	 расширен до размеров скопированного диапазона;


	- если скопированный диапазон
	 больше, чем размер таблицы, в которую выполняется вставка,
	 то будет вставлено столько значений, сколько позволяет таблица.


Примечание.
 Если в выделенном диапазоне находились данные, то они будут заменены.


## Особенности ввода и вставки значений в таблицу


При вводе и вставке значений в таблицу учитывайте следующие особенности:


	- если в ячейке таблицы для значения был задан [формат
	 числа](Format/UiReport_Table_Attribute_Format.htm) «# ##0,00», то все данные отображаются в виде чисел с двумя
	 знаками после запятой;


	- если значение содержит один разделитель в виде точки, то число
	 автоматически переводится в формат даты;


	- если значение содержит дату и в ячейке задается числовой формат
	 «# ##0,00», то дата представляется в виде числа;


	- если значение содержит более двух знаков после запятой и разделитель
	 в виде точки, то число представляется в виде строки;


	- если значение содержит несколько знаков после запятой, то число
	 округляется в зависимости от заданного формата числа.
