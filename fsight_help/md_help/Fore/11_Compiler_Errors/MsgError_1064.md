# После <спецификаторДоступа> ожидается идентификатор поля или
 метода


## Описание


При использовании старого синтаксиса описания свойства после спецификатора
 доступа указан идентификатор объекта, не являющегося полем класса,
 либо процедурой/функцией.


## Способ устранения


Проверьте тип объекта, указанного после соответствующего спецификатора
 доступа.


## Пример


			Class MyClass: Object

    Const i = 100;


    Property Item: Integer Get i;

End Class MyClass;


При компиляции указанного кода на строке «Property Item: Integer Get i;»
 будет сгенерирована ошибка «После get ожидается идентификатор поля или
 метода». Для разрешения ошибки необходимо константу «i» перенести в блок
 переменных и указать для нее целый тип, либо написать новую функцию и
 указать ее идентификатор после спецификатора «Get».
