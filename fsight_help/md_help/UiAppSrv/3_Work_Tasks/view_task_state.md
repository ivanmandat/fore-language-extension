# Просмотр состояния задач


После создания задач будут доступны операции:


	- Просмотр
	 состояния задачи. Состояние задач отображается в рабочей области
	 контейнера в столбце «Состояние».
	 Для получения актуальных данных обновите состояние задач. Возможны
	 следующие состояния:


		- Готова.
		 Задача готова к выполнению;


		- Выполняется.
		 Задача выполняется в данный момент;


		- Выполнена.
		 Задача уже выполнена. В столбце «Результат»
		 будет отображена информация о том, как завершена задача:


			- Завершена с ошибками.
			 Задача выполнена, но в процессе выполнения возникали [ошибки](UiAppSrv_WorkTasks_Errors.htm);


			- Завершена успешно.
			 Задача выполнена успешно;


			- Не активна.
			 Для задачи выключено выполнение расчета в [настройках
			 расписания](UiAppSrv_Work_Tasks_CreateTask_TimeTable.htm);


	- Просмотр
	 результатов отправления задачи. Просмотреть состояние отправки
	 уведомления по электронной почте можно в рабочей области контейнера
	 в столбце «Уведомление». Для
	 получения актуальных данных обновите состояние задач. Возможны следующие
	 состояния:


		- Не
		 отправлено. После выполнения задачи электронное письмо
		 не было отправлено;


Примечание.
 Отправка уведомлений по электронной почте доступна только для задач, выполняемых
 по [расписанию](UiAppSrv_Work_Tasks_CreateTask_TimeTable.htm).
 При запуске задач [вручную](view_task_state.htm#manual_task)
 сообщения не отправляются.


		- Отправлено.
		 После выполнения задачи электронное письмо было отправлено;


		- Неопределенное состояние. Состояние
		 возможно, только если не была [настроена
		 отправка уведомления](UiAppSrv_Work_Tasks_CreateTask_Rezult.htm);


	- Обновление
	 состояния задач. Для обновления состояния задач:


		- нажмите кнопку ![](../2_Work_interface/toolbar%20Update.gif) на панели инструментов;


		- выполните команду «Вид
		 > Обновить» главного меню;


		- нажмите клавишу F5.


После выполнения любого из действий будет
 обновлено состояние задач;


Примечание.
 Обновление состояния задач доступно только в настольном приложении. В
 веб-приложении состояния задач обновляются автоматически.


	- Повторный
	 запуск задачи. Иногда требуется перезапуск созданной задачи.
	 Например, если задача выполнялась однократно, но её выполнение завершилось
	 с ошибкой. Перезапуск может быть произведен для одной или нескольких
	 выбранных задач.


Примечание.
 Повторный запуск задачи доступен только в настольном приложении.


Для перезапуска выбранных задач выполните
 следующие шаги:


		- Сбросьте результаты выполнения
		 задачи. Для этого:


			- выполните команду «Задача
			 > Сбросить результаты выполнения» главного меню;


			- выполните команду «Сбросить
			 результаты выполнения» в контекстном меню задачи.


После выполнения любого из действий будет
 выдано подтверждение выполнения действий;


		- Выполните команду «Выполнить»
		 в контекстном меню задачи. Выбранная задача будет запущена на
		 выполнение.


Примечание.
 Команда недоступна, если состояние задачи «Выполняется».


	- Запуск
	 задач вручную. Для запуска задач в контейнере запланированных
	 задач:


		- в веб-приложении нажмите кнопку ![](b_start.png) «Запустить», расположенную напротив
		 задачи;


		- в настольном приложении выполните команду «Выполнить»
		 в контекстном меню задачи;


	- Сохранение
	 результатов задачи. Для сохранения результатов задачи выполните
	 команду «Сохранить результат
	 как» в контекстном меню задачи.


Примечание.
 Команда доступна для выполненной задачи «Вычисление
 регламентного отчета» только в настольном приложении.


После выполнения данной команды будет открыт стандартный диалог сохранения
 файла. По умолчанию будет предложено сохранить результат в формате *.xls.
 Команда сохраняет результаты для всех выделенных задач.
