# Сохранение изменённых данных


Сохранение изменённых данных в табличном визуализаторе и/или области
 данных в источник будет предложено перед выполнением следующих операций:


	- вычисление/автоматическое вычисление [табличного
	 визуализатора](../../AreaData/DataTable.htm), [области
	 данных](../../AreaData/UiReport_AreaData.htm#areadata_calc), [листа
	 отчёта](UiReport_Reports_Operation_Execute.htm);


	- сохранение отчёта;


	- закрытие отчёта.


При работе с отчётом в [режиме
 просмотра](../../organizational_management/Starting.htm#open):


	- для сохранения изменённых данных нажмите кнопку ![](../../organizational_management/Toolbars/save16.png) «Сохранить
	 изменения» на вкладке «Главная»
	 ленты инструментов. При последующем открытии отчёта будут отображаться
	 новые данные;


	- для отмены изменений нажмите кнопку ![](../../organizational_management/Toolbars/undo16.png) «Отменить изменения» на вкладке «Главная» ленты инструментов.


Примечание.
 Кнопки «Сохранить изменения»
 и «Отменить изменения» доступны,
 если в ячейках таблицы были [введены](../../Table/Work/UiReport_Table_Work_Data.htm)
 или [изменены](../../Table/Work/UiReport_Table_Work_Data_Edit.htm)
 данные.


Сохранение изменённых данных возможно в следующие виды источников: стандартный
 куб, представление-куб, виртуальный куб, база данных временных рядов,
 переменные моделирования.


Примечание.
 Сохранение данных в источник возможно, если пользователь имеет соответствующие
 [права](Admin.chm::/03_Admin/Admin_AdminObjects_Discretionary.htm).


[![](../../../opened.gif)![](../../../closed.gif)Ошибки при
 сохранении данных](javascript:TextPopup(this))


	Если в области данных вместо числа был введен текст или было указано
	 недопустимое значение, то появится информационное сообщение, содержащее
	 текст ошибки. Для разрешения ошибки выполните одно из действий:


		- нажмите кнопку «Продолжить»
		 для продолжения процесса сохранения изменённых данных с пропуском
		 некорректного значения;


		- нажмите кнопку «Прервать»
		 для прерывания процесса сохранения изменённых данных;


		- нажмите кнопку «Подробнее»
		 для просмотра дополнительной информации об ошибке;


		- установите флажок «Пропустить
		 все ошибки» для продолжения процесса сохранения изменённых
		 данных с пропуском всех некорректных данных.


## Создание копии отчёта


Для создания копии отчёта:


	- Выполните команду «Отчет >
	 Сохранить как» в главном меню.


Примечание.
 В веб-приложении выполните команду «Регламентный
 отчет > Сохранить как» в главном меню.


	- В открывшемся окне укажите расположение, наименование и уникальный
	 идентификатор для сохраняемого отчёта. Идентификатор может состоять
	 только из букв латинского алфавита, цифр и знака «_» и не должен начинаться
	 с цифр.


### Особенности создания копии отчёта


	- При создании копии отчёта в репозитории создается новый отчёт,
	 который сразу же открывается в том же режиме, в котором был открыт
	 исходный отчёт.


	- При создании копии отчёта, открытого [в
	 режиме просмотра](../../organizational_management/Starting.htm#open), будет сохранена статическая копия отчёта. Сохраненный
	 таким способом отчёт не будет содержать [источники
	 данных](../../Source/UiReport_Source.htm), [элементы управления](../../Source/Controls.htm)
	 и подключенные [модули](../Event/UiReport_Reports_Event_Module.htm).
