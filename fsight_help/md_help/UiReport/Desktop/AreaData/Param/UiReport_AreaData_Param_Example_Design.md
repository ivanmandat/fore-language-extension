# Пример оформления дополнительных наименований при произвольном размещении


При [произвольном
 размещении](Order/UiReport_AreaData_Param_Order_Tuning.htm) можно задать оформление для заголовка и данных элемента.
 Если для элемента не задано оформление, то будет применено оформление,
 настроенное для уровня, измерения, боковика/шапки.


Рассмотрим область данных, для которой включено произвольное размещение,
 заданы дополнительные наименования элементов измерения, для заголовков
 и данных элементов заданы идентичные оформления:


![](sample%201.gif)


Если для строк/столбцов на вкладке «[Размещение >
 Строки/Столбцы](Order/UiReport_AreaData_Param_Order_RS.htm)» установлен флажок «Размещать
 элементы уровней в отдельные столбцы», то оформление применяется
 следующим образом:


![](sample%202.gif)


Если для строк/столбцов на вкладке «[Размещение >
 Строки/Столбцы](Order/UiReport_AreaData_Param_Order_RS.htm)» установлены флажки «Размещать
 элементы уровней в отдельные столбцы» и «Использовать
 ступенчатую иерархию», то оформление применяется следующим образом:


![](sample%203.gif)


При установке на вкладке «[Размещение
 > Строки/Столбцы](Order/UiReport_AreaData_Param_Order_RS.htm)» флажков «Размещать
 элементы уровней в отдельные столбцы» и «Выводить
 дочерние элементы перед родительскими» оформления к дополнительным
 наименованиям применяется следующим образом:


![](sample%204.gif)


Если в области данных отсутствует произвольное размещение, и на вкладке
 «[Размещение
 > Строки/Столбцы](Order/UiReport_AreaData_Param_Order_RS.htm)» установлен флажок «Размещать
 элементы уровней в отдельные столбцы», то оформление объемлющего
 элемента применяется к соседней объединенной ячейке.
