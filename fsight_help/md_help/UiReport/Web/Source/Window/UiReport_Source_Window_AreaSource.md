# Добавление срезов данных и работа с ними


Многомерные источники данных делятся на срезы данных, используемые для
 построения [визуализаторов](../../Reports/Visualization.htm)
 на листе отчёта. Для работы со срезами данных используется [боковая
 панель](../../organizational_management/Starting.htm). Наименование среза по умолчанию: «Срез
 N», где N - порядковый номер создаваемого среза в пределах одного
 регламентного отчёта.


В качестве источников данных можно использовать:


	- все [виды
	 кубов](UiNavObj.chm::/Cube/UiMd_Cube_Type.htm),
	 включая дочерние кубы контейнера моделирования;


	- [базы
	 данных временных рядов](uidw.chm::/UiDw_Title.htm).


## Работа со срезами


Доступные операции:


[![](../../../opened.gif)![](../../../closed.gif)Добавление
 среза](javascript:TextPopup(this))


	Для добавления нового среза выполните одно из действий:


		- с помощью механизма Drag&Drop:


			- Наведите курсор на нужный
			 источник. Курсор примет вид указателя ![](../../Sheets/cursor.png).


			- Переместите источник в
			 рабочую область. Левая верхняя граница выбранной ячейки будет
			 левой верхней границей таблицы или диаграммы, построенной
			 на основе созданного среза.


			- В меню выбора укажите ![](b_table.png)«Таблица» или ![](../../Diagrams/Type_diagrams/b_bar_chart.png) «Диаграмма>
			 <Тип
			 диаграммы>».
			 К источнику будет автоматически добавлен новый срез, а на
			 лист отчёта будет добавлен выбранный визуализатор.


		- с помощью контекстного меню:


			- Выделите пустую ячейку
			 на листе отчёта. Левая верхняя граница выделенной ячейки будет
			 левой верхней границей таблицы или диаграммы, построенной
			 на основе созданного среза.


			- Выполните одно из действий:


				- выполните команду «Добавить
				 > Срез» в контекстном меню [источника
				 данных](../UiReport_Source.htm) на панели «Источники»;


				- выполните команду «Добавить
				 > Таблицу» или «Добавить
				 > Диаграмму > <Тип
				 диаграммы>»
				 в контекстном меню [источника
				 данных](../UiReport_Source.htm) на панели «Источники».
				 К источнику будет автоматически добавлен новый срез, а
				 на лист отчёта будет добавлен выбранный визуализатор;


		- с помощью вкладки «Вставка»
		 панели инструментов для диаграммы:


			- Выделите пустую ячейку на листе отчёта. Левая верхняя
			 граница выделенной ячейки будет левой верхней границей диаграммы.


			- Выберите тип диаграммы на вкладке «Вставка»
			 панели инструментов.


	В результате выполнения действий будет создан срез данных. Новому
	 срезу по умолчанию будет присвоено имя «Срез
	 N», где N - порядковый
	 номер среза.


	Срез необходим для создания [визуализаторов](../../Reports/Visualization.htm).


	Примечание.
	 Для [источников данных](../UiReport_Source.htm#addsource),
	 состоящих из двух и более кубов, доступно добавление только одного
	 среза данных.


[![](../../../opened.gif)![](../../../closed.gif)Добавление
 визуализатора](javascript:TextPopup(this))


	При работе со срезом данных доступно создание следующих визуализаторов:
	 [таблица](../../Table/UiReport_tables_appointment.htm),
	 [диаграмма](../../Diagrams/UiReport_Diagrams_appointment.htm).


[![](../../../opened.gif)![](../../../closed.gif)Удаление
 среза данных](javascript:TextPopup(this))


	Для удаления среза данных выполните команду «Удалить»
	 в контекстном меню выбранного среза. После выполнения команды появится
	 окно подтверждения действия. При нажатии на кнопку «Да»
	 выбранный срез будет удалён.


	При удалении срезов данных удаляются все дочерние объекты. Для среза
	 удаляются все таблицы и диаграммы, построенные его основе.


[![](../../../opened.gif)![](../../../closed.gif)Переименование
 среза данных](javascript:TextPopup(this))


	Для переименования среза данных выполните команду «Переименовать»
	 в контекстном меню выбранного среза. После выполнения команды введите
	 новое наименование и нажмите клавишу ENTER.


[![](../../../opened.gif)![](../../../closed.gif)Настройка алгоритмов
 расчёта](javascript:TextPopup(this))


	Для расчёта и контроля данных из среза данных используйте готовые
	 алгоритмы расчёта и [задайте их
	 настройки](Calculation_Practices.htm).
