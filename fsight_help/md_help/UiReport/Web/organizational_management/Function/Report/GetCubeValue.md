# GetCubeValue


## Синтаксис


GetCubeValue(DataSource, Dimension)


## Параметры


DataSource. Номер или идентификатор
 источника данных;


Dimension. Доступно только
 после выбора параметра DataSource.
 Номер, наименование или идентификатор измерения в срезе. Наименование
 параметра соответствует наименованию измерения в кубе. Количество параметров
 в формуле зависит от количества измерений в кубе.


## Описание


Возвращает значение из куба.


## Комментарии


После применения функции формула в ячейке имеет вид:


=GetCubeValue ("<Идентификатор контейнера1>.<Идентификатор
 контейнера2>.<Идентификатор куба>";"<Идентификатор
 измерения_1>.[<Уникальный идентификатор элемента>]| <Идентификатор
 измерения_2>.[<Уникальный идентификатор элемента>]")


Для справочников НСИ уникальным идентификатором элемента является ключ,
 для табличного и календарного справочников - идентификатор.


Если у табличного справочника атрибут, назначенный в качестве идентификатора,
 не совпадает с атрибутом, на который установлен индекс первичного ключа,
 то идентификатор должен быть назначен на атрибут, соответствующий этому
 первичному ключу.


Примечание.
 Идентификатор контейнера указывается при его наличии.


## Пример

		 Результат
		 Описание


		 =GetCubeValue("CUBE_SEP", "D_FACTS.[1]|D_CALENDAR.[YEARS:2002]|DIC_SEP.[30100]|DIC_RF.[11]|DIC_SOURCE.[1]")

			- D_FACTS.[1] - элемент «Значение»
			 измерения «Факты»;


			- D_CALENDAR.[YEARS:2002] - элемент «2002»
			 измерения «Календарь»;


			- DIC_SEP.[30100] - элемент «Объём
			 промышленной продукции в фактических ценах предприятий, млн.руб.
			 в год» измерения «Справочник
			 социально-экономических показателей»;


			- DIC_RF.[11] - элемент «Архангельская
			 область» измерения «Территориальные
			 образования»;


			- DIC_SOURCE.[1] - элемент «Данные
			 госкомстата» измерения «Типы
			 данных».
