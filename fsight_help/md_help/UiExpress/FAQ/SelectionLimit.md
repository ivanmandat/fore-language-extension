# Как ограничить чрезмерно большую отметку элементов измерения?


Для предотвращения перегрузки системы при отметке большого количества
 элементов в «Форсайт. Аналитическая платформа»
 доступно ограничение максимального количества отмеченных элементов.


Ограничение отметки задается в одном из подразделов реестра:


	- [HKEY_CURRENT_USER\Software\Foresight\Foresight Analytics Platform\10.0\Pivot]
	 - подраздел для настроек текущего пользователя;


	- [HKEY_LOCAL_MACHINE\Software\Foresight\Foresight Analytics Platform\10.0\Pivot] -
	 подраздел для настроек для всех пользователей.


Примечание.
 Подраздел Pivot необходимо создать.


В подразделе Pivot необходимо создать один из параметров типа REG_DWORD:


	- SelectionLimit. Задает
	 ограничение по количеству образованных декартовых произведений при
	 отмеченных элементах измерений, в том числе фиксированных;


	- MatrixLimit. Задает
	 ограничение по размеру матрицы - количеству непустых ячеек.


Примечание.
 Ограничение применяется без учета фильтрации.


При превышении заданного ограничения вместо таблицы будет выводиться
 сообщение о превышении ограничения.


## Ограничение отметки элементов измерения объектов наблюдения


Ограничение отметки можно задать на количество элементов [измерения объектов
 наблюдения](UiSelection.chm::/Selection/Dimension_type.htm).


Ограничение отметки задается в одном из подразделов реестра:


	- [HKEY_CURRENT_USER\Software\Foresight\Foresight Analytics Platform\10.0\KeExpress] -
	 подраздел для настроек текущего пользователя, не зависимо от разрядности
	 системы;


	- [HKEY_LOCAL_MACHINE\Software\Foresight\Foresight Analytics Platform\10.0\KeExpress] -
	 подраздел для настроек в случае, когда разрядность «Форсайт. Аналитическая платформа»
	 совпадает с разрядностью операционной системы. Для всех пользователей.


Примечание.
 Если подраздел KeExpress отсутствует,
 то его необходимо создать самостоятельно.


В подразделе KeExpress необходимо
 создать параметр ObjectivesSelectionLimit
 типа REG_DWORD.


Примечание.
 По умолчанию, если не задан параметр, ограничение отметки измерения объектов
 наблюдения составляет 2000 элементов.


## Ограничение отметки элементов
 измерения рядов данных диаграммы


Ограничение отметки можно задавать на количество отображаемых [рядов данных диаграммы](uidiagrams.chm::/Series.htm).


Ограничение отметки задается в одном из подразделов реестра, в зависимости
 от используемых инструментов:


	- Для инструментов «Отчёты»,
	 «Аналитические запросы (OLAP)»
	 и «Аналитические панели»:


		- [HKEY_CURRENT_USER\Software\Foresight\Foresight Analytics Platform\10.0\Chart]
		 - подраздел для настроек текущего пользователя;


		- [HKEY_LOCAL_MACHINE\Software\Foresight\Foresight Analytics Platform\10.0\Chart] -
		 подраздел для настроек для всех пользователей.


Примечание.
 Если подраздел Chart отсутствует,
 то его необходимо создать самостоятельно.


В подразделе Chart
 необходимо создать параметр SeriesLimit
 типа REG_DWORD.


Примечание.
 По умолчанию, если не задан параметр, на диаграмме отображается не более
 99 рядов данных.


	- Для инструментов «Аналитические
	 запросы (OLAP)» и «Аналитические
	 панели»:


		- [HKEY_CURRENT_USER\Software\Foresight\Foresight Analytics Platform\10.0\ExpressAnalyze] - подраздел
		 для настроек текущего пользователя;


		- [HKEY_LOCAL_MACHINE\Software\Foresight\Foresight Analytics Platform\10.0\ExpressAnalyze] -
		 подраздел для настроек для всех пользователей.


Примечание.
 Если подраздел ExpressAnalyze
 отсутствует, то его необходимо создать самостоятельно.


В подразделе ExpressAnalyze
 необходимо создать параметр ChartSeriesLimit
 типа REG_DWORD.


Примечание.
 По умолчанию, если не задан параметр, на диаграмме отображается не более
 99 рядов данных.
