# Настройка очередности выполнения операций


Задача ETL может содержать несколько схем преобразования данных. Для
 обеспечения корректного выполнения всех операций следует явно указать
 их очередность.


Для настройки очередности выполнения операций выполните команду «Задача > Очередность выполнения операций»
 в главном меню. Будет
 открыт диалог:


![](UiEtl%20_queue.gif)


Для изменения порядка выполнения операций используйте кнопки «Вниз» и «Вверх».


## Настройка состава объектов выполняемой задачи


Для выполнения задачи ETL можно включить/выключить цепочки операций
 и отдельные объекты рабочей области:


[![](../../Opened.gif)![](../../Closed.gif)Включение/выключение
 операции из выполнения задачи ETL](javascript:TextPopup(this))


	Для включения/выключения выбранной операции из выполнения задачи
	 ETL:


		- нажмите кнопку «Включить/выключить»;


		- установите/снимите флажок напротив наименования операции.


	После выполнения одного из действий выбранная операция сменит свое
	 состояние на противоположное, все объекты цепочки, исключенной из
	 выполнения задачи ETL, на рабочей области будут отображаться полупрозрачными.


	По умолчанию все операции отмечены на выполнение.


[![](../../Opened.gif)![](../../Closed.gif)Включение/выключение
 объекта рабочей области](javascript:TextPopup(this))


	Для включения/выключения объекта рабочей области из цепочки операций
	 выполните команду контекстного меню «Включить
	 в расчет задачи ETL».


	По умолчанию флажок установлен, то есть объект включен в расчет
	 задачи ETL. Если часть объектов цепочки была исключена из выполнения
	 задачи ETL в результате исключения приёмника, то у них данный флажок
	 будет деактивирован.


	Из цепочки операций можно исключить следующие объекты:


		- все типы источников, если после них следуют преобразователи
		 данных «[Объединение](../03_ETLObjects/04_Transformers/02_Union/uietl_union.htm)»
		 и «[Соединение](../03_ETLObjects/04_Transformers/03_Joint/uietl_joint.htm)»;


		- все типы приёмников, если приёмнику предшествовал преобразователь
		 данных «[Разделение](../03_ETLObjects/04_Transformers/01_Split/uietl_split.htm)».
		 После исключения таких приёмников из цепочки расчета происходит
		 автоматическое исключение объектов, следующих за данным приёмником
		 в цепочке операций, для которых этот приёмник является источником
		 и которые таким образом не имеют иных источников;


		- все объекты типа «[Преобразователь](../03_ETLObjects/04_Transformers/UiEtl_Trfs.htm)».


	Если исключаемый объект находится в середине цепочки операций, при
	 выполнении задачи связи будут автоматически перестроены.


	У выключенных объектов становится недоступным редактирование связей.
