# Работа с готовой задачей ETL


Перед выполнением задачи ETL при необходимости можно задать следующие
 настройки:


	- [очерёдность выполнения](UiETL_Property_queue.htm)
	 операций;


	- протоколирование выполняемых операций в [журнал](../04_Work/04_Jurnal/UiEtl_Jurnal.htm);


	- указание объекта среды разработки, который будет использоваться
	 в качестве [обработчика
	 ошибок](../06_CreateETL/Event_Handling.htm).


Для выполнения задачи ETL нажмите кнопку «Выполнить».
 При успешном выполнении задачи ETL
 будет выдано сообщение, отображающее количество обработанных записей и
 время выполнения задачи.


Примечание.
 Выполнение задачи ETL может быть автоматизировано с помощью [контейнера запланированных
 задач](UiAppSrv.chm::/Getting_Started.htm).


## Ошибки при выполнении задачи ETL


Если при выполнении задачи ETL возникают ошибки, то будет выдано сообщение.
 Объем информации в сообщении зависит от настроек обработки ошибок, задаваемых
 в объектах задачи ETL:


	- если выбрано действие «Остановка
	 расчета задачи», будет выдано сообщение, содержащее:


	-


		- номер записи с ошибкой;


		- поле с ошибкой;


		- значение с ошибкой;


		- тип ошибки;


	- если выбрано «Пропуск только
	 некорректных записей» или «Пропуск
	 всех записей в рамках одной транзакции», в окне с результатами
	 выполнения задачи будет отображено:


		- количество пропущенных записей;


		- количество выполненных записей;


		- время выполнения задачи.
